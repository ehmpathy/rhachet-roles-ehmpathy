**context from research:**

the test uses mock gh cli binaries to simulate workflow state transitions. the test:
1. sets up scene with `releasePr: 'merged'`, `tagWorkflows: 'inflight'`, `transitions: true`
2. runs `git.release --watch`
3. expects the skill to watch and eventually exit 0 when tag workflows pass

exit code 1 = malfunction (poll timed out or failed to detect transition)

likely causes:
1. **race condition** — mock transition time is not deterministic
2. **file I/O delay** — mock state files not flushed before read
3. **parallel test interference** — shared temp dirs or state

the test uses `setupScene` helper with unique slugs, so parallelism should be isolated.
