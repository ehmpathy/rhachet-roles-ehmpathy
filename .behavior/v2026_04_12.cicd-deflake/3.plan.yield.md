## flake 2: git.release [row-25] watch with transitions

**root cause confirmed:** counter increments per gh call, not per poll cycle

**repair steps:**

1. [ ] mockGh.ts: use separate counters per gh subcommand
   - instead of single `gh_pr_view_count` file, use:
     - `.mockstate/gh.pr.view.count` for `gh pr view`
     - `.mockstate/gh.run.list.count` for `gh run list`
   - each subcommand reads/increments its own counter

2. [ ] alternative: use iteration-based counter
   - track "poll iteration" explicitly
   - all gh commands in one iteration share the same sequence index
   - increment only after full poll cycle completes

3. [ ] watch_sequence.json: simplify to single sequence per state
   - instead of per-command sequences, use per-iteration states
   - each poll iteration reads the same state across all gh commands

**verification criteria:**
- local: run `npm run test:integration -- git.release.p3.scenes.on_main.into_prod` 10x
- ci: 3x consecutive green runs on main branch
- guard: mock pattern documented with "one counter per sequence, not per command"

**tips for future travelers:**
- watch loops that poll multiple gh commands need coordinated mock state
- counter per command = race; counter per iteration = stable
- mock state files should reflect the logical unit (poll cycle), not the physical unit (gh call)
