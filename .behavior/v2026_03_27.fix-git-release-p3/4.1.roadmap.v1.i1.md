## phase 3: and_then_await operation

**depends on**: phase 1 (output functions), phase 2 (operations)

create `git.release._.and_then_await.sh` with poll loop and timeout logic.

### read before phase

- `.behavior/v2026_03_27.fix-git-release-p3/3.3.1.blueprint.product.v1.i1.md` — codepath tree (lines 25-50)

### tasks

- [ ] create `git.release._.and_then_await.sh`
- [ ] implement `and_then_await()` function
  - [ ] emit `🫧 and then...`
  - [ ] poll loop with 90s timeout (5s interval first 60s, then 15s)
  - [ ] call get_fresh_release_pr or get_fresh_release_tag based on artifact_type
  - [ ] emit poll lines via print_await_poll
  - [ ] on found: emit success via print_await_result, set AWAIT_RESULT, exit 0
  - [ ] on timeout: emit anchor, lookup workflow status, emit diagnostics, exit 2

### acceptance criteria

```
given(artifact found immediately)
  when(and_then_await is called)
    then(output shows `🫧 and then...` followed by blank line)
    then(AWAIT_RESULT contains artifact json)
    then(exit 0)

given(artifact found after 2 polls)
  when(and_then_await is called)
    then(output shows `🫧 and then...` with poll sub-branches)
    then(poll lines show cumulative elapsed time)
    then(final line shows `✨ found! after Xs`)
    then(exit 0)

given(artifact not found within 90s)
  when(and_then_await is called)
    then(output shows `⚓ {artifact} did not appear in 90s`)
    then(output shows `🔴 release-please` with workflow status)
    then(exit 2)
```

### verification

test with mocked gh and git to simulate found/not-found scenarios.
