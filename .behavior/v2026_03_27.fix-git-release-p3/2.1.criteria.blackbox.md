# blackbox criteria: and_then_await

## usecase.1 = artifact found immediately

the artifact appears on first check — no wait needed.

```
given(artifact is fresh and available on first check)
  when(and_then_await is invoked)
    then(output shows `🫧 and then...` followed by blank line)
      sothat(user sees clean transition to next transport)
    then(next transport output follows immediately)
      sothat(flow continues without poll clutter)
    then(exit 0)
```

## usecase.2 = artifact found after wait

the artifact appears after one or more poll cycles.

```
given(artifact is not available on first check)
  when(and_then_await polls and artifact appears)
    then(output shows `🫧 and then...` with poll sub-branches)
      sothat(user sees progress in wait)
    then(poll lines show cumulative elapsed time: `├─ 💤 5s in await`, `├─ 💤 10s in await`)
      sothat(user knows how long the wait has been)
    then(final line shows `└─ ✨ found! after Xs`)
      sothat(user knows the wait succeeded)
    then(next transport output follows)
    then(exit 0)
```

## usecase.3 = artifact timeout with workflow diagnostics

the artifact never appears within 90s — show why.

```
given(artifact does not appear within 90s)
  when(and_then_await times out)
    then(output shows poll lines followed by `└─ ⚓ {artifact} did not appear in 90s`)
      sothat(user knows timeout occurred)
    then(output shows workflow status: `└─ 🔴 release-please` with url and status)
      sothat(user can investigate the workflow)
    then(exit 2)
      sothat(caller knows this is a constraint error)

given(release-please workflow failed)
  when(timeout occurs)
    then(workflow status shows `failed`)

given(release-please workflow is still active)
  when(timeout occurs)
    then(workflow status shows `in_progress`)

given(release-please workflow passed but artifact absent)
  when(timeout occurs)
    then(workflow status shows `passed`)
      sothat(user knows workflow ran but artifact was not created)

given(release-please workflow not found)
  when(timeout occurs)
    then(workflow status shows `not found`)
```

## usecase.4 = stale artifact rejection (critical regression)

old artifacts from prior runs must be rejected.

```
given(old artifact exists from a prior merge)
  when(new merge occurs and and_then_await is invoked)
    then(stale artifact is rejected via commit ancestry check)
      sothat(we never pick up artifacts from prior runs)
    then(poll continues until fresh artifact appears or timeout)
      sothat(user gets the correct artifact for THIS merge)

given(old release PR exists, new merge commit M1)
  when(and_then_await checks the old PR)
    then(old PR is rejected because M1 is not ancestor of old PR head)
    then(poll continues)

given(old release PR is updated with new commit after M1)
  when(and_then_await checks the updated PR)
    then(updated PR is accepted because M1 is ancestor of new PR head)
      sothat(PRs updated by release-please are correctly detected as fresh)
```

## usecase.5 = transport-specific behavior

different transports have different artifact names.

```
given(await release PR after feature merge)
  when(timeout occurs)
    then(message shows "release pr did not appear in 90s")

given(await tag after release PR merge)
  when(timeout occurs)
    then(message shows "tag v{version} did not appear in 90s")
      sothat(user sees the expected tag version)
```
