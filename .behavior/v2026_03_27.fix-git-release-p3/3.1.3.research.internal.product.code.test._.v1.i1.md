## pattern 3: runSkill wrapper [REUSE]

**source**: `git.release.p3.scenes.on_feat.into_main.integration.test.ts:97-125`

runSkill executes the skill with test mode env vars.

```typescript
// citation 3: runSkill pattern (lines 97-115)
const runSkill = (
  args: string[],
  env: { tempDir: string; fakeBinDir: string },
): { stdout: string; stderr: string; status: number } => {
  const result = spawnSync('bash', [SKILL_PATH, ...args], {
    cwd: env.tempDir,
    encoding: 'utf-8',
    env: {
      ...process.env,
      PATH: `${env.fakeBinDir}:${process.env.PATH}`,
      GIT_RELEASE_TEST_MODE: 'true',
      HOME: env.tempDir,
    },
    stdio: ['pipe', 'pipe', 'pipe'],
  });

  return {
    stdout: result.stdout ?? '',
    stderr: result.stderr ?? '',
    status: result.status ?? 1,
  };
};
```

**relation to wish**: p4 tests will [REUSE] this pattern unchanged. the GIT_RELEASE_TEST_MODE=true flag triggers instant poll behavior and safety limits.

## pattern 4: asTimeStable normalizer [REUSE]

**source**: `git.release.p3.scenes.on_feat.into_main.integration.test.ts:127-145`

```typescript
// citation 4: asTimeStable function (lines 127-140)
const asTimeStable = (output: string): string => {
  return output
    .replace(/\d+s in action/g, 'Xs in action')
    .replace(/\d+s watched/g, 'Xs watched')
    .replace(/\d+s in await/g, 'Xs in await')
    .replace(/after \d+s/g, 'after Xs')
    .replace(/in \d+s/g, 'in Xs');
};
```

**relation to wish**: p4 tests will [EXTEND] this with new patterns:
- `\d+s in await` → `Xs in await` (poll lines)
- `after \d+s in wait` → `after Xs in wait` (found message)
- `in \d+s` → `in Xs` (timeout message)

```typescript
describe('git.release.p3', () => {
  given('[case1] feature PR ready for merge', () => {
    const setup = useBeforeAll(() => setupScene({
      scene: { /* mock data */ },
      slug: 'case1-feat-ready',
    }));

    when('[t0] --mode apply is invoked', () => {
      const result = useThen('skill executes', () =>
        runSkill(['--mode', 'apply'], setup),
      );

      then('exit code is 0', () => {
        expect(result.status).toBe(0);
      });

      then('output matches snapshot', () => {
        expect(asTimeStable(result.stdout)).toMatchSnapshot();
      });
    });
  });
});
```

**relation to wish**: p4 tests will [REUSE] this structure for all 18 test matrix cases.

### poll evolution mock

```bash
# mock that changes response over time
CALL_COUNT_FILE="$STATE_DIR/pr_list_calls"
if [[ ! -f "$CALL_COUNT_FILE" ]]; then
  echo "0" > "$CALL_COUNT_FILE"
fi

echo $((CALL_COUNT + 1)) > "$CALL_COUNT_FILE"

if [[ $CALL_COUNT -lt 2 ]]; then
  echo "[]"  # not found yet
else
  echo '[{"number": 123, "headRefOid": "fresh123", ...}]'  # found
fi
```
