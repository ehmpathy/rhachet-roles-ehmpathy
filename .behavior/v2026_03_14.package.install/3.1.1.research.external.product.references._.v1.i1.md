# research: external product references

## summary

this research catalogs reference materials, terms, concepts, and demos that ground our knowledge for `set.package.install` and `set.package.upgrade` skills.

---

## reference materials

### [REF-1] npm registry api documentation
**source**: https://github.com/npm/registry/blob/main/docs/REGISTRY-API.md
**date**: 2024
**credibility**: [official]

> "GET /{package} - Returns the packument for {package}. The packument contains all the metadata for the package."

**terms established**:
- **packument**: the complete package document (metadata, versions, dist-tags)
- **dist-tags**: named version aliases (e.g., `latest`, `next`)

**leverage**: use `GET /{package}` to fetch metadata and expand @latest to actual version.

## concept map

```
npm registry
├── registry.npmjs.org (public api)
│   ├── GET /{package} → packument (metadata, versions, dist-tags)
│   ├── /-/npm/v1/security/advisories/bulk → vulnerability check
│   └── /-/v1/search → package search
├── github advisory database
│   ├── CVE identifiers
│   ├── GHSA identifiers
│   └── severity levels
└── lockfiles
    ├── package-lock.json (npm)
    └── pnpm-lock.yaml (pnpm)
```

## demos and examples

### demo: fetch package metadata
```bash
curl https://registry.npmjs.org/zod | jq '.["dist-tags"].latest'
# returns: "3.24.0" (or current latest)
```

### demo: bulk advisory check
```bash
curl -X POST https://registry.npmjs.org/-/npm/v1/security/advisories/bulk \
  -H "Content-Type: application/json" \
  -d '{"zod":["3.22.4"]}'
# returns: {} (empty = no vulnerabilities)
```

### demo: pnpm add with dev flag
```bash
pnpm add -D jest@29.0.0
# adds to devDependencies
```

### demo: npm install with save-dev
```bash
npm install --save-dev jest@29.0.0
# adds to devDependencies
```
