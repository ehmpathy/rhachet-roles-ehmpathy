# blackbox criteria: set.package.install + set.package.upgrade

## usecase.1 = install new package

```
given(mechanic wants to add a new package)
  when(all required flags provided: --package, --at, --for, --reason)
    when(security audit passes)
      then(reason persisted to .route/v{date}.package.install/3.reason.for_{package}.v1.i1.md)
        sothat(audit trail exists for why this package was added)
      then(package installed to project)
        sothat(mechanic can use the package)
      then(stdout shows success with turtle vibes)
        sothat(mechanic knows it worked)

    when(security audit fails)
      then(install blocked)
        sothat(vulnerable packages never enter codebase)
      then(stdout shows vulnerabilities found)
        sothat(mechanic knows why it was blocked)
      then(no reason file persisted)
        sothat(.route/ only contains successful installs)

  when(--package flag omitted)
    then(error: --package required)
      sothat(mechanic knows what to fix)

  when(--at flag omitted)
    then(error: --at required)
      sothat(explicit version pinned)

  when(--for flag omitted)
    then(error: --for required)
      sothat(prod vs prep distinction enforced)

  when(--reason flag omitted)
    then(error: --reason required)
      sothat(audit trail enforced)

  when(--for value invalid)
    then(error: --for must be prod or prep)
      sothat(only valid dependency types allowed)

  when(package already installed)
    then(error: package already installed, use set.package.upgrade)
      sothat(install vs upgrade distinction clear)
```

## usecase.2 = upgrade extant package

```
given(mechanic wants to upgrade an extant package)
  when(all required flags provided: --package, --to)
    when(package is installed)
      when(security audit passes for target version)
        then(package upgraded to target version)
          sothat(mechanic has newer/different version)
        then(stdout shows success with turtle vibes)
          sothat(mechanic knows it worked)

      when(security audit fails for target version)
        then(upgrade blocked)
          sothat(vulnerable versions never enter codebase)
        then(stdout shows vulnerabilities found)
          sothat(mechanic knows why it was blocked)

    when(package not installed)
      then(error: package not installed, use set.package.install)
        sothat(upgrade vs install distinction clear)

  when(--package flag omitted)
    then(error: --package required)

  when(--to flag omitted)
    then(error: --to required)
      sothat(explicit target version enforced)

  when(--to @latest provided)
    then(latest version looked up and used)
      sothat(@latest convenience supported)
```

## usecase.3 = pnpm vs npm detection

```
given(mechanic invokes either skill)
  when(package-lock.json extant in repo)
    then(npm used for install/upgrade)
      sothat(repo package manager respected)

  when(package-lock.json not extant)
    then(pnpm used for install/upgrade)
      sothat(pnpm is default per wisher preference)
```

## usecase.4 = reason input methods

```
given(mechanic provides reason via @stdin)
  when(heredoc piped to skill)
    then(reason content read from stdin)
    then(reason persisted to .route/)
      sothat(multi-line detailed reasons supported)

given(mechanic provides reason inline)
  when(--reason "inline text" provided)
    then(reason content taken from flag value)
    then(reason persisted to .route/)
      sothat(quick simple reasons supported)
```

## usecase.5 = @latest version expansion

```
given(mechanic specifies --at @latest for install)
  when(skill runs)
    then(actual version number looked up from registry)
    then(actual version recorded in reason file)
      sothat(audit trail has pinned version, not "@latest")
    then(actual version installed)
      sothat(reproducibility maintained)

given(mechanic specifies --to @latest for upgrade)
  when(skill runs)
    then(actual version number looked up from registry)
    then(actual version installed)
      sothat(reproducibility maintained)
```
