## patterns identified

### [PATTERN-1] skill header structure
**source**: `src/domain.roles/mechanic/skills/get.package.docs.sh:1-20`
**mark**: [REUSE]

```bash
#!/usr/bin/env bash
######################################################################
# .what = fetch package documentation (readme or filetree)
#
# .why  = enables quick access to package docs without browser:
#         - read usage patterns before code
#         - discover available features
#         - avoid boilerplate from misuse
#
# usage:
#   get.package.docs.sh readme --of iso-price
#   get.package.docs.sh filetree --of domain-objects
#
# guarantee:
#   - checks node_modules first (local, version-matched)
#   - caches npm api results to .refs/get.package.docs/
#   - fail-fast on errors
######################################################################
set -euo pipefail
```

**relevance**: all skills follow this header pattern with `.what`, `.why`, `usage`, `guarantee` sections.

### [PATTERN-2] argument parse with rhachet passthrough
**source**: `src/domain.roles/mechanic/skills/get.package.docs.sh:30-56`
**mark**: [REUSE]

```bash
while [[ $# -gt 0 ]]; do
  case $1 in
    readme|filetree)
      SUBCOMMAND="$1"
      shift
      ;;
    --of)
      PACKAGE="$2"
      shift 2
      ;;
    --repo|--role|--skill)
      # rhachet passthrough args - ignore
      shift 2
      ;;
    --)
      shift
      ;;
    *)
      echo "⛈️  error: unknown argument: $1" >&2
      exit 2
      ;;
  esac
done
```

**relevance**: all skills must ignore `--repo`, `--role`, `--skill` args that rhachet passes.

### [PATTERN-15] bulk advisory check
**source**: vision requires this, no extant impl
**mark**: [EXTEND]

```bash
# proposed pattern based on research
_check_security() {
  local package="$1"
  local version="$2"
  local response
  response=$(curl -sf -X POST https://registry.npmjs.org/-/npm/v1/security/advisories/bulk \
    -H "Content-Type: application/json" \
    -d "{\"$package\":[\"$version\"]}" 2>/dev/null) || {
    return 1  # fail-closed on network error
  }
  # empty object {} means no vulnerabilities
  if [[ "$response" == "{}" ]]; then
    return 0  # safe
  else
    echo "$response"
    return 1  # vulnerable
  fi
}
```

**relevance**: new pattern needed for pre-install security audit.
