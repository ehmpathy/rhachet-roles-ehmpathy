# blueprint: set.package.install + set.package.upgrade

## summary

create two bash skills for secure, documented package management:

1. **set.package.install.sh** — install new package with security audit and reason documentation
2. **set.package.upgrade.sh** — upgrade extant package with security audit

both skills:
- use npm bulk advisory endpoint for pre-install security check (no temp dir)
- expand @latest to actual semver via `npm view`
- detect package manager via package-lock.json presence
- output turtle vibes tree format

```
set.package.install
├── [←] _check_dependencies() → from set.package.operations.sh

│   ├── --at       → VERSION
│   ├── --for      → DEP_TYPE (prod|prep)
│   ├── --reason   → REASON (@stdin or inline)
│   └── --repo/--role/--skill → ignore (rhachet passthrough)
├── [+] validate_flags()
│   ├── fail if PACKAGE empty
│   ├── fail if VERSION empty
│   ├── fail if DEP_TYPE empty or invalid
│   └── fail if REASON empty
├── [←] _check_package_installed() → from set.package.operations.sh
│   └── fail if package already installed → suggest upgrade
├── [←] _expand_version() → from set.package.operations.sh
│   └── if @latest → npm view $PACKAGE version
├── [←] _check_security() → from set.package.operations.sh
│   ├── POST to /-/npm/v1/security/advisories/bulk
│   ├── if response != {} → fail, show vulnerabilities
│   └── if curl error → fail closed
├── [+] _persist_reason()
│   ├── SAFE_NAME = _sanitize_package_name($PACKAGE)
│   ├── mkdir -p .route/v{isodate}.package.install/

├── [←] _detect_package_manager() → from set.package.operations.sh
│   └── package-lock.json extant? npm : pnpm
├── [+] _install_package()

│   └── if pnpm → pnpm add {-D}? $pkg@$ver
```

### set.package.operations.sh (shared)

```
set.package.operations
├── [+] _check_dependencies()
│   ├── verify jq available → fail with "jq required" if absent
│   ├── verify curl available → fail with "curl required" if absent
│   └── verify npm available → fail with "npm required" if absent
├── [+] _sanitize_package_name()
│   └── replace "/" with "__" for file paths (e.g., @org/pkg → @org__pkg)
├── [+] _expand_version()
│   ├── if version starts with "@" (e.g., @latest, @next, @beta)
│   │   └── npm view $PACKAGE$VERSION version
│   └── else → return as-is
├── [+] _check_security()
│   ├── POST to bulk advisory endpoint
│   ├── parse response with jq
│   ├── if {} → return 0 (safe)
│   └── if non-empty → return 1 (vulnerable), print details
├── [+] _detect_package_manager()
│   └── [[ -f "$REPO_ROOT/package-lock.json" ]] && echo npm || echo pnpm
└── [+] _check_package_installed()
    └── jq '.dependencies["$pkg"] // .devDependencies["$pkg"]' package.json
```

## contracts

### set.package.install

**input:**
```
--package <name>      required, package name
--at <version>        required, semver or @latest
--for <type>          required, prod | prep
--reason <text>       required, @stdin or "inline text"
```

**output (success):**
```
🐢 shell yeah!

🐚 set.package.install
   ├─ package: <name>
   ├─ version: <version>
   ├─ for: <type>
   ├─ security
   │  └─ audit: passed (0 vulnerabilities)
   ├─ reason
   │  └─ saved: .route/v<date>.package.install/3.reason.for_<name>.v1.i1.md
   └─ install
      └─ <pm> add <name>@<version>: success
```

**output (blocked by security):**
```
🐢 bummer dude...

🐚 set.package.install
   ├─ package: <name>
   ├─ version: <version>
   ├─ for: <type>
   └─ security
      └─ audit: FAILED (<n> vulnerabilities)
         ├─ <severity>: <title> (<cve>)
         └─ ...

install blocked. choose a safer package or version.
```

### set.package.upgrade

**input:**
```
--package <name>      required, package name
--to <version>        required, semver or @latest
```

**output (success):**
```
🐢 righteous!

🐚 set.package.upgrade
   ├─ package: <name>
   ├─ from: <old-version>
   ├─ to: <new-version>
   ├─ security
   │  └─ audit: passed (0 vulnerabilities)
   └─ upgrade
      └─ <pm> add <name>@<version>: success
```
