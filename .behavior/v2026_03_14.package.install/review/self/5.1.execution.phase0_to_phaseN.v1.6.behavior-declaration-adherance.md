## edge case adherance

### scoped packages (@org/pkg)

**vision implies:** works with any valid npm package name
**blueprint specifies (line 114-115):** `_sanitize_package_name()` replaces "/" with "__"
**implementation (operations.sh:38-41):**
```bash
_sanitize_package_name() {
  local package="$1"
  echo "$package" | sed 's/\//__/g'
}
```

**why it holds:** `@types/node` becomes `@types__node` in file paths. this prevents directory traversal issues where `/` would create subdirectories.

### @latest/@next/@beta tags

**vision specifies (line 230):** "@latest passed as version" → "allowed — skill expands to actual version"
**implementation (operations.sh:52-61):**
```bash
if [[ "$version" == @* ]]; then
  local tag="${version#@}"
  actual=$(npm view "${package}@${tag}" version 2>/dev/null)
```

**why it holds:** any dist-tag that starts with `@` (e.g., @latest, @next, @beta, @canary) is expanded via `npm view`. the `${version#@}` strips the initial `@` to get the tag name.

### security check network failure

**blueprint specifies (line 67):** "if curl error → fail closed"
**implementation (operations.sh:83-92):**
```bash
response=$(curl -s -X POST ... 2>/dev/null) || curl_status=$?

if [[ -n "${curl_status:-}" || -z "$response" ]]; then
  echo "error: security check failed (network error)" >&2
  return 1
fi
```

**why it holds:** if curl fails (network error), `curl_status` is set. if response is empty (timeout), we also fail. fail-closed means no install without a clean check.
