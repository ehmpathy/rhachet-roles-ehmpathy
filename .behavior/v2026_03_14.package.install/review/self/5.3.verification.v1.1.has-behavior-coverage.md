## why it holds

all behaviors from wish and vision are covered by integration tests:

1. **flag validation** — each required flag has a dedicated case for omission
2. **security gate** — both install and upgrade have dedicated failure cases
3. **reason persistence** — verified via filesystem assertions
4. **package manager detection** — separate cases for npm and pnpm repos
5. **version expansion** — @latest tested for both install and upgrade
6. **input methods** — both @stdin and inline reason tested
7. **state guards** — install-if-extant and upgrade-if-absent both tested

total: 40 tests across 20 cases cover all behavioral requirements.
