### 6. flag naming

**blueprint**: `--package`, `--at`, `--for`, `--reason`, `--to`

**extant patterns**:
- `get.package.docs.sh` uses `--of` for package name
- `git.commit.set.sh` uses `-m` for message
- `sedreplace.sh` uses `--old`, `--new`, `--glob`, `--mode`

**analysis**: `--package` is more explicit than `--of`. both work, but `--package` is self-documenting when reading invocation. `--at`, `--for`, `--to` are prepositions that read naturally in context ("install X --at Y --for Z").

**verdict**: acceptable divergence — `--package` clearer than `--of` for this usecase. no conflict with extant skills.
