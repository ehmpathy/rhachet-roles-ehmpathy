### note: timeout handle

blueprint does not specify explicit timeouts for curl or npm view calls. this is acceptable because:
1. curl/npm have built-in defaults (connect timeout, etc.)
2. fail-fast pattern means errors surface quickly
3. explicit timeouts would add complexity without clear benefit

decision: no change required — trust tool defaults.
