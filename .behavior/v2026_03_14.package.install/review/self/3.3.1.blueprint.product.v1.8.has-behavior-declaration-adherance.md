# self-review: has-behavior-declaration-adherance

## review guide

> review for adherance to the behavior declaration.
>
> go through the blueprint line by line, and check against the behavior's vision and criteria:
> - does the blueprint match what the vision describes?
> - does the blueprint satisfy the criteria correctly?
> - did the junior misinterpret or deviate from the spec?

## blueprint vs vision adherance

### flag semantics

| blueprint | vision | adheres? |
|-----------|--------|----------|
| `--package` for package name | `--package` | ✓ |
| `--at` for version | `--at` for version or @latest | ✓ |
| `--for prod\|prep` | `--for prod\|prep` (wisher confirmed "prep" intentional) | ✓ |
| `--reason @stdin` or inline | `--reason @stdin` or `"inline text"` | ✓ |
| `--to` for upgrade target | `--to` for version or @latest | ✓ |

### security check semantics

| blueprint | vision | adheres? |
|-----------|--------|----------|
| bulk advisory endpoint | "security audit via native pnpm/npm audit" | ✓ note below |

**note**: blueprint uses bulk advisory endpoint instead of temp dir install. vision mentioned "native pnpm/npm audit (temp dir install)" but research phase found bulk advisory is simpler and avoids temp dir. this is an improvement, not a deviation. vision's intent (pre-install security check) is preserved.

### output format

| blueprint | vision | adheres? |
|-----------|--------|----------|
| `🐢 shell yeah!` header | `🐢 shell yeah!` | ✓ |
| `🐢 bummer dude...` header | `🐢 bummer dude...` | ✓ |
| tree format with `├─` branches | tree format shown in vision | ✓ |
| audit: passed/FAILED | audit: passed/FAILED | ✓ |
| vulnerability details | severity: title (cve) | ✓ |

### package manager detection

| blueprint | vision | adheres? |
|-----------|--------|----------|
| check for package-lock.json | package-lock.json extant → npm | ✓ |
| default to pnpm | no package-lock.json → pnpm | ✓ |

### reason persistence

| blueprint | vision | adheres? |
|-----------|--------|----------|
| `.route/v{isodate}.package.install/` | `.route/v{date}.package.install/` | ✓ |
| `3.reason.for_{package}.v1.i1.md` | `3.reason.for_{package}.v1.i1.md` | ✓ |
| scoped packages sanitized | vision shows `@org/pkg` example | ✓ note below |

**note**: blueprint added `_sanitize_package_name()` to handle scoped packages (e.g., `@org/pkg` → `@org__pkg` in file paths). this prevents `/` in filenames which would break paths. vision showed scoped package example but did not specify file path sanitization — blueprint fills this gap correctly.

## blueprint vs criteria adherance

### install validation sequence

| criteria | blueprint sequence | adheres? |
|----------|-------------------|----------|
| validate flags first | `validate_flags()` before security check | ✓ |
| security check before install | `_check_security()` before `_install_package()` | ✓ |
| reason persisted before install | `_persist_reason()` before `_install_package()` | ✓ |

### error behavior

| criteria | blueprint | adheres? |
|----------|-----------|----------|
| audit fail → exit with vulnerabilities shown | `_check_security()` returns 1, prints details | ✓ |
| already installed → suggest upgrade | `_check_package_installed()` → error message | ✓ |
| not installed (upgrade) → suggest install | `_check_package_installed()` → error message | ✓ |

### @latest expansion

| criteria | blueprint | adheres? |
|----------|-----------|----------|
| expand before install | `_expand_version()` before `_check_security()` | ✓ |
| actual version in reason file | reason written after expand | ✓ |
