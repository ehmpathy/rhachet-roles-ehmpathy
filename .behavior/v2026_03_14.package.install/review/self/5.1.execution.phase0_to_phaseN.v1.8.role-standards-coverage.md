## absent patterns verified

### should we have unit tests?

**question:** skills have integration tests but no unit tests

**analysis:** unit tests would test individual functions like `_sanitize_package_name` or `_expand_version`. however:
- these are bash functions, not typescript
- integration tests cover their behavior via end-to-end flows
- unit tests would duplicate coverage without added value

**verdict:** ✓ not required. integration tests provide sufficient coverage for bash skills.

### should we have acceptance tests?

**question:** no .acceptance.test.ts files

**analysis:** acceptance tests are for blackbox verification via public contracts. these skills are internal to the mechanic role, not public APIs. integration tests serve the same purpose here.

**verdict:** ✓ not required for internal skills.
