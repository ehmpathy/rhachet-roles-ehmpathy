# playtest: set.package.install + set.package.upgrade

## prerequisites

- rhachet built: `npm run build`
- rhachet linked: `npx rhachet roles link --role mechanic`
- active internet connection (for npm registry)
- temp directory for sandbox tests

## sandbox

all file operations target `.temp/` to avoid repo pollution.

```sh
mkdir -p .temp/playtest-package
cd .temp/playtest-package
```

---

## happy paths

### 1. install a production dependency

```sh
# setup sandbox
mkdir -p .temp/playtest-install-prod && cd .temp/playtest-install-prod
echo '{"name": "playtest", "version": "1.0.0"}' > package.json

# run skill
npx rhachet run --skill set.package.install \
  --package test-fns \
  --at 1.3.0 \
  --for prod \
  --reason "test utilities for bdd-style tests"
```

**expected outcome:**
- exit code 0
- stdout shows turtle header `🐢 shell yeah!`
- stdout shows tree with `package: test-fns`, `version: 1.3.0`, `for: prod`
- stdout shows `audit: passed`
- stdout shows reason file path `.route/v2026_03_16.package.install/3.reason.for_test-fns.v1.i1.md`
- `package.json` has `"test-fns": "1.3.0"` in `dependencies`
- reason file exists with content

**expected outcome:**

### 5. upgrade an extant package

```sh
# setup sandbox (with pre-installed package)
mkdir -p .temp/playtest-upgrade && cd .temp/playtest-upgrade
echo '{"name": "playtest", "version": "1.0.0", "dependencies": {"zod": "3.22.0"}}' > package.json
npm install --package-lock-only

# run skill
npx rhachet run --skill set.package.upgrade \
  --package zod \
  --to 3.23.0
```

**expected outcome:**
- exit code 0
- stdout shows `🐢 righteous!`
- stdout shows `from: 3.22.0`, `to: 3.23.0`
- `package.json` has `"zod": "3.23.0"` in `dependencies`

## edgey paths

### 7. install blocked by security (vulnerable package)

```sh
# setup sandbox
mkdir -p .temp/playtest-security && cd .temp/playtest-security
echo '{"name": "playtest", "version": "1.0.0"}' > package.json

# try to install a package with known vulnerabilities
npx rhachet run --skill set.package.install \
  --package lodash \
  --at 4.17.4 \
  --for prod \
  --reason "test vulnerable version"
```

**expected outcome:**
- exit code 2 (failure)
- stdout shows `🐢 bummer dude...`
- stdout shows `audit: FAILED`
- stdout shows vulnerability details
- no reason file created
- package not installed

### 8. install blocked — package already installed

```sh
# setup sandbox with extant package
mkdir -p .temp/playtest-already && cd .temp/playtest-already
echo '{"name": "playtest", "version": "1.0.0", "dependencies": {"zod": "3.22.0"}}' > package.json

# try to install again
npx rhachet run --skill set.package.install \
  --package zod \
  --at 3.22.0 \
  --for prod \
  --reason "duplicate install"
```

**expected outcome:**
- exit code 2 (failure)
- stdout mentions "already installed"
- stdout suggests "use set.package.upgrade"

**expected outcome:**

- stdout mentions "not installed"
- stdout suggests "use set.package.install"
