# distill: domain

## usecases

### usecase 1: install new package
mechanic adds a new dependency with explicit version, security audit, and documented reason.

### usecase 2: upgrade extant package
mechanic upgrades an extant dependency to a different version with security audit.

---

## domain objects

### entities

**PackageInstallReason**
- purpose: captures the justification for a package addition
- unique: `[packageName]`
- properties:
  - packageName: string
  - version: string (semver, expanded from @latest)
  - dependencyType: 'prod' | 'prep'
  - reason: string (content from --reason)
  - auditResult: 'passed' | 'failed'
  - auditedAt: isodate
  - installedAt: isodate

note: not a TypeScript class — realized as a markdown file at `.route/v{date}.package.install/3.reason.for_{package}.v1.i1.md`

### literals

**PackageVersion**
- purpose: represents a specific package version
- properties:
  - raw: string (user input, may be "@latest")
  - expanded: string (actual semver)

**AuditResult**
- purpose: represents security audit outcome
- properties:
  - passed: boolean
  - vulnerabilities: array of { severity, cve, description }

**DependencyType**
- purpose: distinguishes production vs development dependencies
- values: 'prod' | 'prep'
- maps to: '--save' | '--save-dev' (npm) or default | '-D' (pnpm)

### events

not applicable — bash skill, no event store pattern.

## domain operations

### for set.package.install

| operation | purpose | input | output |
|-----------|---------|-------|--------|
| validateFlags | ensure required flags present | flags object | validation result |
| expandVersion | convert @latest to actual semver | package, raw version | expanded version |
| checkSecurity | audit package@version via bulk advisory | package, version | AuditResult |
| persistReason | write reason file to .route/ | package, version, reason, audit | file path |
| installPackage | run pnpm/npm install | package, version, dependencyType | success/failure |
| detectPackageManager | check for package-lock.json | repo root | 'npm' \| 'pnpm' |
| checkPackageInstalled | verify if package extant | package, repo root | boolean |

### for set.package.upgrade

| operation | purpose | input | output |
|-----------|---------|-------|--------|
| validateFlags | ensure required flags present | flags object | validation result |
| expandVersion | convert @latest to actual semver | package, raw version | expanded version |
| checkSecurity | audit package@version via bulk advisory | package, version | AuditResult |
| checkPackageInstalled | verify package is extant | package, repo root | boolean |
| upgradePackage | run pnpm/npm install (overwrites) | package, version | success/failure |
| detectPackageManager | check for package-lock.json | repo root | 'npm' \| 'pnpm' |

## relationships

### treestruct

```
set.package
├── set.package.install
│   ├── validateFlags
│   ├── expandVersion
│   ├── checkSecurity
│   ├── checkPackageInstalled (must NOT be installed)
│   ├── persistReason
│   └── installPackage
└── set.package.upgrade
    ├── validateFlags
    ├── expandVersion
    ├── checkSecurity
    ├── checkPackageInstalled (must BE installed)
    └── upgradePackage
```

### shared operations

| operation | shared by |
|-----------|-----------|
| validateFlags | both (different flag sets) |
| expandVersion | both |
| checkSecurity | both |
| detectPackageManager | both |
| checkPackageInstalled | both (opposite expectations) |

### dependencies

```
installPackage depends on:
  ← checkSecurity (must pass)
  ← persistReason (must succeed)
  ← checkPackageInstalled (must be absent)

upgradePackage depends on:
  ← checkSecurity (must pass)
  ← checkPackageInstalled (must be present)
```

## composition for wish fulfillment

### set.package.install flow

```
t0: validateFlags(--package, --at, --for, --reason)
    └─ fail-fast if absent or invalid
t1: checkPackageInstalled(package)
    └─ if present → error "use set.package.upgrade"
t2: expandVersion(package, @latest) → actualVersion
t3: checkSecurity(package, actualVersion)
    └─ if fail → block, show vulnerabilities
t4: persistReason(package, actualVersion, reason, auditResult)
    └─ write to .route/v{date}.package.install/
t5: detectPackageManager() → npm | pnpm
t6: installPackage(package, actualVersion, dependencyType)
t7: output success tree
```

### set.package.upgrade flow

```
t0: validateFlags(--package, --to)
    └─ fail-fast if absent
t1: checkPackageInstalled(package)
    └─ if absent → error "use set.package.install"
t2: expandVersion(package, @latest) → actualVersion
t3: checkSecurity(package, actualVersion)
    └─ if fail → block, show vulnerabilities
t4: detectPackageManager() → npm | pnpm
t5: upgradePackage(package, actualVersion)
t6: output success tree
```
