## matrix.5 = brain comparison scenarios

what happens when we compare brains?

| ind: brain A sensitivity | ind: brain A specificity | ind: brain B sensitivity | ind: brain B specificity | dep: winner |

| high | high | low | high | A |
| high | high | high | low | A |

| low | low | high | high | B |
| high | high | low | low | A |
| low | low | low | low | tie (both bad) |

**decomposition signal:** 4 independent dimensions is at the limit. if we add more axes (cost, latency, etc.), this matrix explodes. consider separate matrices for:
- accuracy comparison (sensitivity + specificity)
- cost comparison ($/1M tokens)
- latency comparison (ms per call)

## summary

| matrix | dimensions | combinations | gaps |
|--------|------------|--------------|------|
| confusion matrix | 2 | 4 | none |
| severity combinations | 4 | 18 (with n/a) | none |
| execution outcomes | 3 | 5 | none |
| check count outcomes | 2 | 5 | none |
| brain comparison | 4 | 8+ | at limit — decompose if more axes added |
| test invocation | 2 | 6 | none |
| scene lifecycle | 2 | 4 | none |

**decomposition opportunities:**

1. matrix.5 (brain comparison) is at the limit with 4 dimensions. if we need to compare on more axes (cost, latency, context window), create separate focused matrices rather than one large one.

2. the overall design is well-bounded — each matrix focuses on one behavioral concern with 2-4 dimensions. no matrices exceed the explosion threshold.
