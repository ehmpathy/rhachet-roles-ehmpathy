# blocker.2: unclear grain for validation/error handling functions

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.prefer.decomposable-architecture.md

**locations**:
- src/domain.roles/mechanic/skills/claude.tools/rmsafe.integration.test.ts:140
- src/domain.roles/mechanic/skills/claude.tools/rmsafe.integration.test.ts:80

Functions like asSpawnResultOrThrow and asTimeoutMsOrDefault are labeled as 'transformer (pure computation, no i/o)' but include complex conditional logic with multiple if-throws (side effects) and error construction. Transformers must be pure computation with no side effects. This creates unclear grain (mixed validation + throwing in what should be pure leaf). Per the rule, for each operation ask: 'grain clarity: is this a transformer, communicator, or orchestrator?' and flag unclear/mixed as blocker.

**snippet**:
```ts
const asSpawnResultOrThrow = (input: {
    result: SpawnSyncReturns<string>;
    label: string;
    command: string;
    args: string[];
    cwd: string;
    hint: string;
  }): { stdout: string; stderr: string; exitCode: number } => {
    if (input.result.error) {
      throw new MalfunctionError(`${input.label} spawn failed: ${input.result.error.message}`, {
        command: input.command,
        args: input.args,
        cwd: input.cwd,
        hint: input.hint,
      });
    }
    if (input.result.signal === 'SIGTERM') {
      throw new MalfunctionError(`${input.label} timed out after ${commandTimeoutMs}ms`, {
        command: input.command,
        args: input.args,
        cwd: input.cwd,
        hint: 'command took too long; check for infinite loops or deadlocks',
      });
    }
    // ... additional ifs with throws
    return {
      stdout: input.result.stdout ?? '',
      stderr: input.result.stderr ?? '',
      exitCode: input.result.status,
    };
  };
```

Per the rule, orchestrators must read as narrative prose ('each line tells what, not how') with no decode-friction inline.
