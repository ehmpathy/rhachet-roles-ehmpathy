# vision: sedreplace safety tuning

## the outcome world

### before: the :pikachu-shockface: moment

a clone wants to rename `getUserById` to `findUserByUuid` in typescript files.

```sh
npx rhachet run --skill sedreplace --old "getUserById" --new "findUserByUuid" --mode apply
```

500 files changed. node_modules? no, that's gitignored. but... every markdown file, every json config, every test fixture, every snapshot. the clone forgot `--glob` and now the repo is a mess.

or worse:

```sh
npx rhachet run --skill sedreplace --old "id" --new "uuid" --glob "**/*" --mode apply
```

"**/*" matches everything. customer_id becomes customer_uuid. invalid becomes invaluuid. the clone has to `git checkout .` and start over, wasting time and tokens.

### after: the pit of success

same clone, same intent:

```sh
npx rhachet run --skill sedreplace --old "getUserById" --new "findUserByUuid" --mode apply
```

**blocked.** the tool responds:

```
🐢 bummer dude...

🐚 sedreplace
   └─ ✋ blocked: cannot apply without plan

to prevent unexpected consequences, sedreplace requires you to preview changes first.

run with --mode plan (or omit --mode) first, then re-run with --mode apply.
```

the clone plans first:

```sh
npx rhachet run --skill sedreplace --old "getUserById" --new "findUserByUuid"
```

sees output showing 47 files across the entire repo. realizes they wanted just `.ts` files. tries again:

```sh
npx rhachet run --skill sedreplace --old "getUserById" --new "findUserByUuid" --glob "**/*.ts"
```

**blocked.** the tool responds:

```
🐢 bummer dude...

🐚 sedreplace
   └─ ✋ blocked: unbound glob

glob patterns that start with **/ are too broad and can cause unexpected changes.

use a bounded glob instead:
  --glob 'src/**/*.ts'     (files in src/)
  --glob 'src/**/*.test.ts' (test files in src/)
  --glob '*.ts'            (files in root only)
```

the clone bounds their glob:

```sh
npx rhachet run --skill sedreplace --old "getUserById" --new "findUserByUuid" --glob 'src/**/*.ts'
```

sees exactly 12 files in src/. perfect. applies:

```sh
npx rhachet run --skill sedreplace --old "getUserById" --new "findUserByUuid" --glob 'src/**/*.ts' --mode apply
```

works. the plan was tracked internally, so apply is allowed.

### the "aha" moment

the value clicks when:
- a clone tries `--mode apply` first and gets blocked with a helpful message
- a clone uses `**/*.ts` and learns to bound it to `src/**/*.ts`
- a clone realizes they can work confidently because the tool prevents footguns

## user experience

### usecases

| goal | glob | result |
|------|------|--------|
| rename function in src | `'src/**/*.ts'` | bounded, safe |
| rename across test files | `'src/**/*.test.ts'` | bounded, safe |
| update import path | `'src/**/*.{ts,tsx}'` | bounded, safe |
| rename everywhere | (no glob) | must plan first, see full scope |

### contract inputs & outputs

**inputs:**
- `--old "pattern"` — text to find (required)
- `--new "replacement"` — text to replace with (required)
- `--glob "pattern"` — file filter (optional, but unbound globs blocked)
- `--mode plan|apply` — preview or execute (plan by default)

**outputs (plan mode):**
```

   ├─ matches: 34

      │  │  - const user = await getUserById(id);
      │  │  + const user = await findUserByUuid(id);
      │  └─
      └─ ... (10 more files)

run with --mode apply to execute
```

**outputs (apply mode, after plan):**

      │  └─
      └─ ... (10 more files)

### timeline

1. **clone wants bulk rename** — invokes sedreplace
2. **attempts direct apply** — blocked, told to plan first
3. **plans with broad glob** — blocked, told to bound the glob
4. **plans with bounded glob** — sees exactly what will change
5. **reviews diff output** — confirms intent
6. **applies** — allowed because matching plan was recorded
7. **done** — clean, intentional, reversible changes

## mental model

### how users describe this to a friend

> "sedreplace is like find-and-replace but it makes you preview first. it also won't let you do `**/*` because that's asking for trouble. you have to say which folder you actually mean."

### analogies

| analogy | fit |
|---------|-----|
| terraform plan/apply | exact fit — preview before mutate |
| git add -p | similar — review before commit |
| rm -i | similar — confirm before delete |
| seatbelt | apt — prevents accidents, doesn't slow you down |

### terms

| user says | we say |
|-----------|--------|
| "preview" | plan mode |
| "do it" | apply mode |
| "everywhere" | unbound glob (blocked) |
| "in src" | bounded glob (allowed) |

## evaluation

### how well does it solve the goals?

| goal | solution | effectiveness |
|------|----------|---------------|
| prevent unplanned applies | hardnudge pattern, track plans | high — can't skip preview |
| prevent unbound globs | block `**/` prefix | high — forces bounded scope |
| exclude node_modules | already done (gitignore) | done — no changes needed |
| fix shell glob expansion | update all examples to use `'...'` | **critical** — docs + permissions.jsonc |

### pros

- **pit of success** — hard to make mistakes, easy to do the right thing
- **fast feedback** — blocked immediately with helpful message
- **reversible** — `git checkout .` always available
- **consistent** — same pattern as terraform, familiar to devs

### cons

- **extra step** — must plan before apply (but this is the point)
- **learning curve** — clones must learn bounded globs (but message teaches them)
- **state tracking** — must track plans in `.claude/*.local.json` (complexity)

### edgecases

| edgecase | how contract handles |
|----------|---------------------|
| clone runs apply, never planned | blocked — must plan first |
| clone plans, waits too long, applies | allow — staleness acceptable |
| clone plans glob A, applies glob B | blocked — plan must match |
| clone uses `**/*.ts` | blocked — must be `src/**/*.ts` or similar |
| clone uses no glob at all | allowed for plan (shows full scope), blocked for apply without prior plan |
| clone uses `'**/*.ts'` (quoted) | works — shell doesn't expand |

### awkward discoveries

1. **plan tracking state** — we need `.claude/sedreplace.nudges.local.json` to track plans and allow matching applies. this adds filesystem state but follows the hardnudge pattern already used elsewhere.

2. **bounded glob definition** — "starts with `**/`" is the heuristic. `src/**/*.ts` doesn't start with `**/` so it passes. `**/*.ts` starts with `**/` so it blocks. simple rule, easy to explain.

3. **shell expansion (critical)** — shells expand `*` before the command runs, so unquoted `--glob **/*.ts` breaks. must use single quotes: `--glob '**/*.ts'`. **all examples must be updated:**
   - `sedreplace.sh` header comments
   - `rule.prefer.sedreplace-for-renames.md` brief
   - `init.claude.permissions.jsonc` allowlist examples
   - any other docs that mention sedreplace globs

4. **apply without glob** — if clone plans with no glob (sees full scope), then applies with no glob, should that be allowed? probably yes — they previewed, they know. the danger is `--mode apply` without any prior plan.

---

## summary

the vision is a sedreplace that guides clones into the pit of success:

1. **must plan before apply** — no surprises
2. **must bound globs** — no "everywhere" accidents
3. **helpful blocks** — teaches correct usage inline
4. **minimal friction** — one extra step that prevents hours of cleanup
5. **single-quoted examples everywhere** — critical update to `permissions.jsonc`, briefs, and skill header to prevent shell expansion footguns
