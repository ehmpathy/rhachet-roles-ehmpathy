## gap analysis

### unit tests (sedreplace.test.ts)

**blueprint**: create unit tests for `isUnboundGlob` and `computePlanKey`

**gap**: not created

**justification**: sedreplace is a bash executable. the functions `isUnboundGlob` (glob validation logic) and `computePlanKey` (sha256 computation) are internal bash functions that cannot be exported to TypeScript for unit testing.

**resolution**: the functionality is comprehensively tested via integration tests:
- glob validation: case2/t0.1 tests unbound glob blocking
- plan key: case13/t0-t3 tests plan tracking with key matching

**recommendation**: gap acceptable. integration tests provide sufficient coverage.

### stale plan expiry

**blueprint**: case7 tests stale plan cleanup

**implementation**: stale cleanup implemented (lines 143-152) but no dedicated test

**justification**: stale cleanup runs automatically on every invocation. testing would require time manipulation (waiting 1+ hours or mocking time).

**resolution**: code review confirms correct implementation:
```bash
local one_hour_ago=$((now - 3600))
jq --argjson cutoff "$one_hour_ago" 'with_entries(select(.value.time > $cutoff))'
```

**recommendation**: gap acceptable. could add test with injected stale timestamp if needed in future.
