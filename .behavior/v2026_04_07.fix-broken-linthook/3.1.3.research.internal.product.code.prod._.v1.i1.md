```bash
# print turtle header
# usage: print_turtle_header "sweet" | "crickets..." | "heres the wave..."
print_turtle_header() {

  echo ""
}

# print shell header with skill name
# usage: print_tree_start "cpsafe"
print_tree_start() {
  local skill="$1"
  echo "🐚 $skill"
}

# print tree branch (not last item)
# usage: print_tree_branch "from" "src/*.md"
print_tree_branch() {
  local key="$1"
  local value="$2"
  echo "   ├─ $key: $value"
}

# print tree leaf (last item or section header)
# usage: print_tree_leaf "copied"
print_tree_leaf() {
  local content="$1"
  echo "   └─ $content"
}
```

## pattern.3 = findsert idiom for file creation [REUSE]

teesafe implements findsert semantics for idempotent file creation.

### citation 4: teesafe.sh (lines 139-156)

```bash
  findsert)
    # if file exists with same content, no-op; different content = error
    if [[ -f "$INTO_ABS" ]]; then
      CONTENT_EXTANT=$(cat "$INTO_ABS")
      if [[ "$STDIN_CONTENT" == "$CONTENT_EXTANT" ]]; then
        # same content, echo to stdout and exit (no-op for file)
        printf '%s' "$STDIN_CONTENT"
        exit 0
      else
        # different content, error
        echo "error: file exists with different content (findsert mode)" >&2
        echo "  dest: $INTO_ABS" >&2
        exit 2
      fi
    fi
    # file does not exist, write it
    printf '%s' "$STDIN_CONTENT" | tee "$INTO_ABS"
```

**relation to wish**: git.repo.test will findsert .gitignore in log directory.

## pattern.4 = repo root validation [REUSE]

skills validate paths are within repo before operations.

### citation 5: teesafe.sh (lines 102-126)

```bash
# ensure we're in a git repo
if ! git rev-parse --git-dir >/dev/null 2>&1; then
  echo "error: not in a git repository" >&2
  exit 2
fi

# get repo root (expand symlinks fully)
REPO_ROOT=$(realpath "$(git rev-parse --show-toplevel)")

# validate dest is within repo
if [[ "$INTO_ABS" != "$REPO_ROOT" && "$INTO_ABS" != "$REPO_ROOT/"* ]]; then
  echo "error: destination must be within the git repository" >&2
  echo "  repo root: $REPO_ROOT" >&2
  echo "  dest:      $INTO_ABS" >&2
  exit 2
fi
```

**relation to wish**: git.repo.test will validate it's in a git repo and log directory is within repo.
