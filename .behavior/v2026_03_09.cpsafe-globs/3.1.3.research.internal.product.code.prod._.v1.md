# research: internal product code (prod)

## summary

analyzed production code for cpsafe.sh, mvsafe.sh, rmsafe.sh, and sedreplace.sh to identify patterns to reuse, extend, or replace.

## cpsafe.sh analysis

### [REPLACE] single-file arg parse (root cause)

**location**: lines 76-81

```bash
if [[ ${#POSITIONAL_ARGS[@]} -ge 1 ]]; then
  FROM="${POSITIONAL_ARGS[0]}"
fi
if [[ ${#POSITIONAL_ARGS[@]} -ge 2 ]]; then
  INTO="${POSITIONAL_ARGS[1]}"
fi
```

**problem**: only first positional arg used. when shell expands glob before skill receives it, only first file processed.

**fix**: treat FROM as glob pattern, expand internally via git ls-files.

### [REUSE] repo boundary validation

**location**: lines 84-99

```bash
REPO_ROOT=$(git rev-parse --show-toplevel 2>/dev/null)
if [[ -z "$REPO_ROOT" ]]; then
  echo "error: must be run from within a git repository"
  exit 1
fi

SOURCE_REAL=$(realpath "$FROM")
if [[ ! "$SOURCE_REAL" == "$REPO_ROOT"* ]]; then
  echo "error: source must be within the git repository"
  exit 1
fi
```

**verdict**: keep this pattern. apply per-file in loop.

### [REPLACE] output format

**location**: lines 114-115

```bash
echo "copied: $FROM -> $INTO"
```

**problem**: plain echo, no tree structure, no turtle header.

**fix**: use sedreplace-style tree output with incremental file lines.

## patterns to implement

### 1. glob expansion (from sedreplace)

```bash
shopt -s globstar nullglob
FILES=()
for f in $FROM_PATTERN; do
  [[ -f "$f" ]] && FILES+=("$f")
done
```

or via git ls-files:

```bash
mapfile -t FILES < <(git ls-files --cached --others --exclude-standard ":(glob)$FROM_PATTERN")
```

### 2. tree output structure

```bash
print_header() {
  local count=$1
  if [[ $count -eq 0 ]]; then
    echo "🐢 crickets..."
  else
    echo "🐢 sweet"
  fi
  echo ""
}

print_tree_start() {
  local skill=$1
  echo "🐚 $skill"
}

print_tree_branch() {
  local key=$1 value=$2
  echo "   ├─ $key: $value"
}

print_tree_leaf() {
  local key=$1
  echo "   └─ $key"
}
```

### 3. incremental file output

```bash
# print tree header first
print_tree_start "cpsafe"
print_tree_branch "from" "$FROM_PATTERN"
print_tree_branch "into" "$INTO"
print_tree_branch "files" "${#FILES[@]}"
echo "   └─ copied"

# then print each file as operation completes
for file in "${FILES[@]}"; do
  dest="$INTO/$(basename "$file")"
  cp "$file" "$dest"
  echo "      ├─ $file -> $dest"
done
```

## validation approach

per-file validation before operation:
1. expand glob via git ls-files or bash globstar
2. for each file: validate within repo boundary
3. for cpsafe/mvsafe: validate destination is directory if >1 file
4. perform operation, print line immediately
5. if zero matches: show crickets header + (none)
