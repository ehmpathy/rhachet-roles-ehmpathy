# handoff: route.drive should not traverse symlinks outside repo

## .what

`route.drive` (from rhachet-roles-bhrain) fails with EACCES when it encounters symlinks pointing to protected directories outside the repo.

## .observed

```
error: EACCES: permission denied, scandir '/home/vlad/git/ehmpathy/_worktrees/rhachet-roles-ehmpathy.vlad.cpsafe-globs/.temp/genTempDir.symlink/2026-03-09T20-13-18.384Z.cpsafe-test.e2a536ba/sneaky-dir/systemd-private-873621281c694c87b9ccd98e8c9e235a-ModemManager.service-AP6QEE'
```

## .root cause

1. integration tests create temp directories in `.temp/` with symlinks pointing to `/tmp` (for security boundary tests)
2. `/tmp` contains systemd private directories with restricted permissions
3. `route.drive` scans the filesystem and follows symlinks
4. when it tries to scan the protected systemd directory → EACCES

## .fix needed (in rhachet-roles-bhrain)

`route.drive` should:
1. respect gitignore patterns (don't scan `.temp/`, `node_modules/`, etc.)
2. not follow symlinks that point outside the repo root
3. handle EACCES errors gracefully (skip inaccessible directories with warning)

## .workaround

clean up `.temp/` after integration tests:
```sh
npx rhachet run --skill rmsafe --path '.temp' --recursive
```

## .location

- skill: `repo=bhrain/role=driver/skill=route.drive`
- package: `rhachet-roles-bhrain@0.17.0`
- entry: `node -e "import('rhachet-roles-bhrain/cli/route').then(m => m.routeDrive())"`
