## YAGNI analysis

### output.sh - should this be deleted?

**the honest question**: was this extracted too early? is it YAGNI?

**what the vision requested**: "tree output format" + "output structure matches sedreplace exactly"
**what was built**: shared output.sh with 5 helper functions

**the case FOR deletion**:
- wasn't explicitly requested
- adds a new file to maintain
- adds indirection (source statement in each script)
- could inline 19 lines × 3 = 57 lines total

**the case AGAINST deletion**:
- 3 usages = rule of three threshold met
- vision said output "matches sedreplace exactly" = consistency requirement
- if inline, any tree format change requires editing 3 files
- 57 lines duplicated > 19 lines shared

**decision**: keep output.sh
- meets rule of three threshold
- consistency requirement implies shared formatting
- maintenance burden is lower with shared helper

**what this means**: extraction was justified but borderline. if only 2 files used it, would have been YAGNI.

### is_glob_pattern() duplication - should this also be extracted?

**the function** (3 lines, duplicated 3 times = 9 lines total):
```bash
is_glob_pattern() {
  local pattern="$1"
  [[ "$pattern" == *"*"* || "$pattern" == *"?"* || "$pattern" == *"["* ]]
}
```

**option A**: extract to output.sh
- pros: single source of truth, easier to add `{` later
- cons: output.sh becomes "utils.sh" (scope creep)

**option B**: keep duplicated
- pros: each file self-contained, can evolve independently
- cons: bug fix requires 3 edits

**decision**: keep duplicated (option B)
- only 3 lines each
- function is unlikely to change (glob syntax is stable)
- extraction would blur output.sh's purpose
- rule of three met but cost of duplication is low

### directory backward compat (IS_DIR_MOVE/IS_DIR_REMOVE)

**question**: was this "while we're here" scope creep?

**answer**: no - literal directory paths were already supported. removing would break extant callers.

**verdict**: backward compat, not scope creep

## features actively NOT added

these were considered and rejected:

| feature | why considered | why rejected |
|---------|----------------|--------------|
| `--mode plan` for fileops | sedreplace has it | wish didn't request preview mode |
| `--exclude` patterns | useful for `*.md` except `readme.md` | adds complexity, not requested |
| recursive dir copy in cpsafe | `cp -r` equivalent | cpsafe handles files; use mvsafe for dirs |
| color output | makes tree prettier | terminal compat issues, not requested |
| verbose/quiet flags | control output level | tree output is already minimal |
| limit output to first N files | large batches could spam | sedreplace shows all, so we do too |
| progress bar | visual feedback | tree itself IS the progress |

**key principle**: match sedreplace behavior exactly unless wish says otherwise
