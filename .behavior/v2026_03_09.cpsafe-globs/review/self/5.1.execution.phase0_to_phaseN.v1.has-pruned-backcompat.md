### 4. symlink handle in rmsafe

**what:** rmsafe removes symlinks (checks `-L` in addition to `-f`)

**explicitly requested?** no — wish didn't mention symlinks

**code location:**
```
$ grep -n "\-L" src/domain.roles/mechanic/skills/claude.tools/rmsafe.sh
109:  eval "for f in $TARGET; do [[ -f \"\$f\" || -L \"\$f\" ]] && FILES+=(\"\$f\"); done"
119:  elif [[ -f "$TARGET" || -L "$TARGET" ]]; then
133:  if [[ -L "$TARGET" ]]; then
201:  if [[ -L "$FILE" ]]; then
```

lines 109, 119, 133, 201 all check `-L` (symlink) in addition to `-f` (file)

**why needed:** symlinks are removable filesystem entries. without `-L` check, globs would skip symlinks silently — worse UX than current behavior

**decision:** keep — removal would silently skip symlinks (worse behavior)
