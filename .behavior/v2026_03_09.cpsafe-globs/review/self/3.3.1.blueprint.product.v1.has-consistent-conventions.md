### 6. skill invocation pattern — consistent

**search performed:** `grep -r "--from" src/` and `grep -r "--into" src/`

```bash

```

**our usage (cpsafe/mvsafe):**
```bash

```

**semantic alignment:**
| flag | brief.condense meaning | cpsafe/mvsafe meaning | consistent? |
|------|----------------------|----------------------|-------------|
| --from | input path/glob | source path/glob | ✓ yes |
| --into | output path | destination path | ✓ yes |

**other skills with different semantics:**
```bash

--old 'pattern'  # what to find
--new 'text'     # what to replace with

# symlink.sh — link creation
--at 'path'      # where to create link
--to 'target'    # what link points to
```

**analysis:**
- `--from`/`--into` already used by brief.condense for path operations
- we match that semantic exactly
- sedreplace uses `--old`/`--new` for content (different domain)
- symlink uses `--at`/`--to` for link semantics (different domain)

**verdict:** consistent with extant `--from`/`--into` convention (brief.condense).
