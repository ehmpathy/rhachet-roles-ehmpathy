## hidden assumptions surfaced

### assumption 1: glob expansion should happen inside the skill

**what do we assume without evidence?**
- the skill receives a literal string `'*.md'` and expands it internally
- bash globstar handles `**` patterns correctly

**what evidence supports this?**
- single quotes prevent shell expansion (shell 101)
- we control glob expansion via `shopt -s globstar` + array expansion

**what if the opposite were true?**
- if shell expands before skill: skill receives multiple positional args
- this is actually how it works TODAY with unquoted globs!
- current code: `FROM="${POSITIONAL_ARGS[0]}"` — only takes first arg

**what exceptions exist?**
- double quotes: `"*.md"` in some shells expands inside the skill, in others not
- recommendation: single-quote all examples for safety

**verdict**: ✅ assumption valid — single quotes + internal expansion is correct approach

### assumption 2: destination must be a directory for multi-file operations

**what do we assume without evidence?**
- when glob matches >1 file, `--into` must be a directory
- `cp *.md merged.md` is invalid

**what evidence supports this?**
- standard unix behavior: `cp file1 file2 dir/` works, `cp file1 file2 file3` fails
- no reasonable interpretation of "copy 5 files into 1 file"

**what if the opposite were true?**
- could concatenate files? that's `cat`, not `cp`
- could require explicit flag like `--merge`? overcomplicates

**verdict**: ✅ assumption valid — destination must be directory for batch ops

### assumption 8: bash nullglob handles empty matches correctly

**what do we assume without evidence?**
- when glob matches 0 files, we get an empty array
- we can detect "no matches" and error appropriately

**what evidence supports this?**
- requires `shopt -s nullglob` to return empty array
- WITHOUT nullglob, bash keeps the literal pattern: `*.xyz` → `*.xyz`
- this would then fail `-e` check (good) but with confusable error message

**what if we forget nullglob?**
- `*.xyz` stays as literal string
- `-e "*.xyz"` fails
- error: "source does not exist: *.xyz"
- user might not realize it was a "no matches" case vs "bad pattern"

**verdict**: ✅ solvable — need explicit nullglob + better error message for "no matches"
