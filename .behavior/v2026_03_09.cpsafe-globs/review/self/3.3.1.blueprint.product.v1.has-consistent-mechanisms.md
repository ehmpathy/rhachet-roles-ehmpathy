### 2. is_glob_pattern() — no extant equivalent

**function duplicated 3 times — identical implementation:**

```bash
# cpsafe.sh:119-122
# mvsafe.sh:119-122
# rmsafe.sh:97-100
is_glob_pattern() {
  local pattern="$1"
  [[ "$pattern" == *"*"* || "$pattern" == *"?"* || "$pattern" == *"["* ]]
}
```

**search for extant mechanisms:**

| search | result |
|--------|--------|
| `grep -r "is_glob" src/` | only our 3 new files |
| `grep -r "glob.*pattern" src/` | sedreplace.sh (different mechanism) |
| `grep -r "\*\"\*\*" src/` | no matches for similar char checks |

**sedreplace.sh approach (lines 195-200) — different usecase:**
```bash
# sedreplace uses --glob as a FILTER parameter, not pattern detection
if [[ -n "$GLOB" ]]; then
  FIND_CMD+=("--glob" "$GLOB")
fi
```

key difference: sedreplace always receives a glob filter via `--glob` flag. our skills receive a `--from`/`--path` that could be either literal or glob.

**why not extract to output.sh?**

considered adding `is_glob_pattern()` to output.sh:
- pro: single source of truth
- con: output.sh becomes "utils.sh" (scope creep)
- con: function is 3 lines — duplication cost is 6 lines total
- con: glob syntax is stable — unlikely to need changes

**verdict:** no extant mechanism to reuse. acceptable duplication per rule.prefer.wet-over-dry (3 lines x 3 files = 9 lines total, function stable).

### 3. glob expansion loop — no extant equivalent

**pattern duplicated 3 times — with slight variance:**

```bash
# cpsafe.sh:32 + line 130
shopt -s globstar nullglob
eval "for f in $FROM; do [[ -f \"\$f\" ]] && FILES+=(\"\$f\"); done"

# mvsafe.sh:32 + line 131 — IDENTICAL to cpsafe
shopt -s globstar nullglob
eval "for f in $FROM; do [[ -f \"\$f\" ]] && FILES+=(\"\$f\"); done"

# rmsafe.sh:33 + line 109 — DIFFERENT (adds -L for symlinks)
shopt -s globstar nullglob
eval "for f in $TARGET; do [[ -f \"\$f\" || -L \"\$f\" ]] && FILES+=(\"\$f\"); done"
```

**search for extant file iteration patterns:**

| skill | approach | mechanism |
|-------|----------|-----------|
| sedreplace.sh | `rg --files-with-matches` | ripgrep finds files |
| symlink.sh | single path only | no iteration needed |
| git.commit/*.sh | git operations | no file listing |
| brief.condense.sh | single path | no iteration needed |
| **cpsafe/mvsafe/rmsafe** | `eval for glob` | bash native expansion |

**why not use ripgrep like sedreplace?**

sedreplace.sh (lines 195-210):
```bash
FIND_CMD=(rg --files-with-matches)
if [[ -n "$GLOB" ]]; then
  FIND_CMD+=("--glob" "$GLOB")
fi
```

key difference: sedreplace searches for content matches, then applies replacement. fileops need just file listing from glob — ripgrep is overhead.

**why not use `find`?**

| approach | pros | cons |
|----------|------|------|
| `find -name "*.md"` | cross-platform | different syntax, requires -exec |
| `bash glob` | native, simple | requires shopt for `**` |

bash glob chosen for simplicity and precedent (shell scripting norm).

**variance justified (rmsafe):**

rmsafe adds `-L` check (line 109) because:
- symlinks should be removable
- rmsafe explicitly handles symlink locations, not targets
- cpsafe/mvsafe only care about regular files

**verdict:** new mechanism, no extant equivalent. variance between files is intentional and minimal.

## potential centralizations considered

### could we centralize output.sh?

**option:** create shared `lib/output.sh` used by all skill groups

**analysis:**
- would couple unrelated domains (git.commit ↔ fileops ↔ brief.condense)
- domain-specific functions would still need local additions
- extant pattern is per-domain duplication
- rule.prefer.wet-over-dry: tolerate duplication when abstraction would couple unrelated code

**verdict:** keep per-domain pattern. centralization would fight the extant architecture.

### could we centralize is_glob_pattern()?

**option:** extract to shared utility

**analysis:**
- only 3 lines per file
- only 3 usages (cpsafe, mvsafe, rmsafe)
- all in same directory (claude.tools/)
- function unlikely to change (glob syntax is stable)

**verdict:** acceptable duplication per rule.prefer.wet-over-dry. if glob detection logic needed changes, could extract then.
