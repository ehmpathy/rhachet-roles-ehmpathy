### 2. is_glob_pattern() — does extant equivalent exist?

**search performed:**
```
$ grep -rn "is_glob" src/
(only matches in cpsafe.sh, mvsafe.sh, rmsafe.sh — all new)

$ grep -rn "glob.*pattern" src/
src/domain.roles/mechanic/skills/claude.tools/sedreplace.sh:194: (uses --glob as filter, different purpose)
```

**sedreplace.sh approach:**
```bash
# sedreplace uses --glob as a FILTER parameter
if [[ -n "$GLOB" ]]; then
  FIND_CMD+=(\"--glob\" \"$GLOB\")
fi
```

key difference: sedreplace always receives glob via `--glob` flag. our skills receive `--from`/`--path` that could be literal or glob.

**why new function needed:**
- no extant mechanism to detect if a string is a glob pattern
- sedreplace's glob usage is semantic (filter), not detection

**verdict:** ✓ new mechanism justified, no extant equivalent
