# self-review: has-pruned-backcompat

## backward compat concerns identified

### 1. directory move support (IS_DIR_MOVE in mvsafe.sh)

**what it does**: if `--from` is a literal directory path, move it directly

**was this requested?**: NO - wish only mentioned glob patterns

**evidence it's needed**:
```bash
# this worked before
mvsafe.sh --from 'src/olddir' --into 'src/newdir'
```

**if we removed it**:
- callers who move directories would get "no files matched" (crickets)
- or error: "use glob pattern" - unclear since they have a literal path

**decision**: OPEN QUESTION FOR WISHER
- option A: keep directory support (current behavior)
- option B: require glob for directory moves: `mvsafe --from 'src/olddir/**/*' --into 'dest/'`
- option C: error with helpful message: "use mvsafe -r for directories"

### 2. directory remove support (IS_DIR_REMOVE in rmsafe.sh)

**what it does**: if `--path` is a literal directory, remove it with -r flag

**was this requested?**: NO - wish only mentioned glob patterns

**evidence it's needed**:
```bash
# this worked before
rmsafe.sh -r 'build/cache'
```

**if we removed it**:
- callers who remove directories would fail
- would need to use: `rmsafe --path 'build/cache/**/*' --recursive` (weird)

**decision**: keep - `-r` flag is explicit about intent

### 3. literal file path support (non-glob paths)

**what it does**: if `--from` or `--path` is a literal path (no *, ?, [), treat it as single file

**was this requested?**: IMPLICITLY YES - wish showed before/after with literal paths that worked

**if we removed it**:
- users would need to always use globs: `cpsafe --from 'src/file.md'` (works)
- but `cpsafe --from 'src/file.md'` also works as literal - consistent

**decision**: keep - this is core functionality, not backward compat

## summary

| concern | was requested? | decision |
|---------|----------------|----------|
| directory move (mvsafe) | NO | **OPEN QUESTION** |
| directory remove (rmsafe -r) | NO | keep - explicit flag |
| literal file paths | implicitly yes | keep - core feature |

## open question for wisher

should mvsafe continue to support move of directories via literal paths?

```bash
# current behavior (works)
mvsafe --from 'src/olddir' --into 'src/newdir'

# alternative: require glob
mvsafe --from 'src/olddir/**/*' --into 'src/newdir/'
```

**wisher decision:** yes, directory moves supported via `--recursive` flag. keep IS_DIR_MOVE branch.
