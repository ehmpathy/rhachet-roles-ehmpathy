### readable.narrative/rule.forbid.else-branches

**brief:** avoid else branches, use early returns

**grep search:** `grep -n "else" src/domain.roles/mechanic/skills/claude.tools/cpsafe.sh`

```
84:else
131:else
200:  else
206:    else
```

**analysis of each:**
- line 84: argument parsing fallback (bash case statement pattern)
- line 131: glob vs literal branch (structural decision, not flow hazard)
- line 200: multi-file vs single-file dest path (assignment logic)
- line 206: directory extant vs not extant (realpath mode selection)

**context:** in bash, these are assignment branches not control flow hazards. the rule targets TypeScript's hidden state mutation via else. bash variables are explicitly assigned in both branches.

**new file output.sh:57:**
```bash
if [[ "$is_last" == "true" ]]; then
  echo "      └─ $content"
else
  echo "      ├─ $content"
fi
```

this is a simple ternary-equivalent for tree character selection. no implicit hazard.

**verdict:** ⚠️ minor deviation — else branches used for conditional assignment. acceptable in bash context where early returns don't apply mid-function. no implicit hazards.
