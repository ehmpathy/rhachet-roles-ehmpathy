# self-review: has-edgecase-coverage

## what could go wrong?

| scenario | what breaks | playtest case | how verified |
|----------|-------------|---------------|--------------|
| glob matches zero files | user sees no output, confused | **case.5** | `*.xyz` pattern, expects "crickets..." |
| shell expands glob before skill | only first file passed | **all cases** | single quotes `'*.md'` throughout |
| recursive glob `**` fails | nested files missed | **case.4** | `src/**/*.ts` with nested dirs |
| sedreplace no matches | user unsure if worked | **case.8** | `NONEXISTENT` pattern, expects "crickets..." |
| mixed file types | wrong files affected | **case.3** | `.tmp` vs `.txt`, only `.tmp` removed |

## unusual but valid inputs

| input | why unusual | playtest case | verification |
|-------|-------------|---------------|--------------|
| recursive glob `**/*.ts` | double-star is less common | **case.4** | creates nested `src/utils/`, `src/core/` |
| pattern matches zero | user might expect error | **case.5** | verifies exit code 0 (not error) |
| selective pattern `.tmp` | must not affect `.txt` | **case.3** | `keep.txt` must survive |

## boundaries tested

| boundary | count | playtest case | pass criteria |
|----------|-------|---------------|---------------|
| zero files matched | 0 | **case.5, case.8** | output shows "crickets...", files: 0 |
| few files matched | 2-3 | **case.1-4** | all files listed in output |
| nested directories | 2 levels | **case.4** | `src/utils/foo.ts` and `src/core/bar.ts` |

## boundaries not tested (justification)

| boundary | why excluded | mitigation |
|----------|--------------|------------|
