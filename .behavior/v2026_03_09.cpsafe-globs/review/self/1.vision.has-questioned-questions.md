## open questions triaged

### question 1: fail-fast vs continue?

**can this be answered via logic or precedent?**

yes — looking at extant patterns:
- `sedreplace` uses `set -euo pipefail` (fail-fast)
- `git add` fails fast on invalid paths
- `cp` fails fast on files not found

**why fail-fast:**
- safer: no silent partial success
- if user wants "process what you can", they can re-run without the bad file
- consistent with other skills in the repo

**verdict**: ✅ self-answered — fail-fast (current behavior)

### question 2: recursive globs (`**/*.md`)?

**can this be answered via extant behavior?**

yes — sedreplace already supports recursive globs:
- permissions.jsonc line 203: `--glob 'src/**/*.ts'`
- sedreplace.sh uses `git ls-files --cached --others ":(glob)$GLOB_FILTER"`
- this correctly handles `**` patterns

**why support:**
- consistency with sedreplace
- users expect `**` to mean recursive
- bash globstar (`shopt -s globstar`) handles this natively

**verdict**: ✅ self-answered — support recursive globs via same mechanism as sedreplace
