# research: factory opportunities

## summary

primary opportunity: extract shared output helper for tree-format output across all fileops skills.

---

## the constraint

**bottleneck**: output format logic will be duplicated across cpsafe.sh, mvsafe.sh, rmsafe.sh if not extracted.

**fix**: create shared output.sh helper (like git.commit/output.sh) that all fileops skills can source.

---

## flow and queues

**queues**: none significant
- tests run locally, no CI queue wait
- build step is fast (~2s)
- link step is instant

**wip limit**: can work on all four skills (cpsafe, mvsafe, rmsafe, sedreplace) in parallel since they're independent

---

## waste

**repeated setup**: tree output format code would be written 3 times without extraction

**solution**: extract to shared helper:
```bash
# output.sh (shared)
print_turtle_header() { ... }
print_tree_start() { ... }
print_tree_branch() { ... }
print_tree_leaf() { ... }
```

**handoffs**: none - work is fully owned by mechanic

---

## small batches

**smallest verifiable unit**: single skill (e.g., just cpsafe.sh)

**incremental verification**:
1. ✅ can verify cpsafe glob support alone
2. ✅ can verify mvsafe glob support alone
3. ✅ can verify rmsafe glob support alone
4. ✅ can verify sedreplace phrase updates alone

**feedback**: tests provide immediate feedback per skill

---

## autonomy

**full autonomy**: yes
- no external team dependencies
- no approvals needed
- no infra provision required
- can ship independently

---

## density

**verification density**: ~90% execution, ~10% setup

- test setup: create temp dir, write files (~0.1s)
- test execution: run skill, check output (~0.5s)
- no network wait, no database setup

**time waste**: minimal. tests are fast and focused.

---

## systems view

**scope of improvement**: this behavior + all future fileops skills

extract output.sh helper benefits:
- this behavior: reduces code duplication
- future skills: provides reusable tree output pattern
- maintenance: single place to update output format

**not a local optimum**: the improvement compounds

---

## recommendation

**do**: extract output.sh to `claude.tools/output.sh` before implement glob support

**pattern**:
```
src/domain.roles/mechanic/skills/claude.tools/
  ├─ output.sh        # NEW: shared output helper
  ├─ cpsafe.sh        # sources output.sh
  ├─ mvsafe.sh        # sources output.sh
  ├─ rmsafe.sh        # sources output.sh
  └─ sedreplace.sh    # sources output.sh
```

**benefit**: write output logic once, use across all fileops
