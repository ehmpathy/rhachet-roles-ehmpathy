# handoff: cpsafe glob only copies first match

## .what

`cpsafe` with glob pattern only copies the first matched file, not all matches.

## .observed

```sh
npx rhachet run --skill cpsafe --from '.behavior/v2025_12_17.domain-transfer/*.md' --into '.behavior/v2025_12_17.domain-transfer/priors/v2/'
```

**expected**: copies all 20+ `.md` files that match the glob
**actual**: copies only `0.wish.md` (first match)

## .repro

```sh
# setup
mkdir -p /tmp/cpsafe-test/src /tmp/cpsafe-test/dst
touch /tmp/cpsafe-test/src/{a,b,c}.md

# test
npx rhachet run --skill cpsafe --from '/tmp/cpsafe-test/src/*.md' --into '/tmp/cpsafe-test/dst/'

# check
ls /tmp/cpsafe-test/dst/
# expected: a.md b.md c.md
# actual: a.md (only first match)
```

## .fix

in `cpsafe.sh`, the glob expansion likely returns multiple files but the copy logic only processes the first one.

likely fix: iterate over all glob matches:

```bash
# before (pseudocode)
SOURCE="$1"
cp "$SOURCE" "$DEST"

# after (pseudocode)
for SOURCE in $1; do
  cp "$SOURCE" "$DEST"
done
```
