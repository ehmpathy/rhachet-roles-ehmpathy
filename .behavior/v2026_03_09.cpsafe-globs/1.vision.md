# vision: glob-aware safe fileops + chill output vibes

## the outcome world

### day-in-the-life

a mechanic needs to archive 20+ behavior stones to a `priors/` folder:

```sh
# before: painful, error-prone
npx rhachet run --skill cpsafe --from '.behavior/v2025_12_17/*.md' --into '.behavior/v2025_12_17/priors/v2/'
# only copies first file (0.wish.md), silently ignores rest

# after: works as expected
npx rhachet run --skill cpsafe --from '.behavior/v2025_12_17/*.md' --into '.behavior/v2025_12_17/priors/v2/'
# copies all 20+ matching files
# output:
#   copied: .behavior/v2025_12_17/0.wish.md -> .behavior/v2025_12_17/priors/v2/0.wish.md
#   copied: .behavior/v2025_12_17/1.vision.md -> .behavior/v2025_12_17/priors/v2/1.vision.md
#   ... (all files)
#   ✓ copied 23 files
```

### before/after contrast

| aspect | before | after |
|--------|--------|-------|
| glob pattern `*.md` | copies first match only | copies all matches |
| shell expansion | unquoted globs expand before skill sees them | single-quoted globs pass as literals |
| feedback | silent partial success (dangerous!) | explicit per-file output + summary |
| permissions examples | `--from "*.md"` (double quotes) | `--from '*.md'` (single quotes) |

## scope

this behavior addresses:

1. **cpsafe**: fix glob to copy all matches, add chill tree output (printed incrementally)
2. **mvsafe**: fix glob to move all matches, add chill tree output (printed incrementally)
3. **rmsafe**: fix glob to remove all matches, add chill tree output (printed incrementally)
4. **sedreplace**: update output phrases for chill vibes
   - plan mode: "heres the wave..." (keep)
   - apply mode: "cowabunga!" → "sweet"
   - no matches: "lets see..." → "crickets..."
5. **documentation**: update examples to use single-quoted globs

**incremental output**: each file line prints as the operation completes, so the tree itself serves as live progress feedback

### timeline

1. user invokes skill with glob pattern
2. skill expands glob internally (not shell)
3. skill validates all matches are within repo boundary
4. skill performs operation on each match
5. skill outputs per-file result + summary count


## mental model

### how users describe this

> "cpsafe with wildcards - like `cp *.md dest/` but safe"

### analogies

- **like `git add '*.ts'`**: git handles the glob, not the shell
- **like `find -exec`**: operates on all matches, not just first
- **like array.forEach()**: processes each element, not just first

### terms

| user says | we say |
|-----------|--------|
| "wildcard" | "glob pattern" |
| "all files matching" | "glob expansion" |
| "copy everything" | "batch copy" |


## evaluation

### how well does it solve the goals?

| goal | solution |
|------|----------|
| copy multiple files at once | ✅ glob expansion handles all matches |
| prevent shell expansion issues | ✅ single-quote examples in docs |
| maintain security boundary | ✅ each file validated before operation |
| clear feedback | ✅ per-file output + summary count |

### pros

- **familiar syntax**: uses standard glob patterns (`*`, `**`, `?`)
- **safe by default**: all matches validated within repo
- **explicit output**: no silent partial success
- **consistent**: same pattern across cpsafe, mvsafe, rmsafe

### cons

- **no plan mode**: can't preview matches without executing (acceptable tradeoff)
- **no exclude patterns**: `--exclude` would add complexity (future enhancement)

### edgecases & pit of success

| edgecase | handling |
|----------|----------|
| glob matches 0 files | error: "no files matched pattern: *.xyz" |
| glob matches files outside repo | fail-fast with error (consistent with security boundary) |
| destination is file, not dir | error if >1 source file (can't copy many to one) |
| unquoted glob in shell | shell expands → only first arg reaches skill (documented in examples) |


## open questions & assumptions

### assumptions

1. **glob expansion happens inside skill**: we use bash glob expansion after receiving the pattern as a string literal
2. **single quotes prevent shell expansion**: `'*.md'` passes literally, `"*.md"` may expand
3. **destination is always a directory for multi-file ops**: when glob matches >1 file

### questions (resolved)

1. **fail-fast vs continue on error?** ✅
   - resolved: fail-fast for safety (per precedent: sedreplace, git add, cp)

2. **symlinks in globs?** ✅
   - resolved: follow symlinks, validate symlink location (per precedent: git behavior)

3. **recursive globs `**/*.md`?** ✅
   - resolved: yes, support recursive patterns (per extant behavior: sedreplace)


## what is awkward?

### single-quote requirement

users must remember to single-quote glob patterns:
```sh
# works
npx rhachet run --skill cpsafe --from '*.md' --into 'dest/'

# broken (shell expands *.md before skill runs)
npx rhachet run --skill cpsafe --from *.md --into dest/
```

**mitigation**: clear documentation + examples with single quotes everywhere

### directory as destination

when copying multiple files, `--into` must be a directory:
```sh
# works (dest/ is a directory)
npx rhachet run --skill cpsafe --from '*.md' --into 'dest/'

# fails (can't copy multiple files to single file)
npx rhachet run --skill cpsafe --from '*.md' --into 'merged.md'
```

**mitigation**: clear error message when destination should be directory

### large batches show many lines

copying 100+ files shows 100+ lines of output.

**mitigation**: print each line as the file is processed. the tree itself becomes live progress — each new line shows work done. no spinner or progress bar needed.
