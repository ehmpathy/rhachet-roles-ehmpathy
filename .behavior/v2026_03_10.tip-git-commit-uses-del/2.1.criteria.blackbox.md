# blackbox criteria: git.commit.uses del

## usecase.1 = revoke quota via verbose command sees tip

```
given a human with quota granted
  when they run `rhx git.commit.uses set --quant 0 --push block`
    then output shows "revoked" as first item
      sothat human knows quota was revoked
    then output shows tip as peer: "tip: 'rhx git.commit.uses del' does the same"
      sothat human discovers the shorter command
    then tip is dim/muted color
      sothat tip doesn't compete with main output
```

## usecase.2 = revoke quota via del shortcut

```
given a human with quota granted
  when they run `rhx git.commit.uses del`
    then output shows "revoked"
      sothat human knows quota was revoked
    then NO tip is shown
      sothat output stays minimal (user already knows the shortcut)
```

## usecase.3 = del when no quota exists

```
given a human with no quota set
  when they run `rhx git.commit.uses del`
    then command succeeds (idempotent)
      sothat retries are safe
    then output shows "revoked"
      sothat behavior is consistent
```

## usecase.4 = verbose revoke with --push allow does NOT show tip

```
given a human with quota granted
  when they run `rhx git.commit.uses set --quant 0 --push allow`
    then output shows commits: 0 and push: allowed
      sothat human sees the state
    then NO tip is shown
      sothat tip only appears for full revoke (del semantics)
```

## usecase.5 = short form revoke shows tip

```
given a human with quota granted
  when they run `rhx git.commit.uses set --quant 0`
    then --push defaults to block
      sothat revoke is easy
    then output shows "revoked" with tip
      sothat human discovers del shortcut
```

## exchange contracts

### del command

| input | output |
|-------|--------|
| `rhx git.commit.uses del` | revoked (no tip) |

### set --quant 0 (with or without --push block)

| input | output |
|-------|--------|
| `set --quant 0` | revoked + tip |
| `set --quant 0 --push block` | revoked + tip |
| `set --quant 0 --push allow` | commits: 0, push: allowed (no tip) |

### tip format

```
🐚 git.commit.uses set
   ├─ revoked
   └─ tip: 'rhx git.commit.uses del' does the same
```

- tip is peer of "revoked", not child
- tip text is dim/muted via ANSI codes
