# factory opportunities research: git.commit.uses del

## the constraint

**THE bottleneck:** context rebuild between sessions.

when context is lost (session timeout, context limit), research must be redone. the behavior route system mitigates this via persisted research artifacts, but the bottleneck remains: reconstruct mental model from artifacts.

---

## flow and queues

| queue | wait time | controllable? |
|-------|-----------|---------------|
| npm run build | ~2s | yes (incremental builds) |
| npm run test:integration | ~3s | yes (filter to file) |
| CI pipeline | ~2-5min | no (external) |

**work-in-progress limit:** 1 behavior at a time (serial route).

---

## waste

| waste type | occurrence | impact |
|------------|------------|--------|
| repeated build | every test run | minor (~2s) |
| manual snapshot update | when output format changes | minor |
| re-read source files | after context loss | moderate |

**repeated setup:** temp git repo created per test case. could share across cases in same `given` block, but isolation is valuable.

---

## small batches

verification is already incremental:
- single test file: `npm run test:integration -- git.commit.uses`
- single test case: `npm run test:integration -- git.commit.uses -t "case1"`
- snapshot updates: `RESNAP=true npm run test:integration`

**smallest verifiable unit:** single `then` block assertion.

## systems view

**opportunity that compounds:** add `print_tip()` to output.sh.

this benefits:
- this behavior (shows dim tip)
- all future skills that want tip-style output
- consistent visual language across all git.commit.* skills

**local vs global:** implement `print_tip()` is a global improvement disguised as a local need.

## summary

| category | opportunity | compound value |
|----------|-------------|----------------|
| constraint | persist research artifacts | already handled by behavior route |
