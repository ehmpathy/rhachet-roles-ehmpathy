## user experience

### usecases

| goal | command before | command after |
|------|----------------|---------------|
| revoke all quota | `set --quant 0 --push block` | `del` |
| grant quota | `set --quant N --push allow\|block` | unchanged |
| check quota | `get` | unchanged |

### timeline

1. human grants quota: `rhx git.commit.uses set --quant 3 --push allow`
2. mechanic commits
3. human wants to revoke quota left
4. human runs `set --quant 0` тЖТ sees tip тЖТ next time uses `del`

## mental model

### how users describe it

> "del removes the quota, set grants it, get checks it"

follows the get/set/del pattern familiar from key-value stores and caches.

### analogies

- `redis-cli SET key value` vs `redis-cli DEL key`
- `rm file` vs `echo "" > file`
- revoke vs explicitly-grant-zero

### terms

| user says | we say |
|-----------|--------|
| "revoke" | del |
| "cancel" | del |
| "clear" | del |
| "remove quota" | del |

## evaluation

### pros

- shorter command for common operation
- discoverable via tip (no docs search)
- follows get/set/del convention
- tip appears only on revoke, not spam

### cons

- adds another code path (minimal)
- tip adds one line to output (acceptable)

### edge cases

| case | behavior |
|------|----------|
| `del` when no quota exists | succeeds (idempotent) |
| `set --quant 0` (short form) | tip shown (defaults to --push block) |
| `set --quant 0 --push block` (full form) | tip shown |
| `set --quant 0 --push allow` | NO tip (not equivalent to `del`) |
| tip style | dim/muted via ANSI escape codes |

### implementation notes

output.sh currently lacks color support. implementation requires:
- add `print_tip` function with ANSI dim style (`\033[2m` for dim, `\033[0m` for reset)
- or inline ANSI codes in git.commit.uses.sh

## what is awkward?

- `del` feels right but `unset` matches `set` better grammatically
  - **decision:** use `del` per wisher confirmation
- tip only shows when result equals `del` semantics (quant=0 AND push=block)
  - `set --quant 0 --push allow` = no tip (intentional тАФ not a full revoke)
  - **decision:** this is correct behavior, not awkward

overall: feels natural, minimal change, good ergonomics.
