# playtest: git.commit.uses del

## prerequisites

```sh
# build the skill
npm run build

# link to .agent
npx rhachet roles link --role mechanic

# reinitialize
npx rhachet roles init --role mechanic
```

## happy paths

### path 1: verbose revoke shows tip

```sh
# grant quota first
rhx git.commit.uses set --quant 1 --push block

# revoke with verbose command
rhx git.commit.uses set --quant 0 --push block
```

**expected output:**
```
🐢 groovy, break time

🐚 git.commit.uses set
   ├─ revoked
   └─ tip: 'rhx git.commit.uses del' does the same
```

**verify:**
- [ ] turtle header shows "groovy, break time"
- [ ] tree shows "git.commit.uses set"
- [ ] first branch shows "revoked" with `├─`
- [ ] second branch shows tip with `└─`
- [ ] tip text is dim/muted (visually lighter than main text)

---

### path 2: del shortcut works

```sh
# grant quota first
rhx git.commit.uses set --quant 1 --push block

# revoke with del shortcut
rhx git.commit.uses del
```

**expected output:**
```
🐢 groovy, break time

🐚 git.commit.uses del
   └─ revoked
```

**verify:**
- [ ] turtle header shows "groovy, break time"
- [ ] tree shows "git.commit.uses del" (not "set")
- [ ] single leaf shows "revoked" with `└─`
- [ ] NO tip is shown

---

### path 3: short form revoke shows tip

```sh
# grant quota first
rhx git.commit.uses set --quant 1 --push block

# revoke without explicit --push
rhx git.commit.uses set --quant 0
```

**expected output:**
```
🐢 groovy, break time

🐚 git.commit.uses set
   ├─ revoked
   └─ tip: 'rhx git.commit.uses del' does the same
```

**verify:**
- [ ] --push defaults to block
- [ ] tip is shown (same as verbose revoke)

---

## edge paths

### edge 1: del when no quota exists (idempotent)

```sh
# ensure no quota (run del twice or start fresh)
rhx git.commit.uses del
rhx git.commit.uses del
```

**expected output:**
```
🐢 groovy, break time

🐚 git.commit.uses del
   └─ revoked
```

**verify:**
- [ ] command succeeds (exit code 0)
- [ ] output shows "revoked" (consistent behavior)
- [ ] no error about absent quota

---

### edge 2: push-only mode has NO tip

```sh
# set push-only mode (quant=0 but push=allow)
rhx git.commit.uses set --quant 0 --push allow
```

**expected output:**
```
🐢 let it ride

🐚 git.commit.uses set
   ├─ commits: 0
   └─ push: allowed
```

**verify:**
- [ ] shows "let it ride" (not "groovy, break time")
- [ ] shows "commits: 0" and "push: allowed"
- [ ] NO tip is shown (this is NOT equivalent to del)
