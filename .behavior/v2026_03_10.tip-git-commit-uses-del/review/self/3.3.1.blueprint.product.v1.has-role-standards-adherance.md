## potential issues investigated

### issue: else branch in lines 121-126

**rule.forbid.else-branches says:** no else branches; use early returns

**investigation:** the else branch selects between two similar outputs:
- VIA_DEL set: `└─ revoked` (terminal)
- VIA_DEL unset: `├─ revoked` + tip (non-terminal)

**refactor attempt:**
```bash
# early return version
if [[ -n "$VIA_DEL" ]]; then
  echo "   └─ revoked"
  return  # or exit block
fi
echo "   ├─ revoked"
print_tip "..."
```

**analysis:** bash lacks clean early returns within blocks. using `return` would exit the function. the current if/else is clearer for this output-only branch.

**verdict:** acceptable deviation. the rule targets control flow logic, not output selection.

### issue: del command not in get/set/gen

**rule.require.get-set-gen-verbs says:** operations use get, set, or gen

**investigation:** `del` is a CLI subcommand, not a domain operation name. vision explicitly chose this verb for UX reasons (parallels redis DEL, get/set/del triad).

**verdict:** acceptable. rule applies to domain operations, not CLI aliases.
