## potential issues investigated

### issue: tree start label for del command

**concern:** does `${VIA_DEL:+del}` correctly output "del" vs "set"?

**verification:**
- when VIA_DEL="true": `${VIA_DEL:+del}` expands to "del"
- when VIA_DEL unset: `${VIA_DEL:+del}` expands to ""
- output: "git.commit.uses del" vs "git.commit.uses set"

**verdict:** correct. vision line 37 shows `🐚 git.commit.uses del`, blueprint achieves this.

### issue: fall-through semantics with `;&`

**concern:** does bash `;&` correctly fall through without re-evaluation of case?

**verification:**
- bash 4.0+ `;&` means unconditional fall-through
- del case sets variables, then falls through to set logic
- set logic uses those variables

**verdict:** correct. standard bash behavior.

### issue: tip not shown for del (even though condition matches)

**concern:** del has QUANT=0 and PUSH=block, so condition line 118 is true. why no tip?

**verification:**
- line 121: `if [[ -n "$VIA_DEL" ]]` checks VIA_DEL first
- if VIA_DEL is set, outputs `└─ revoked` and exits before tip
- else branch outputs `├─ revoked` + tip

**verdict:** correct. VIA_DEL flag distinguishes del from set.
