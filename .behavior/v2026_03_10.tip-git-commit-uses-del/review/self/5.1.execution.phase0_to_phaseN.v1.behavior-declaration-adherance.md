**blueprint spec** (lines 117-127):

**actual implementation** (git.commit.uses.sh):

```
line 159:   if [[ -n "$VIA_DEL" ]]; then

line 166:   fi
```

**deviation analysis**:

blueprint used `${VIA_DEL:+del}` then nested if for revoked output.
implementation uses nested if for both tree_start and revoked output.

| scenario | blueprint output | actual output |
|----------|------------------|---------------|
| via del | `git.commit.uses del` + `└─ revoked` | same |
| via set | `git.commit.uses ` + `├─ revoked` + tip | `git.commit.uses set` + `├─ revoked` + tip |

note: blueprint has bug: `${VIA_DEL:+del}` produces "del" when set, empty when not.
so blueprint would output "git.commit.uses del" or "git.commit.uses " (extra space at end).

actual implementation correctly outputs "git.commit.uses set" when not via del.

**verdict**: implementation is BETTER than blueprint spec ✓
