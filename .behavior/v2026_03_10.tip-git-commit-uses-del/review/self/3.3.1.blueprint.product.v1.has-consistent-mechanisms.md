## new mechanism: print_tip()

**blueprint says:**
```bash
print_tip() {
  local text="$1"
  # \033[2m = dim, \033[0m = reset
  echo -e "   └─ \033[2mtip: $text\033[0m"
}
```

**does this duplicate extant functionality?**
no. extant functions do not support dim ANSI.

**could we reuse extant component?**
- `print_tree_branch("tip: ...", true)` → outputs `└─ tip: ...` but no dim
- `print_nested_leaf("tip", "...", true)` → outputs `└─ tip: ...` but no dim

neither provides dim style. need new function.

**could we modify extant function?**
could add `style` parameter to `print_tree_branch`. but this adds complexity to all callers.

**verdict:** new function is correct. simple, focused, no parameter bloat.
