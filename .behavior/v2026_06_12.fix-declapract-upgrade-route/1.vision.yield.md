# vision: fix declapract.upgrade route commands

## the outcome world

### what does a day-in-the-life look like?

when a mechanic runs `rhx declapract.upgrade init`, the route is created with guards that use the standardized `$rhx` command. guards execute without confusion about which invocation style to use.

### before/after contrast

| before | after |
|--------|-------|
| `npx rhachet run --repo bhrain --skill route.stone.judge ...` | `$rhx route.stone.judge ...` |
| `npx rhachet run --repo bhrain --skill route.bind.set ...` | `$rhx route.bind.set ...` |
| verbose, 47+ chars per invocation | concise, ~20 chars per invocation |
| inconsistent with other skills | consistent with rhachet-roles-bhrain conventions |

### the "aha" moment

when guards execute cleanly and the command output is scannable — no verbose `npx rhachet run --repo` noise cluttering logs.

## user experience

### usecases

1. mechanic inits a declapract.upgrade route
2. route guards execute during stone transitions
3. `route.stone.judge` evaluates approval conditions

### contract inputs & outputs

no change to contracts. this is a pure find-and-replace:

```bash
# before
npx rhachet run --repo bhrain --skill route.stone.judge --mechanism approved? --stone $stone --route $route

# after
$rhx route.stone.judge --mechanism approved? --stone $stone --route $route
```

### timelines

immediate — no migration needed. routes created after this fix use `$rhx`. extant routes continue to work (both patterns invoke the same underlying skill).

## mental model

### how would users describe this?

"we shortened the skill invocation from `npx rhachet run --repo X --skill Y` to just `$rhx Y`"

### analogies

like shell aliases: `ll` instead of `ls -la`. same behavior, less typing.

### terms

| user term | internal term |
|-----------|---------------|
| `$rhx` | rhachet shorthand alias |
| skill invocation | `npx rhachet run --repo ... --skill ...` |

perfectly — pure mechanical replacement, no semantic change.

- maintainability: one invocation pattern to remember

### cons
