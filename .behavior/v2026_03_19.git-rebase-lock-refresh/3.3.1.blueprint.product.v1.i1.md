# 3.3.1.blueprint.product: git.branch.rebase lock refresh

## summary

build two features:
1. `rhx git.branch.rebase lock refresh` — regenerate lock file mid-rebase
2. proactive suggestion in `take` output when lock files are settled

---

## filediff tree

```
src/domain.roles/mechanic/skills/git.branch.rebase/
├─ [~] git.branch.rebase.sh                          # add "lock" case, dispatch to lock.sh
├─ [+] git.branch.rebase.lock.sh                     # lock subcommand handler (handles "refresh")
├─ [+] git.branch.rebase.lock.integration.test.ts
├─ [~] git.branch.rebase.take.sh                     # add suggestion for lock files
└─ [~] git.branch.rebase.take.integration.test.ts    # add test for suggestion output
```

**note:** dispatcher routes "lock" to `git.branch.rebase.lock.sh` which handles "refresh" as first positional arg

## codepath tree

### git.branch.rebase.sh (dispatcher)
```
[○] parse rhachet args
[○] parse subcommand
[~] validate subcommand
   └─ [+] add "lock" to valid cases: begin|continue|take|abort|lock
[○] dispatch via exec
```

### git.branch.rebase.lock.sh (new — lock subcommand handler)
```
[+] define turtle vibes output functions (inline)
[+] source shared operations
[+] parse rhachet args
[+] parse subcommand (first positional arg)
   └─ [+] "refresh" → proceed with lock refresh
   └─ [+] unknown → error "unknown lock subcommand"
[+] guard: is_rebase_in_progress
[+] detect lock file
   ├─ [+] check pnpm-lock.yaml extant
   ├─ [+] check package-lock.json extant
   └─ [+] check yarn.lock extant
[+] guard: at least one lock file extant
[+] detect package manager
   ├─ [+] pnpm-lock.yaml → check pnpm installed
   ├─ [+] package-lock.json → npm (always available)
   └─ [+] yarn.lock → check yarn installed
[+] guard: package manager available
[+] run install
   ├─ [+] pnpm install (for pnpm)
   ├─ [+] npm install (for npm)
   └─ [+] yarn install (for yarn)
[+] guard: install succeeded
[+] stage lock file
[+] output with turtle vibes
   ├─ [+] detected: {pm}
   ├─ [+] run: {pm} install
   ├─ [+] staged: {lockfile} ✓
   └─ [+] done
```

### git.branch.rebase.take.sh (update — add suggestion)
```
[○] settle files
[○] output settled files
[~] if any settled file is a lock file
   └─ [+] add suggestion once (after all settled files)
      └─ [+] "lock taken, refresh it with: ⚡"
         └─ [+] "rhx git.branch.rebase lock refresh"
[○] output done
```

## test coverage

### git.branch.rebase.lock.integration.test.ts
```
[case1] rebase in progress with pnpm-lock.yaml
  [t0] lock refresh with pnpm
    → lock regenerated, staged, exit 0
    → output shows detected: pnpm, done

[case2] rebase in progress with package-lock.json (no pnpm-lock)
  [t0] lock refresh with npm
    → lock regenerated, staged, exit 0
    → output shows detected: npm, done

[case3] rebase in progress with yarn.lock (no pnpm, no npm lock)
  [t0] lock refresh with yarn
    → lock regenerated, staged, exit 0
    → output shows detected: yarn, done

[case4] no rebase in progress
  [t0] attempt lock refresh
    → exit 1
    → output shows error: no rebase in progress

[case5] rebase in progress but no lock file
  [t0] attempt lock refresh
    → exit 1
    → output shows error: no lock file found

[case6] pnpm-lock.yaml extant but pnpm not installed
  [t0] attempt lock refresh
    → exit 1
    → output shows error: pnpm not found

[case7] yarn.lock extant but yarn not installed
  [t0] attempt lock refresh
    → exit 1
    → output shows error: yarn not found

[case8] both pnpm-lock and package-lock extant
  [t0] lock refresh (pnpm available)
    → uses pnpm (preferred)
  [t1] lock refresh (pnpm not available)
    → uses npm (fallback)

[case9] install fails
  [t0] lock refresh with bad package.json
    → exit 1
    → output shows install error
```

### git.branch.rebase.take.integration.test.ts (update)

```
[case11] take lock file shows suggestion
```

## operations decomposition

### reuse from shared operations
- `is_rebase_in_progress()` — check rebase state
- `get_git_dir()` — get git directory path

### new operations (in git.branch.rebase.lock.sh)
- `detect_lock_file()` — returns lock file path or empty
- `detect_package_manager()` — returns pnpm|npm|yarn
- `is_pm_installed()` — check if pm binary available
- `run_install()` — execute pm install

### turtle vibes output
- inline output functions in lock.sh (follows YAGNI — single consumer)
- pattern: `print_turtle_header()`, `print_tree_start()` (same as git.commit/output.sh)

---

## priority order (package manager selection)

1. pnpm-lock.yaml + pnpm installed → pnpm
2. pnpm-lock.yaml + pnpm not installed → error
3. package-lock.json → npm
4. yarn.lock + yarn installed → yarn
5. yarn.lock + yarn not installed → error
6. no lock file → error
