# 2.2.criteria.blackbox.matrix: git.branch.rebase lock refresh

## matrix.1 = lock refresh command

dimensions:
- rebase in progress
- lock file extant
- package manager installed
- install succeeds

| ind: rebase? | ind: lock extant? | ind: pm installed? | ind: install ok? | dep: outcome |
|--------------|-------------------|--------------------|--------------------|--------------|
| no | - | - | - | error: "no rebase in progress" |
| yes | no | - | - | error: "no lock file found" |
| yes | yes (pnpm) | no | - | error: "pnpm not found, install pnpm or use npm" |
| yes | yes (yarn) | no | - | error: "yarn not found, install yarn" |
| yes | yes | yes | no | error output + manual fix suggestion |
| yes | yes | yes | yes | lock regenerated + staged + success output |

**notes:**
- npm is assumed always installed (fallback)
- `-` means dimension not applicable for this row

---

## matrix.2 = proactive suggestion after take

dimensions:
- file type (lock vs other)
- multiple files taken

| ind: file type | ind: multiple files? | dep: suggestion shown? | dep: suggestion count |
|----------------|----------------------|------------------------|----------------------|
| lock file | no | yes | 1 |
| lock file | yes (includes lock) | yes | 1 (once, not per file) |
| other file | no | no | 0 |
| other file | yes (no lock) | no | 0 |

**notes:**
- suggestion appears under the settled lock file in tree output
- when multiple lock files in `take .`, suggestion shown once

---

## matrix.3 = package manager detection

dimensions:
- pnpm-lock.yaml extant
- package-lock.json extant
- yarn.lock extant
- pnpm installed

| ind: pnpm-lock? | ind: package-lock? | ind: yarn.lock? | ind: pnpm installed? | dep: pm used |
|-----------------|--------------------|-----------------|-----------------------|--------------|
| yes | no | no | yes | pnpm |
| yes | no | no | no | error: pnpm not found |
| no | yes | no | - | npm |
| no | no | yes | - | yarn (if installed) |
| yes | yes | no | yes | pnpm (preferred) |
| yes | yes | no | no | npm (fallback) |
| no | yes | yes | - | npm (package-lock preferred over yarn) |

**gap flagged:**
- what if yarn.lock extant but yarn not installed? → error: "yarn not found"
- what if all three lock files extant? → use pnpm if installed, else npm

**added combinations:**

| ind: pnpm-lock? | ind: package-lock? | ind: yarn.lock? | ind: pnpm installed? | ind: yarn installed? | dep: pm used |
|-----------------|--------------------|-----------------|-----------------------|----------------------|--------------|
| no | no | yes | - | no | error: yarn not found |
| yes | yes | yes | yes | - | pnpm (preferred) |
| yes | yes | yes | no | - | npm (second preference) |

---

## decomposition analysis

**matrix.1 (lock refresh):** 4 dimensions, 6 meaningful rows — acceptable scope
**matrix.2 (suggestion):** 2 dimensions, 4 rows — clean scope
**matrix.3 (pm detection):** 5 dimensions (with yarn installed) — borders on complex

**recommendation:** matrix.3 could be simplified via a clear priority order:
1. pnpm-lock.yaml + pnpm installed → pnpm
2. pnpm-lock.yaml + pnpm not installed → error (or npm fallback)
3. package-lock.json → npm
4. yarn.lock + yarn installed → yarn
5. yarn.lock + yarn not installed → error
6. none → error: "no lock file found"

this reduces the matrix to a linear priority list, which is easier to implement and test.
