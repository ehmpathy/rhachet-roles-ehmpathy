## 2. temp git repo helper pattern [REUSE]

### what it is

create isolated git repositories for integration tests via `genTempDir` and git init.

### where found

- `src/domain.roles/mechanic/skills/git.commit/git.commit.set.integration.test.ts:25-45`
- `src/domain.roles/mechanic/skills/claude.tools/cpsafe.integration.test.ts:30-55`

### key elements

```typescript
import { genTempDir } from '@ehmpathy/as-command';
import { spawnSync } from 'child_process';
import * as fs from 'fs';
import * as path from 'path';

const setupTempGitRepo = (options?: {
  files?: Record<string, string>;
  withCommit?: boolean;
}) => {
  const tempDir = genTempDir();

  // init git repo
  spawnSync('git', ['init'], { cwd: tempDir });
  spawnSync('git', ['config', 'user.email', 'test@test.com'], { cwd: tempDir });
  spawnSync('git', ['config', 'user.name', 'Test User'], { cwd: tempDir });

  // create files
  if (options?.files) {
    for (const [filePath, content] of Object.entries(options.files)) {
      const fullPath = path.join(tempDir, filePath);
      fs.mkdirSync(path.dirname(fullPath), { recursive: true });
      fs.writeFileSync(fullPath, content);
    }
  }

  // optionally commit
  if (options?.withCommit) {
    spawnSync('git', ['add', '-A'], { cwd: tempDir });
    spawnSync('git', ['commit', '-m', 'initial commit'], { cwd: tempDir });
  }

  return tempDir;
};
```

### disposition

**[REUSE]** — init.declapract.upgrade tests need temp git repos with declapract.use.yml files.

## 3. spawnSync execution pattern [REUSE]

### what it is

execute shell skills via `spawnSync` and capture stdout, stderr, exit code.

### where found

- `src/domain.roles/mechanic/skills/claude.tools/cpsafe.integration.test.ts:60-80`
- `src/domain.roles/mechanic/skills/git.commit/git.commit.set.integration.test.ts:70-90`

### key elements

```typescript
import { spawnSync } from 'child_process';
import * as path from 'path';

const runSkill = (args: string[], options?: { cwd?: string; env?: Record<string, string> }) => {
  const skillPath = path.join(__dirname, '../skillname.sh');

  const result = spawnSync('bash', [skillPath, ...args], {
    cwd: options?.cwd ?? process.cwd(),
    encoding: 'utf-8',
    env: { ...process.env, ...options?.env },
    stdio: ['pipe', 'pipe', 'pipe'],
  });

  return {
    stdout: result.stdout,
    stderr: result.stderr,
    exitCode: result.status,
  };
};
```

### disposition

**[REUSE]** — init.declapract.upgrade tests should use spawnSync to execute the skill and capture output.

  const result = runSkill(['--arg', 'value'], { cwd: tempDir });
  expect(result.exitCode).toBe(0);

// user error case (bad input)

  const result = runSkill(['--invalid'], { cwd: tempDir });
  expect(result.exitCode).toBe(1);
  expect(result.stderr).toContain('error:');

// system error case (prerequisite missing)

  const result = runSkill([], { cwd: tempDir });
  expect(result.exitCode).toBe(2);
  expect(result.stderr).toContain('not found');
});
```

### disposition

**[REUSE]** — init.declapract.upgrade should use exit code 2 for missing prerequisites (declapract.use.yml, pnpm, fnm).

given('[case1] repo with staged changes', () => {

    const tempDir = setupTempGitRepo({
      files: { 'declapract.use.yml': 'practices:\n  - declapract-typescript-ehmpathy' },
      withCommit: true,
    });

    // add unstaged change

    spawnSync('git', ['add', 'new-file.txt'], { cwd: tempDir });

    return { tempDir };
  });

  when('[t0] skill is invoked', () => {
    then('sees the staged file', () => {
      const result = runSkill(['--mode', 'plan'], { cwd: scene.tempDir });
      expect(result.stdout).toContain('new-file.txt');
    });
  });
});
```

### disposition

**[REUSE]** — init.declapract.upgrade tests should use useBeforeAll for shared repo setup within given blocks.

## implementation implications

### test file structure

```
src/domain.roles/mechanic/skills/
├── init.declapract.upgrade.sh
└── init.declapract.upgrade.integration.test.ts
```

### key test cases for init.declapract.upgrade

1. **[case1] repo with declapract.use.yml** — happy path, route instantiation
2. **[case2] repo without declapract.use.yml** — fail with "not a declapract repo"
3. **[case3] pnpm not available** — fail with "pnpm not found"
4. **[case4] plan mode** — shows route preview without creation
5. **[case5] apply mode** — creates route directory and files
6. **[case6] idempotent apply** — running twice produces same result

### assertions to include

- exit code 0 for success, 2 for missing prerequisites
- route directory created at `.route/v$isodate.declapract.upgrade/`
- all stone files present (1.upgrade.invoke, 2.detect.hazards, 3.1-3.4.*)
- all guard files present (2.detect.hazards, 3.1-3.4.*)
- route bind file created (via rhachet route.bind invocation)
- turtle vibes output with 🐢 emoji
