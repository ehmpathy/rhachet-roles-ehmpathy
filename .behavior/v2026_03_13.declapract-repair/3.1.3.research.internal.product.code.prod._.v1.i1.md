## 1. skill structure pattern [REUSE]

### what it is

all mechanic skills follow a standard structure with header documentation, fail-fast guards, named argument parse, plan/apply modes, and turtle-vibed output.

### where found

- `src/domain.roles/mechanic/skills/git.commit/git.commit.set.sh`
- `src/domain.roles/mechanic/skills/show.gh.action.logs.sh`
- `src/domain.roles/mechanic/skills/declapract.upgrade.sh`

### key elements

```bash
#!/usr/bin/env bash
######################################################################
# .what = one-line description
#
# .why  = why this skill exists
#         - benefit 1
#         - benefit 2
#
# usage:
#   skill.sh --arg "value"
#
# guarantee:
#   - what this skill guarantees
######################################################################
set -euo pipefail

# argument parse with rhachet passthrough handle
while [[ $# -gt 0 ]]; do
  case $1 in
    --skill|--repo|--role)
      shift 2  # rhachet passthrough - ignore
      ;;
    --my-arg)
      MY_ARG="$2"
      shift 2
      ;;
  esac
done

# fail-fast guards
if ! command -v pnpm &> /dev/null; then
  echo "error: pnpm not found"
  exit 2
fi
```

### disposition

**[REUSE]** — the skill header format, argument parse pattern, and fail-fast guards are standard patterns to reuse exactly.

## 2. plan/apply mode pattern [REUSE]

### what it is

skills default to plan mode (preview only) and require explicit `--mode apply` to execute mutations.

### where found

- `src/domain.roles/mechanic/skills/git.commit/git.commit.set.sh:114,532-602,604-715`
- `src/domain.roles/mechanic/skills/git.rebase/git.rebase.begin.sh`

### key elements

```bash
MODE="plan"  # default to plan

while [[ $# -gt 0 ]]; do
  case $1 in
    --mode)
      MODE="$2"
      shift 2
      ;;
  esac
done

# validate mode
if [[ "$MODE" != "plan" && "$MODE" != "apply" ]]; then
  echo "error: --mode must be 'plan' or 'apply'"
  exit 2
fi

# plan mode: show preview
if [[ "$MODE" == "plan" ]]; then
  echo "preview: would execute..."
  echo ""
  echo "run with --mode apply to execute"
  exit 0
fi

# apply mode: execute
execute_mutations
```

### disposition

**[REUSE]** — init.declapract.upgrade should default to plan mode (instantiate route preview) and require apply to execute.

## 5. quota/meter pattern [EXTEND]

### what it is

track and limit resource usage via `.meter/` directory with jsonc state files.

### where found

- `src/domain.roles/mechanic/skills/git.commit/git.commit.uses.sh`
- `src/domain.roles/mechanic/skills/git.commit/git.commit.set.sh:46-49,416-436`

### key elements

```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
METER_DIR="$REPO_ROOT/.meter"
STATE_FILE="$METER_DIR/git.commit.uses.jsonc"

# read state
USES=$(jq -r '.uses' "$STATE_FILE")

# decrement uses
NEW_USES=$((USES - 1))
cat > "$STATE_FILE" << EOF
{
  "uses": $NEW_USES
}
EOF
```

### disposition

**[EXTEND]** — init.declapract.upgrade does not need quota meter, but may track route state in a similar pattern (e.g., `.route/v$isodate.declapract.upgrade/.state.jsonc`).

## 10. guard patterns from behaviors [REUSE]

### what it is

guards block passage until satisfied. they can require self-review, judge invocations, or human approval.

### where found

- `.behavior/v2026_03_13.declapract-repair/1.vision.guard`
- `.behavior/v2026_03_11.git-rebase/3.3.1.blueprint.product.v1.guard`

### key elements

```yaml
# self-review guard
self-reviews:
  - question: did we complete all required steps?
    required: true

# judge guard (human approval)
judges:
  - npx rhachet run --repo bhrain --skill route.stone.judge --mechanism approved? --stone $stone --route $route

# compound guard (self-review + judge)
self-reviews:
  - question: are all tests green?
    required: true
judges:
  - npx rhachet run --repo bhrain --skill route.stone.judge --mechanism approved? --stone $stone --route $route
```

### disposition

**[REUSE]** — the route guards must follow this pattern:
- repair stones: self-review only (all tests/cicd pass)
- reflect stones: self-review + `approved?` judge (human approval)
