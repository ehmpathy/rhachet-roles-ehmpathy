- date: 2025 (current)
- credibility: [official]
> "The tool enables definition of best-practices and bad-practices via examples of what files should look like."

- source: https://github.com/ehmpathy/declapract-typescript-ehmpathy
- date: 2025 (current)
- credibility: [official]
> "The ehmpathy org's declared best practices for TypeScript, for usage with declapract."

### key terms

- **practice**: a declarative definition of what files should look like
- **best-practice**: an example of the desired state
- **bad-practice**: an anti-pattern to detect and fix
- **apply**: command that fixes files to match declared practices

## 7. test-driven development (TDD)

**[20] Wikipedia — Test-driven development**
- source: https://en.wikipedia.org/wiki/Test-driven_development
- date: 2025 (current)
- credibility: [official]
> "TDD involves: write an automated unit-level test case that fails, write just enough code to pass the test, then refactor."

**[21] CircleCI — TDD Explained**
- source: https://circleci.com/blog/test-driven-development-tdd/
- date: 2025
- credibility: [practitioner]
> "Early defect elimination usually avoids lengthy and tedious debug sessions later in the project."

**[22] IBM — What is TDD**
- source: https://www.ibm.com/think/topics/test-driven-development
- date: 2025 (current)
- credibility: [official]
> "Automated tests tend to cover every code path because no more code is written than necessary to pass a test case."

### key terms

- **red-green-refactor**: TDD cycle (fail, pass, improve)
- **unit test**: test for smallest testable unit
- **test coverage**: percentage of code exercised by tests
