# blackbox criteria: init.declapract.upgrade

## episode experience

# usecase.1 = upgrade repo to latest practices via rigid route

given(a repo with declapract.use.yml)
  when(mechanic invokes `rhx init.declapract.upgrade`)
    then(route instantiated at .route/v$isodate.declapract.upgrade/)
      sothat(the upgrade follows a structured path)
    then(route bound to current branch)
      sothat(route.drive can shepherd execution)
    then(mechanic sees confirmation of route instantiation)
      sothat(they know where to find the route)

  when(route.drive shepherds through 1.upgrade.invoke)
    then(declapract packages upgraded to latest)
    then(practices applied to repo)
    then(dependencies reinstalled)
    then(any infrastructure defects documented in artifact)
      sothat(infrastructure team can prevent recurrence)

  when(route.drive shepherds through 2.detect.hazards)
    then(diffs scanned for backwards incompat changes)
    then(hazards documented: tests, cicd, paths)
    then(guard validates completeness via self-review)

  when(route.drive shepherds through 3.1.repair.test.defects)
    then(all tests pass with zero failures)
    then(each defect documented with root cause)
      sothat(we understand why it arose from upgrade)

  when(route.drive shepherds through 3.2.reflect.test.defects)
    then(each defect classified: repo quirk vs practice bug vs adoption candidate)
    then(guard requires human approval)
      sothat(feedback to infrastructure is validated)

  when(route.drive shepherds through 3.3.repair.cicd.defects)
    then(mechanic requests commit permission from human)
    then(changes pushed with commit 'fix(practs): bump to latest best')
    then(cicd watched via `git release --watch`)
    then(cicd defects fixed until green)
    then(each defect documented with root cause)

  when(route.drive shepherds through 3.4.reflect.cicd.defects)
    then(each cicd defect classified)
    then(guard requires human approval)
    then(upgrade complete)
      sothat(repo runs on latest practices with documented learnings)

---

# usecase.2 = fail fast on absent prerequisites

given(a repo without declapract.use.yml)
  when(mechanic invokes `rhx init.declapract.upgrade`)
    then(skill fails with clear error: "not a declapract repo")
      sothat(mechanic knows to run declapract init first)

given(a machine without pnpm)
  when(mechanic invokes `rhx init.declapract.upgrade`)
    then(skill fails with clear error: "pnpm not found")
      sothat(mechanic knows to install pnpm)

given(a machine without fnm)
  when(1.upgrade.invoke runs)
    then(stone fails with clear error: "fnm not found")
      sothat(mechanic knows to install fnm)

---

# usecase.3 = handle unfixable defects

given(a defect caused by external dependency)
  when(mechanic cannot fix the defect)
    then(mechanic documents blocker with evidence)
    then(mechanic seeks human guidance)
      sothat(human can decide to skip or defer)

## exchange experience

# exchange.1 = init.declapract.upgrade invocation

input:
  - command: `rhx init.declapract.upgrade`
  - prerequisites: declapract.use.yml exists, pnpm available

output:
  - route instantiated at .route/v$isodate.declapract.upgrade/
  - route bound to current branch
  - confirmation message with route path

errors:
  - "not a declapract repo" — declapract.use.yml absent
  - "pnpm not found" — pnpm not on PATH

---

# exchange.2 = 1.upgrade.invoke stone

input:
  - repo with declapract.use.yml
  - fnm available

output:
  - artifact: 1.upgrade.invoke.v1.i1.md
  - declapract packages at latest
  - practices applied (twice for idempotency)
  - dependencies reinstalled

errors:
  - "fnm not found" — fnm not on PATH
  - any pnpm install failure — documented with root cause
  - any declapract apply failure — documented with root cause

---

# exchange.3 = stone artifacts

each stone emits an artifact:
  - 1.upgrade.invoke.v1.i1.md — execution log, infra defects
  - 2.detect.hazards.v1.i1.md — identified hazards
  - 3.1.repair.test.defects.v1.i1.md — test fixes with root causes
  - 3.2.reflect.test.defects.v1.i1.md — defect classifications
  - 3.3.repair.cicd.defects.v1.i1.md — cicd fixes with root causes
  - 3.4.reflect.cicd.defects.v1.i1.md — defect classifications

---

# exchange.4 = guard gates

guards block passage until satisfied:

| guard | requirement |
|-------|-------------|
| 2.detect.hazards.guard | self-review: hazards complete, mistaken items disproved |
| 3.1.repair.test.defects.guard | self-review: all tests pass, zero failures |
| 3.2.reflect.test.defects.guard | self-review + human approval |
| 3.3.repair.cicd.defects.guard | self-review: cicd green, zero failures |
| 3.4.reflect.cicd.defects.guard | self-review + human approval |
