# distill: domain objects and operations for init.declapract.upgrade

## usecases and contracts

### usecase 1: invoke the init skill

**contract:**
```sh
rhx init.declapract.upgrade                    # plan mode (preview)
rhx init.declapract.upgrade --mode apply       # apply mode (instantiate route)
```

**input:**
- cwd must contain `declapract.use.yml`
- pnpm must be available

**output:**
- route instantiated at `.route/v$isodate.declapract.upgrade/`
- route bound to current branch

### usecase 2: traverse the route via route.drive

**contract:**
- route.drive reads stones and guards in sequence
- each stone emits an artifact
- each guard validates passage

## domain objects

### entity: Route

```
Route
├── slug: string         // e.g., "v2026_03_13.declapract.upgrade"
├── path: string         // e.g., ".route/v2026_03_13.declapract.upgrade"
├── stones: Stone[]      // ordered list of stones
└── guards: Guard[]      // guards keyed by stone they follow
```

**unique key:** slug
**primary key:** path

### literal: Stone

```
Stone
├── name: string         // e.g., "1.upgrade.invoke"
├── path: string         // e.g., "1.upgrade.invoke.stone"
├── content: string      // the prompt/instructions for the driver
└── artifact: string     // e.g., "1.upgrade.invoke.v1.i1.md"
```

### literal: Guard

```
Guard
├── name: string         // e.g., "2.detect.hazards"
├── path: string         // e.g., "2.detect.hazards.guard"
├── selfReviews: string[]// list of self-review questions
└── judges: string[]     // list of judge commands
```

### literal: RouteBind

```
RouteBind
├── branch: string       // e.g., "vlad/declapract-repair"
├── boundBy: string      // e.g., "route.bind skill"
└── path: string         // e.g., ".route/.bind.vlad.declapract-repair.flag"
```

## domain operations

### getOneIsoDate

```typescript
getOneIsoDate(): string
// returns current date as YYYY_MM_DD (e.g., "2026_03_13")
```

### getOneRouteSlug

```typescript
getOneRouteSlug(input: { prefix: string }): string
// returns "v{isodate}.{prefix}" (e.g., "v2026_03_13.declapract.upgrade")
```

### getOneRoutePath

```typescript
getOneRoutePath(input: { slug: string }): string
// returns ".route/{slug}" (e.g., ".route/v2026_03_13.declapract.upgrade")
```

### setRoute (findsert)

```typescript
setRoute(input: {
  slug: string;
  stones: Stone[];
  guards: Guard[]
}): Route
// creates route directory and writes all stone/guard files
// findsert semantics: if route exists, returns extant
```

### setRouteBind

```typescript
setRouteBind(input: {
  routePath: string;
  branch: string
}): RouteBind
// invokes `npx rhachet run --repo bhrain --skill route.bind`
// creates .route/.bind.{branchslug}.flag
```

## relationships

### route contains stones and guards

```
Route
├── contains → Stone[] (ordered)
└── contains → Guard[] (keyed by stone name)
```

### stone precedes guard

```
Stone (1.upgrade.invoke)      → no guard
Stone (2.detect.hazards)      → Guard (2.detect.hazards)
Stone (3.1.repair.test)       → Guard (3.1.repair.test)
Stone (3.2.reflect.test)      → Guard (3.2.reflect.test)
Stone (3.3.repair.cicd)       → Guard (3.3.repair.cicd)
Stone (3.4.reflect.cicd)      → Guard (3.4.reflect.cicd)
```

### route binds to branch

```
Route ← binds → Branch (via RouteBind)
```

## composition to support wish

### init.declapract.upgrade skill flow

```
1. check prerequisites
   ├── declapract.use.yml exists? → fail fast if not
   └── pnpm available? → fail fast if not

2. compute route metadata
   ├── getOneIsoDate() → "2026_03_13"
   ├── getOneRouteSlug({ prefix: "declapract.upgrade" }) → "v2026_03_13.declapract.upgrade"
   └── getOneRoutePath({ slug }) → ".route/v2026_03_13.declapract.upgrade"

3. plan mode: preview
   └── show route path and stones that would be created

4. apply mode: instantiate
   ├── setRoute({ slug, stones, guards }) → creates files
   └── setRouteBind({ routePath, branch }) → binds to branch

5. confirm
   └── output turtle vibes confirmation
```

### stone content generation

stones are static content defined in the skill:
- 1.upgrade.invoke.stone — instructions to run declapract upgrade
- 2.detect.hazards.stone — instructions to scan diffs
- 3.1.repair.test.defects.stone — instructions to fix tests
- 3.2.reflect.test.defects.stone — instructions to classify test defects
- 3.3.repair.cicd.defects.stone — instructions to fix cicd
- 3.4.reflect.cicd.defects.stone — instructions to classify cicd defects

### guard content generation

guards are static content defined in the skill:
- 2.detect.hazards.guard — self-reviews for hazard completeness
- 3.1.repair.test.defects.guard — self-review for zero test failures
- 3.2.reflect.test.defects.guard — self-review + human approval
- 3.3.repair.cicd.defects.guard — self-review for zero cicd failures
- 3.4.reflect.cicd.defects.guard — self-review + human approval
