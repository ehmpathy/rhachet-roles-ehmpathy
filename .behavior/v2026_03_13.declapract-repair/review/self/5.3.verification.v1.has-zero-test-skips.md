## silent credential bypass search

### grep for credential bypass patterns

**command:**

**result:** no matches

### analysis of SKIP_ROUTE_BIND

the test file uses `SKIP_ROUTE_BIND=1` env var:

```typescript
// line 49
SKIP_ROUTE_BIND: '1', // skip route.bind.set (no rhachet in temp dir)
```

**is this a credential bypass?** no.

**why not:**
1. this skips calling `npx rhachet run --repo bhrain --skill route.bind.set` — an external dependency not installed in temp test directories
2. the core behavior is still tested: route directory creation, file copy, chmod 555
3. this is test isolation, not test skip — the test still verifies all testable behavior
4. the bind itself is a side effect (writes to git config) that cannot be tested in isolation

**what would be a credential bypass:**
```typescript
// this would be bad
if (!process.env.GITHUB_TOKEN) return; // skip test entirely
```

the test does NOT do this. it runs all 4 test cases.

## lessons

### lesson 1: distinguish test isolation from test skips

`SKIP_ROUTE_BIND=1` is test isolation — it mocks an external dependency.
`.skip()` is a test skip — it silently ignores a test.

the former is acceptable. the latter is forbidden.

### lesson 2: search for all skip patterns

jest has multiple patterns: `.skip()`, `.only()`, `xit`, `xdescribe`, `xtest`. search for all.

### lesson 3: credential bypass is a form of skip

```typescript
if (!process.env.TOKEN) return; // this is a skip
```

this pattern silently exits the test. it is as bad as `.skip()`.
