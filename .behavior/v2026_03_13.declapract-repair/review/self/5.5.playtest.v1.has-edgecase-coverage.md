## edge cases NOT covered (and why)

### not covered: route.bind.set failure
**reason:** route.bind.set is an external skill from bhrain repo. test of its failure modes is out of scope for this skill's playtest. the skill does invoke it, but its success is assumed.

### not covered: fnm not available
**reason:** fnm is only used inside 1.upgrade.invoke.sh (a template). the playtest verifies the skill (init.declapract.upgrade.sh), not the template execution. fnm availability is checked when the driver runs the stone, not at skill invocation time.

### not covered: git repo without commits
**reason:** this is a rare edge case. the skill checks for declapract.use.yml and pnpm, not git state. however, route.bind.set may fail if git has no commits. this is acceptable out-of-scope.

### not covered: permission denied on .route/ creation
**reason:** filesystem permission errors are system-level failures. the skill uses standard mkdir -p which will fail loudly if permissions are denied. no special handler needed.

## lessons

### lesson 1: categorize edge cases systematically

organized edge cases into categories (input validation, idempotency, permissions). this ensures comprehensive coverage.

### lesson 2: document acceptable omissions

edge cases not covered are documented with reasons. this prevents future reviewers from a flag of them as gaps.

### lesson 3: skill scope vs route scope

the playtest scopes to skill behavior (init.declapract.upgrade). edge cases for route execution (fnm, git state) are out of scope.
