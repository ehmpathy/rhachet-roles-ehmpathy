## deeper scrutiny: what could be absent?

### question 1: should there be a --force flag to overwrite extant route?

**analysis**: findsert semantics mean same-day invocations reuse the route. if user wants a fresh route same-day, they would need to delete the directory manually.

**is this a gap?**: no — findsert is the requested behavior per wish. force flag would violate findsert semantics.

**verdict**: **not a gap**.

---

### question 2: should there be a --branch flag to bind to different branch?

**analysis**: route.bind.set binds to current branch automatically. to bind to a different branch would confuse the user — user should checkout the branch first.

**verdict**: **not a gap**.

---

### question 3: should stone templates have version numbers?

**analysis**: stones use `.stone.tpl` extension. artifacts use `.v1.i1.md`. the templates themselves don't need version numbers — they're templates, not versioned artifacts.

**verdict**: **not a gap**.

---

### question 4: should there be cleanup instructions if route fails mid-way?

**analysis**: findsert semantics mean partial route can be reused. user can re-run apply mode to continue. no explicit cleanup needed.

**verdict**: **not a gap**.

---

### question 5: is validation of declapract.use.yml content needed?

**analysis**: blueprint only checks file existence, not content validity. declapract apply will fail if content is invalid — that's fail-fast behavior.

**verdict**: **not a gap** — rely on declapract apply to validate.
