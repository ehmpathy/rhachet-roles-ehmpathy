## lessons

### lesson 1: else in display code is acceptable

the rule.forbid.else-branches targets control flow hazards (hidden logic paths). display conditionals like `├─` vs `└─` are presentational and exempt. the hazard the rule prevents is "hidden logic that changes behavior" — display code has no such hazard.

### lesson 2: SCRIPT_DIR is exempt from forbid.term rules

`SCRIPT_DIR` is a bash idiom (`dirname "${BASH_SOURCE[0]}"`) for locator the directory of the current file. this is not the term the rule forbids — that rule targets documentation and naming that uses vague placeholders.

### lesson 3: .what/.why can be split across header lines

the header format supports multi-line `.why`:
```
# .what = one line
#
# .why  = first line
#         continuation
```
this is standard and compliant.
