#### question 1: the command sequence has `fnm use` in the wrong place

**the discovery:**
the wish says: `npm install ... && npx declapract apply && npx declapract apply && fnm use && npm install && npm fix`

i noticed `fnm use` comes after the first install. but if the repo's `.nvmrc` requires a different node version, the first install fails before `fnm use` runs.

**can this be answered via logic?**
yes. the sequence is: install needs node, fnm sets node, therefore fnm must come first.

**can this be answered via extant docs?**
no docs needed. pure logic.

**should this go to research?**
no. the answer is self-evident.

**must i ask the wisher?**
ideally yes, to confirm the reorder. but the logic is clear: fnm first.

**what i did:**
added to vision under "questions answered via logic" with the corrected sequence.

#### question 2: the route assumes github actions but does not say so

**the discovery:**
the vision mentions `git release --watch`. this command works with github actions. repos with circleci or jenkins would fail.

**can this be answered via logic?**
yes. if the route uses github-specific commands, it should say so.

**can this be answered via extant docs?**

**should this go to research?**
no. the answer is: make the scope explicit.

**must i ask the wisher?**
no. the wisher uses github actions. the route is for ehmpathy repos. the scope is implicit but should be explicit.

**what i did:**
added to vision under "questions answered via logic" with explicit scope note.
