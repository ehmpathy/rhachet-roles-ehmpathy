### assumption 2: the upgrade command sequence is correct

**what do we assume here without evidence?**
we assume `pnpm install declapract@latest declapract-typescript-ehmpathy@latest --save-dev && npx declapract apply && npx declapract apply && fnm use && pnpm install && pnpm fix` is the right sequence.

**what evidence supports this assumption?**
the wish explicitly states this sequence: "it should effectively do this; npm install declapract@latest declapract-typescript-ehmpathy@latest --save-dev && npx declapract apply && npx declapract apply && fnm use && npm install && npm fix" (with pnpm substituted per wisher directive).

**what if the opposite were true?**
different order could cause issues. e.g., `fnm use` before `pnpm install` might be needed if the repo's node version differs from current shell.

**did the wisher actually say this, or did we infer it?**
wisher specified the sequence. but we should question the order.

**what exceptions or counterexamples exist?**
repos with `.nvmrc` or `.node-version` that need node switch first. repos where `pnpm fix` does not exist.

**verdict:** sequence comes from wisher, but `fnm use` placement should be reconsidered. the current skill runs `fnm use` after install — this may fail if node version mismatch prevents install. flag for validation.

### assumption 6: zero test failures is achievable

**what do we assume here without evidence?**
the wish says "it must take initiative and fix all defects, even if they existed before this pr". we assume the mechanic can fix all failures.

**what evidence supports this assumption?**
mechanics are capable. but some failures may be outside their control (e.g., external service down, flaky third-party api).

**what exceptions or counterexamples exist?**

**verdict:** the requirement is strict but intentional. if truly unfixable, the mechanic must document why and seek human guidance. the vision should note: "if a defect is truly unfixable, document the blocker and seek human guidance".
