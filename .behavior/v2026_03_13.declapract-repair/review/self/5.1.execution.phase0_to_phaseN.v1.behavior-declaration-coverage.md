## gaps found and fixed

### gap 1: blueprint prescribed $ROUTE substitution that was not needed

**blueprint codepath tree (line 55-56):**
```
│   ├── [+] copy templates/* → route directory
│   │   ├── [+] substitute $ROUTE → actual route path
│   │   └── [+] chmod 555 for *.sh files
```

**implementation (init.declapract.upgrade.sh line 109):**
```bash
cp -f "$TEMPLATES_DIR"/* "$ROUTE_PATH/"
```

**investigation:**
```sh
grep -r '\$ROUTE\|\${ROUTE}' templates/
# result: no $ROUTE references found
```

**discovery:** the blueprint prescribed substitution but the templates lack `$ROUTE` references. the guards use `$stone` and `$route` variables, but these are runtime variables that route.drive evaluates, not build-time substitutions.

**verdict:** not a gap — the blueprint's prescription was more conservative than needed. the implementation correctly skips substitution because no substitution targets exist.

**lesson:** blueprints may prescribe operations that turn out unnecessary once templates are authored. this is acceptable when the omission is deliberate and documented.

### gap 2: verified guards contain expected runtime variables

**3.2.reflect.test.defects.guard (line 15-16):**
```yaml
judges:
  - npx rhachet run --repo bhrain --skill route.stone.judge --mechanism approved? --stone $stone --route $route
```

**question:** are `$stone` and `$route` supposed to be substituted by init skill?

**answer:** no. these are runtime variables that route.drive substitutes when it reads the guard. the init skill correctly leaves them as template variables.

**evidence:** this matches how guards work in other routes (e.g., behavior routes in bhuild).
