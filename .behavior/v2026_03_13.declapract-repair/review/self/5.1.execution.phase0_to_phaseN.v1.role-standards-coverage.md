## lessons

### lesson 1: shell skills have different standards

shell skills don't use:
- (input, context) pattern — use arguments and env vars
- HelpfulError — use exit codes and stderr
- domain objects — use strings and files

shell skills must use:
- `set -euo pipefail`
- `exit 2` for prerequisite failures
- clear error messages
- turtle vibes vocabulary

### lesson 2: test coverage must include failure modes

not just happy path. every prerequisite check must have a test that verifies:
- the check triggers correctly
- the exit code is correct
- the error message is clear

### lesson 3: idempotency is implicit in shell idioms

- `mkdir -p` = create if absent
- `cp -f` = overwrite if present
- `|| true` = ignore failure

these idioms provide findsert semantics without explicit logic.
