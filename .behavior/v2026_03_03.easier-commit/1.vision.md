# vision: easier commit

## the problem

**the skill works. the permissions hook blocks it.**

clones try to commit:
```sh
echo "fix(api): validate input

- added schema check" | rhx git.commit.set -m @stdin
```

the permissions nudge hook blocks this even though `echo:*` is allowlisted.

clones then:
1. get confused by the block message
2. try workarounds (printf, variable assignment)
3. lose time
4. document workarounds as "the way"

---

## the outcome world

### before/after contrast

**before**: mechanic reads skill header, unclear on exact pattern
```sh
# usage:
#   echo "fix(api): validate input\n\n- added input schema" | git.commit.set -m @stdin
```
- uses `\n` escape (shell-specific behavior)
- not clear if `echo` or `printf` preferred
- flags scattered in different places

**after**: mechanic reads skill header, knows exactly what to do
```sh
# usage:
#   echo "fix(scope): summary
#
#   - detail 1
#   - detail 2" | git.commit.set -m @stdin                    # plan (preview)
#   echo "..." | git.commit.set -m @stdin --mode apply        # apply (commit)
#   echo "..." | git.commit.set -m @stdin --mode apply --push # apply + push
```
- real newlines (works in any shell)
- clear examples for plan vs apply
- all common flags visible

### the "aha" moment

> "oh, i just echo my message and pipe it. plan first, then apply."

## user experience

### usecases

| goal | command |
|------|---------|
| preview commit | `echo "msg" \| rhx git.commit.set -m @stdin` |
| apply commit | `echo "msg" \| rhx git.commit.set -m @stdin --mode apply` |
| apply + push | `echo "msg" \| rhx git.commit.set -m @stdin --mode apply --push` |
| include untracked | add `--unstaged include` |
| ignore untracked | add `--unstaged ignore` |

### message format

```
fix(scope): one-line summary

- bullet point 1
- bullet point 2
```

first line = header (required)
blank line + bullets = body (required)

### what we ARE to fix

1. **fix permissions nudge hook** (`pretooluse.check-permissions.sh`):

   **the bug**: composite commands are blocked even though `echo:*` is allowlisted

   **test cases to add** (from wish — should fail before fix, pass after):
   ```sh
   # FAILS today: echo with literal string
   echo "fix: msg" | rhx git.commit.set -m @stdin

   # FAILS today: variable assignment + echo
   MESSAGE="fix: msg" && echo "$MESSAGE" | rhx git.commit.set -m @stdin

   # FAILS today: npx variant
   echo "fix: msg" | npx rhachet run --skill git.commit.set -m @stdin
   ```

   **works today** (workaround):
   ```sh
   # WORKS: printf is prefix-approved
   printf "fix: msg" | rhx git.commit.set -m @stdin
   ```

   **expected after fix**: all echo variants exit 0 (allowed)
   **actual today**: echo variants exit 2 (blocked)

2. **update skill header** in `git.commit.set.sh` — clearer usage examples with real newlines
