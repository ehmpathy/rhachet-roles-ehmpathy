# playtest: git.stage.add + unlimited quota

## prerequisites

- repo with uncommitted changes (or create test files)
- no prior quota file (run `git.commit.uses del` to clear if needed)

---

## happy paths

### 1. grant stage permission via set

```sh
npx rhachet run --skill git.commit.uses set --quant 3 --push allow --stage allow
```

**expected outcome:**
- output shows `uses: 3`, `push: allowed`, `stage: allowed`
- file `.meter/git.commit.uses.jsonc` created with `"stage": "allow"`

**verify:**
```sh
cat .meter/git.commit.uses.jsonc
```

---

### 2. grant unlimited quota via allow

```sh
npx rhachet run --skill git.commit.uses allow
```

**expected outcome:**
- output shows `uses: unlimited`, `push: allowed`, `stage: allowed`
- state file shows `"uses": "infinite"` (string, not number)

**verify:**
```sh
cat .meter/git.commit.uses.jsonc | grep -E '"uses"|"push"|"stage"'
```

---

### 3. check quota via get

```sh
npx rhachet run --skill git.commit.uses get
```

**expected outcome:**
- output shows all three permissions: uses, push, stage
- stage permission visible in output

---

### 4. stage specific files

create test files first:
```sh
echo "test content" > playtest-file1.ts
echo "test content" > playtest-file2.ts
```

grant permission and stage:
```sh
npx rhachet run --skill git.commit.uses allow
npx rhachet run --skill git.stage.add playtest-file1.ts playtest-file2.ts
```

**expected outcome:**
- output shows tree-formatted list of staged files
- both files appear in staged area

**verify:**
```sh
git diff --cached --name-only | grep playtest
```

**cleanup:**
```sh
git reset HEAD playtest-file1.ts playtest-file2.ts
rm playtest-file1.ts playtest-file2.ts
```

---

### 5. stage all with dot

create test files:
```sh
mkdir -p playtest-dir
echo "content" > playtest-dir/file1.ts
echo "content" > playtest-dir/file2.ts
echo "content" > playtest-root.ts
```

stage all with `.`:
```sh
npx rhachet run --skill git.commit.uses allow
npx rhachet run --skill git.stage.add .
```

**expected outcome:**
- all files staged (playtest-dir/file1.ts, playtest-dir/file2.ts, playtest-root.ts)
- output shows `staged: .`

**verify:**
```sh
git diff --cached --name-only | grep playtest
```

**cleanup:**
```sh
git reset HEAD playtest-dir/ playtest-root.ts
rm -rf playtest-dir playtest-root.ts
```

---

### 6. revoke permissions via del

```sh
npx rhachet run --skill git.commit.uses del
```

**expected outcome:**
- output confirms revocation
- `.meter/git.commit.uses.jsonc` file removed

**verify:**
```sh
ls .meter/git.commit.uses.jsonc 2>&1 | head -1
```

should show "No such file" or similar.

---

### 7. unlimited quota does not decrement on commit

```sh
# grant unlimited
npx rhachet run --skill git.commit.uses allow

# check uses is infinite
npx rhachet run --skill git.commit.uses get

# make a commit (in plan mode to avoid actual commit)
echo "test" > playtest-commit.ts
git add playtest-commit.ts
printf 'test(playtest): verify unlimited\n\n- test commit' | npx rhachet run --skill git.commit.set -m @stdin --mode apply

# check uses is still infinite
npx rhachet run --skill git.commit.uses get
```

**expected outcome:**
- before commit: `uses: unlimited`
- after commit: `uses: unlimited` (not decremented)

**cleanup:**
```sh
git reset --soft HEAD~1
git reset HEAD playtest-commit.ts
rm playtest-commit.ts
npx rhachet run --skill git.commit.uses del
```

## edgey paths

### 1. stage without permission

```sh
npx rhachet run --skill git.commit.uses set --quant 3 --push allow --stage block
echo "test" > playtest-noperm.ts
npx rhachet run --skill git.stage.add playtest-noperm.ts
```

**expected behavior:**
- error message: "stage not allowed"
- file not staged

**cleanup:**
```sh
rm playtest-noperm.ts
npx rhachet run --skill git.commit.uses del
```

---

### 2. stage file that does not exist

```sh
npx rhachet run --skill git.commit.uses allow
npx rhachet run --skill git.stage.add nonexistent-file.ts
```

**expected behavior:**
- error message: "file not found: nonexistent-file.ts"

**cleanup:**
```sh
npx rhachet run --skill git.commit.uses del
```

---

### 3. path outside repo blocked

```sh
npx rhachet run --skill git.commit.uses allow
npx rhachet run --skill git.stage.add /etc/passwd
```

**expected behavior:**
- error message: "path must be within repo"

**cleanup:**
```sh
npx rhachet run --skill git.commit.uses del
```

---

### 4. del with no quota file (idempotent)

```sh
npx rhachet run --skill git.commit.uses del
npx rhachet run --skill git.commit.uses del
```

**expected behavior:**
- both commands succeed
- no error on second del

---

### 5. stage already staged file (idempotent)

```sh
npx rhachet run --skill git.commit.uses allow
echo "test" > playtest-idem.ts
npx rhachet run --skill git.stage.add playtest-idem.ts
npx rhachet run --skill git.stage.add playtest-idem.ts
```

**expected behavior:**
- both stage commands succeed
- second stage is no-op

**cleanup:**
```sh
git reset HEAD playtest-idem.ts
rm playtest-idem.ts
npx rhachet run --skill git.commit.uses del
```

---

### 6. --stage defaults to block

```sh
npx rhachet run --skill git.commit.uses set --quant 3 --push allow
cat .meter/git.commit.uses.jsonc | grep stage
```

**expected behavior:**
- state shows `"stage": "block"` (default when not specified)

**cleanup:**
```sh
npx rhachet run --skill git.commit.uses del
```

---

### 7. global blocker overrides local permission

```sh
# set local permission
npx rhachet run --skill git.commit.uses allow

# set global blocker
npx rhachet run --skill git.commit.uses block --global

# try to stage
echo "test" > playtest-global.ts
npx rhachet run --skill git.stage.add playtest-global.ts
```

**expected behavior:**
- error message indicates global block

**cleanup:**
```sh
rm playtest-global.ts
npx rhachet run --skill git.commit.uses allow --global
npx rhachet run --skill git.commit.uses del
```

---

## pass/fail criteria

- ✓ pass if: all happy paths produce expected outcomes
- ✓ pass if: all edgey paths produce expected error messages
- ✓ pass if: state file shows correct field values
- ✓ pass if: unlimited quota does not decrement after commit
- ✓ pass if: stage permission is independent of push permission
- ✗ fail if: any happy path produces error
- ✗ fail if: any edgey path fails silently instead of explicit error
- ✗ fail if: unlimited quota decrements after commit
- ✗ fail if: stage permission defaults to allow (should default to block)
