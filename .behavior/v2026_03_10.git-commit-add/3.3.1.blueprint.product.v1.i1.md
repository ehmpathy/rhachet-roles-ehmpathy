# blueprint: product

## summary

implement git.stage.add skill with unlimited quota support:
1. extend git.commit.uses.local.sh:
   - add `--stage allow|block` option (parallels `--push`)
   - change `allow` command to set `uses: "infinite"` (not `999999`)
2. extend git.commit.set.sh to skip decrement when uses === "infinite"
3. create git.stage.add.sh skill for file stage operations

note: architecture after rebase:
- `git.commit.uses.sh` is now a dispatcher (routes to local.sh or global.sh via --global flag)
- `git.commit.uses.local.sh` handles per-repo quota (set/get/del/block/allow)
- `git.commit.uses.global.sh` handles global blocker (block/allow/get)
- `allow` command sets uses=infinite (explicit unlimited, not magic number)

## test coverage

### git.commit.uses.integration.test.ts [~]

| case | test |
|------|------|
| --stage allow | set --stage allow → state shows `"stage": "allow"` |
| --stage block | set --stage block → state shows `"stage": "block"` |
| --stage default | set without --stage → state shows `"stage": "block"` |
| allow sets stage | allow → state shows `"stage": "allow"` |
| del resets stage | del → state shows `"stage": "block"` (default) |
| block resets stage | block → state shows `"stage": "block"` (default) |
| get with stage | get → displays stage permission alongside uses and push |
| output on set | set with --stage → output shows uses, push, stage |

### git.commit.set.integration.test.ts [~]

| case | test |
|------|------|
| unlimited no decrement | commit with uses=infinite → uses stays infinite |

### git.stage.add.integration.test.ts [+]

| case | test |
|------|------|
| stage files | stage src/file1.ts src/file2.ts → both staged |
| no permission | stage without --stage allow → "error: stage not allowed" |
| file not found | stage absent.ts → "error: file not found: absent.ts" |
| bounded glob | --glob 'src/**/*.ts' → files staged |
| unbound glob | --glob '**/*.ts' → "error: unbound glob is forbidden" |
| already staged | stage staged file → no-op |
| path outside repo | stage /etc/passwd → "error: path must be within repo" |
| no files | stage with no args → "error: no files specified" |
| glob no match | --glob '*.xyz' → "error: no files matched glob: *.xyz" |
| output format | stage → tree format output |
| atomic on error | file2 absent → file1 not staged |
| --all blocked | --all flag → "error: --all is forbidden" |
| paths + glob | paths with --glob → "error: cannot combine paths and --glob" |
| global blocked | stage with global blocker → "error: globally blocked" |
