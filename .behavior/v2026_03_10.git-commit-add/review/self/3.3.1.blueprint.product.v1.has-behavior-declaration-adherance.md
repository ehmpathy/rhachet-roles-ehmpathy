**question:** could a junior confuse `--stage allow` with `--push allow`?

**vision says:** these are independent flags. `--stage` controls git.stage.add, `--push` controls git.commit.push.

**blueprint says:** write_state has separate fields for push and stage.

**test coverage:**
- `--stage allow` test sets only stage
- `--stage default` test verifies stage defaults independently

**verdict:** no misinterpretation. blueprint correctly separates the permissions.

### misinterpretation 2: infinite applies to uses only

**question:** could a junior interpret infinite as applying to all permissions?

**vision says:** "infinite" is for `--quant` only. it means unlimited commits.

**blueprint says:**

**test coverage:**

**verdict:** no misinterpretation. blueprint correctly limits infinite to uses field.
