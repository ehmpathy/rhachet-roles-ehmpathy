**triage:**

**answer:** **forbid `--all`**. allow explicit paths and bounded globs only. this follows the established sedreplace pattern of block unbounded operations.

the vision document should be updated to remove `--all` from the proposed contract.

**status:** resolved via code research (sedreplace precedent)

### q10: does `git.stage.add` decrement `uses`?

**triage:**
- can answer via logic and wish text? yes

**context from wish:**
> "let a clone push and stage as many times as they want"

this phrase appears in the context of infinite. but what about when quant is a number like 5?

**rationale:**
- stage is preparatory — you might stage many times before one commit
- commit is the action that changes history
- push is the action that shares with the world
- stage just moves files to the index — reversible, no lasting effect
- decrementing uses for stage would be surprising and limiting
- the wish says "stage as many times as they want" — suggests unlimited

**comparison:**
| operation | effect | should count? |
|-----------|--------|---------------|
| commit | creates history | yes — uses counts commits |
| push | shares with world | no — controlled by --push allow|block |
| stage | prepares index | no — controlled by --stage allow|block |

**answer:** `git.stage.add` does NOT decrement `uses`. it is controlled purely by `--stage allow|block`. the quota tracks commits, not preparatory operations.

**status:** resolved via logic
