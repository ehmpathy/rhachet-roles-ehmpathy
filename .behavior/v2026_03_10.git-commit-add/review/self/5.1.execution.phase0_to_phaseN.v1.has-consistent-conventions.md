# consistent conventions review: execution phase

## review question

for each new artifact in the code, ask:
- does it follow extant naming conventions?
- does it follow extant output format conventions?
- does it follow extant error message conventions?

## conventions reviewed

### 1. skill naming convention

**extant pattern:**
```
git.[namespace]/
  git.[namespace].[verb].sh
```

**examples:**
- git.commit/git.commit.set.sh
- git.commit/git.commit.push.sh
- git.rebase/git.rebase.begin.sh
- git.rebase/git.rebase.continue.sh

**git.stage.add.sh:**
```
git.stage/
  git.stage.add.sh
```

**status**: follows extant pattern

**why it holds**: namespace is `git.stage`, verb is `add`. follows same structure as git.commit and git.rebase.

### 2b. turtle header phrase convention

| phrase | meaning | used in |
|--------|---------|---------|
| "bummer dude..." | error/failure | git.commit.set, git.commit.push, git.stage.add |

**status**: follows majority pattern

**note on extant inconsistency**: git.commit.set.sh line 296 uses "hold up, dude..." (with comma), while all other skills use "hold up dude..." (without comma). git.stage.add follows the majority pattern (no comma). this pre-extant inconsistency is out of scope for this task.

### 5. permission check convention

**extant pattern in git.commit.set.sh:**
```bash
# check global blocker first
if [[ -f "$GLOBAL_METER_FILE" ]]; then
  GLOBAL_USES=$(jq -r '.uses // 0' "$GLOBAL_METER_FILE")
  if [[ "$GLOBAL_USES" == "0" ]]; then
    print_tree_error "git.commit globally blocked"
    exit 2
  fi
fi

# check local permission
if [[ "$PUSH_ALLOWED" != "allow" && "$PUSH" == "yes" ]]; then
  print_tree_error "push not allowed"
  exit 2
fi
```

**git.stage.add.sh:**
```bash
# check global blocker first
if [[ -f "$GLOBAL_METER_FILE" ]]; then
  GLOBAL_USES=$(jq -r '.uses // 0' "$GLOBAL_METER_FILE")
  if [[ "$GLOBAL_USES" == "0" ]]; then
    print_tree_error "globally blocked"
    exit 2
  fi
fi

# check local permission
STAGE_ALLOWED=$(jq -r '.stage // "block"' "$STATE_FILE")
if [[ "$STAGE_ALLOWED" != "allow" ]]; then
  print_tree_error "stage not allowed"
  exit 2
fi
```

**status**: follows extant permission check pattern

**why it holds**: same order (global first, local second), same jq pattern, same exit code.
