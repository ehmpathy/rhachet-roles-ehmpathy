## deeper analysis: any term misalignment?

paused here. let me check if I've rubber-stamped or genuinely reviewed.

**question 1:** should it be `git.stage.add` or `git.add.stage`?

**analysis:**
- extant: `git.commit.set` = `{namespace}.{verb}`
- `git.stage.add` = `git.{namespace}.{verb}` where namespace=stage, verb=add
- `git.add.stage` would be `git.{verb}.{noun}` which breaks pattern

**verdict:** `git.stage.add` is correct

**question 2:** should command be `add` or `stage`?

**analysis:**
- wish says "git.stage.add skill"
- `add` is the git verb (`git add`)
- `stage` is already in namespace
- `git.stage.stage` would be redundant

**verdict:** `add` is correct

**question 3:** should `del` be `delete` or `rm`?

**analysis:**
- `del` is terse (3 chars) like `set`, `get`
- `delete` is verbose (6 chars)
- `rm` is bash convention, not domain convention
- dao pattern uses `del`

**verdict:** `del` is correct

**question 4:** is `infinite` the best term for unlimited?

**analysis:**
- `infinite` is explicit string, not magic number
- clear intent: "infinite" means unlimited
- no ambiguity with numeric values
- alternatives have issues:
  - `999999` is magic number, unclear intent
  - `-1` confuses with error codes
  - `*` conflicts with glob patterns

**verdict:** `infinite` is correct. wisher explicitly confirmed this term.
