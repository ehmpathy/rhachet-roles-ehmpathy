### pre-extant failure found and fixed

one acceptance test failed locally but passed on CI:

**file**: `blackbox/guardBorder.onWebfetch.acceptance.test.ts`
**case**: `[case7] XAI_API_KEY not configured`

**symptoms**:
- test set `XAI_API_KEY: ''` in env
- expected hook to fail with "ask the human" error
- locally: hook succeeded (wrong)
- CI: hook failed as expected (correct)

**root cause investigation**:
1. hook sources `~/.config/rhachet/apikeys.env` at startup
2. this file exists on local machine with real API key
3. CI has no such file — test env's empty value is effective
4. locally, the sourced file overrides the empty value from test env

**fix applied** (line 307):
```typescript
env: { XAI_API_KEY: '', HOME: tempDir },
```

setting `HOME: tempDir` prevents hook from finding `~/.config/rhachet/apikeys.env`. the test now passes both locally and on CI.

**why it holds**:
- root cause was environmental divergence between local and CI
- fix addresses the divergence by isolating home directory
- fix is minimal and targeted (one additional env var)
- failure was not carried forward — it is now fixed
