# research: internal prod codepath patterns

## scope

this research enumerates prod codepath patterns in this repo relevant to:
1. brief structure for work.flow practices
2. hook registration and execution

## findings

### pattern.1: brief structure in work.flow

**source:** `src/domain.roles/mechanic/briefs/practices/work.flow/diagnose/rule.require.test-covered-repairs.md` [1]

**structure:**
```markdown
# rule.require.{topic}

## .what
{one sentence description}

## .why
{rationale with bullet points}

## .the rule
{table with scenario → required}

## .pattern
{code example}

## .antipattern
{code example with bad vs good}

## .benefits
{table with without vs with}

## .enforcement
{blocker conditions}
```

**key elements:**
- header uses `rule.require.` prefix for mandatory rules
- sections use `.what`, `.why` format (not markdown headers for these)
- tables for scenario mapping
- code blocks with `# bad:` and `# good:` comments
- ends with enforcement conditions

**action:** [REUSE] — use this structure for `rule.require.trust-but-verify.md`

### pattern.2: hook types in role definition

**source:** `src/domain.roles/mechanic/getMechanicRole.ts` [2]

**supported hook types:**
```typescript
hooks: {
  onBrain: {
    onBoot: [...],   // SessionStart hooks
    onTool: [...],   // PreToolUse/PostToolUse hooks
    onStop: [...],   // Stop hooks
  },
}
```

**hook structure:**
```typescript
{
  command: './node_modules/.bin/rhachet run --repo ehmpathy --role mechanic --init claude.hooks/{hookname}',
  timeout: 'PT5S',
  filter?: { what: 'Bash|Write|Edit', when: 'before' | 'after' },
}
```

**observation:** no `onCompact` type in rhachet Role interface. PostCompact hook may need direct `.claude/settings.json` registration.

**action:** [EXTEND] — if PostCompact not in rhachet, register directly in settings.json

### pattern.3: hook registration in settings.json

**source:** `.claude/settings.json` [3]

**Claude Code hook types observed:**
- `SessionStart` — maps to `onBrain.onBoot`
- `PreToolUse` — maps to `onBrain.onTool` with `filter.when: 'before'`
- `Stop` — maps to `onBrain.onStop`

**settings.json hook structure:**
```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "./node_modules/.bin/rhachet run ...",
            "timeout": 5,
            "author": "repo=ehmpathy/role=mechanic"
          }
        ]
      }
    ]
  }
}
```

**observation:** PostCompact is not present in current settings. would be a new key:
```json
{
  "hooks": {
    "PostCompact": [...]
  }
}
```

**action:** [EXTEND] — add PostCompact section if rhachet gains support, or register manually

### pattern.4: hook file structure

**source:** `src/domain.roles/mechanic/inits/claude.hooks/sessionstart.notify-permissions.sh` [4]

**structure:**
```bash
#!/usr/bin/env bash
######################################################################
# .what = {description}
#
# .why  = {rationale}
#
# usage:
#   {invocation}
#
# guarantee:
#   - {guarantee 1}
#   - {guarantee 2}
######################################################################
set -euo pipefail

# procedure body...

exit 0
```

**output pattern:** hooks emit to stdout, Claude receives as system-reminder

**action:** [REUSE] — use this structure for PostCompact hook

### pattern.5: brief location in work.flow

**source:** glob of `src/domain.roles/mechanic/briefs/practices/work.flow/**/*.md` [5]

**subdirectories observed:**
- `diagnose/` — bisect, logservation, test-covered-repairs
- `refactor/` — sedreplace, review-test-changes
- `release/` — commit-scopes, watch-release, git-release
- `tools/` — git-commit-set, terraform, externalized-knowledge, package-docs

**action:** [EXTEND] — create new subdirectory `verify/` or place in `diagnose/` (verification is pre-diagnosis)

**decision:** place in `diagnose/` — trust-but-verify is about root cause verification before action, aligns with diagnosis workflow
