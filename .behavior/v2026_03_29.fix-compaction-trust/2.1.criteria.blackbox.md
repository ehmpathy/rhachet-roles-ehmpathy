# blackbox criteria: trust but verify

## usecase.1 = brief teaches verification pattern

```
given(mechanic has the brief in context)
  when(mechanic encounters an inherited claim)
    then(brief reminds: verify claims before you act)
      sothat(mechanic pauses to verify instead of act blindly)
    then(brief provides the pattern: claim → verify → act)
      sothat(mechanic knows what steps to take)
    then(brief provides the antipattern: claim → act)
      sothat(mechanic recognizes the failure mode)
    then(brief provides the mantra: "trust but verify — don't even trust yourself")
      sothat(mechanic has a memorable phrase to recall)

given(mechanic reads the brief)
  when(brief lists claim types that may be stale)
    then(diagnoses are listed: "X is the problem")
    then(objectives are listed: "we need to do Y")
    then(state claims are listed: "file contains Z")
    then(conclusions are listed: "the fix is W")
      sothat(mechanic knows what to question)

given(mechanic reads the brief)
  when(brief lists verification methods)
    then(CI diagnosis: `rhx show.gh.test.errors`)
    then(file state: read the file)
    then(PR status: `gh pr view`)
      sothat(mechanic knows how to verify)
```

## usecase.2 = postcompact hook reminds at critical moment

```
given(mechanic is in a session)
  when(compaction occurs)
    then(mechanic sees reminder before they respond)
      sothat(the reminder is visible at the moment of need)
    then(reminder lists claim types that may be stale)
      sothat(mechanic knows what to question)
    then(reminder points to the brief)
      sothat(mechanic can read more if needed)

given(compaction has not occurred)
  when(session starts fresh)
    then(hook does not fire)
      sothat(noise is minimized for fresh sessions)

given(compaction has occurred)
  when(mechanic sees the reminder)
    then(reminder is concise, not verbose)
      sothat(it does not become ignored noise)
```

## usecase.3 = pattern applies beyond compaction

```
given(mechanic is mid-session, no compaction)
  when(mechanic recalls a conclusion from earlier in the session)
    then(brief applies: verify before you act)
      sothat(own conclusions are also questioned)

given(mechanic receives external input)
  when(input contains claims about system state)
    then(brief applies: verify before you act)
      sothat(external claims are also questioned)
```

## edgecases

```
given(claim is actually correct)
  when(mechanic verifies it)
    then(verification confirms quickly)
    then(minimal time lost)
      sothat(verification overhead is low for correct claims)

given(verification is expensive)
  when(mechanic needs to verify)
    then(brief suggests cheap checks first)
      sothat(mechanic starts with error logs, file reads before full test suite)

given(no obvious verification method)
  when(mechanic is uncertain how to verify)
    then(brief suggests: ask human)
      sothat(mechanic has an escape hatch)
```
