            # handoff: trust but verify

            ## context

            session worked on PR #295 (boot.yml for token reduction). CI failed.

            compacted summary from prior session said "orphan processes are the issue." mechanic was about to continue that path. human interrupted and asked to verify.

            actual error: obsolete snapshots from a CI-skipped test. one file deletion fixed it.

            prior sessions had made 20+ commits in attempts to fix orphan processes — wrong problem entirely.

            ## root cause

            compacted summaries preserve conclusions. conclusions can be wrong. mechanic acted on inherited diagnosis without verification.

            ## the lesson

            > trust but verify — don't even trust yourself.

            claims drift from reality:
            - compacted summaries preserve stale conclusions
            - prior sessions may have misdiagnosed
            - world state changes between sessions
            - your own memory is a summary too

            ## what to implement

            ### 1. brief: rule.require.trust-but-verify

            location: `.agent/repo=ehmpathy/role=mechanic/briefs/practices/work.flow/`

            content:
            - verify claims before you act on them
            - applies to all inherited claims (summaries, prior sessions, your own conclusions)
            - pattern: claim → verify → act (never: claim → act)
            - mantra: "trust but verify — don't even trust yourself"
            - antipattern: the orphan processes story from this session

            ### 2. sessionstart hook (optional)

            location: `.agent/repo=ehmpathy/role=mechanic/inits/claude.hooks/`

            on resume from compaction, emit:
            ```
            ⚠️ resumed from compacted session

            the summary above contains inherited claims that may be stale or wrong:
            - diagnoses ("X is the problem")
            - objectives ("we need to do Y")
            - state ("PR is Z", "file contains W")
            - conclusions ("the fix is V")
            - etc

            before you act on inherited claims, verify them.

            remember: trust no one blindly. not even yourself. trust but verify.

            see: rule.require.trust-but-verify
            ```

            ## verification examples

            | claim | verification |
            |-------|--------------|
            | "CI fails because X" | `rhx show.gh.test.errors` |
            | "file contains Y" | read the file |
            | "PR is in state Z" | `gh pr view` |
            | "the fix is W" | confirm the problem first |

            ## done when

            - [ ] brief exists and is booted with mechanic role
            - [ ] (optional) sessionstart hook warns on compaction resume
