# blueprint: trust but verify

## summary

build two artifacts to institutionalize the "trust but verify" lesson:

1. **brief** — `rule.require.trust-but-verify.md` in work.flow/
   - teaches the verification pattern: claim → verify → act
   - lists claim types that may be stale
   - provides verification methods
   - establishes the mantra: "trust but verify — don't even trust yourself"

2. **hook** (optional) — `postcompact.trust-but-verify.sh` in claude.hooks/
   - fires after compaction via PostCompact event
   - emits a concise reminder to verify inherited claims
   - points to the brief for details

## codepath tree

### brief codepath

```
boot → briefs → work.flow/rule.require.trust-but-verify.md
  └── [+] brief content
      ├── .what = verification pattern
      ├── .why = prevent blind trust
      ├── .the rule = claim types + verification methods
      ├── .pattern = claim → verify → act
      ├── .antipattern = claim → act (orphan processes story)
      └── .enforcement = verification before action
```

### hook codepath

```
PostCompact event → postcompact.trust-but-verify.sh
  └── [+] hook execution
      ├── emit reminder to stdout
      │   ├── list claim types that may be stale
      │   └── point to brief
      └── exit 0 (allow continuation)
```

### registration codepath

```
rhachet roles init
  ├── [~] boot.yml
  │   └── always.briefs.say += rule.require.trust-but-verify.md
  └── [~] getMechanicRole.ts
      └── hooks.onBrain.onBoot += { filter.what: PostCompact, command: ...postcompact.trust-but-verify }
```

## contracts

### brief contract

```markdown
# rule.require.trust-but-verify

## .what
verify inherited claims before you act on them.

## .why
{rationale}

## .the rule
{table: claim type → verification method}

## .pattern
claim → verify → act

## .antipattern
claim → act (orphan processes story)

## .mantra
"trust but verify — don't even trust yourself"

## .enforcement
action on unverified inherited claim = blocker
```

### hook contract

```bash
#!/usr/bin/env bash
# .what = remind mechanic to verify claims after compaction
# .why = compaction summaries may contain stale conclusions
# guarantee: emits reminder to stdout, exits 0

# emit reminder
cat << 'EOF'
⚠️ compaction occurred

inherited claims may be stale:
- diagnoses ("X is the problem")
- objectives ("we need to do Y")
- state claims ("file contains Z")
- conclusions ("the fix is W")

verify before you act.

see: rule.require.trust-but-verify
EOF

exit 0
```
