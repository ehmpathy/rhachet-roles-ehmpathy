# research: internal test codepath patterns

## scope

this research enumerates test codepath patterns in this repo relevant to:
1. brief validation (boot.yml subjects and token counts)
2. hook validation (hook behavior and exit codes)

## findings

### pattern.1: hook integration test structure

**source:** `src/domain.roles/mechanic/inits/claude.hooks/pretooluse.check-permissions.integration.test.ts` [1]

**structure:**
```typescript
import { spawnSync } from 'child_process';
import * as fs from 'fs';
import { genTempDir, given, then, when } from 'test-fns';

describe('hookname.sh', () => {
  const hookPath = path.join(__dirname, 'hookname.sh');

  const runHook = (args: {
    command: string;
    settings?: { allow?: string[] };
  }): { stdout: string; stderr: string; exitCode: number; tempDir: string } => {
    const tempDir = genTempDir({ slug: 'hook-test', git: true });
    const claudeDir = path.join(tempDir, '.claude');
    fs.mkdirSync(claudeDir, { recursive: true });

    // build stdin JSON (Claude Code format)
    const stdinJson = JSON.stringify({
      tool_name: 'Bash',
      tool_input: { command: args.command },
    });

    // run the hook
    const result = spawnSync('bash', [hookPath], {
      cwd: tempDir,
      encoding: 'utf-8',
      input: stdinJson,
      stdio: ['pipe', 'pipe', 'pipe'],
    });

    return {
      stdout: result.stdout ?? '',
      stderr: result.stderr ?? '',
      exitCode: result.status ?? 1,
      tempDir,
    };
  };

  given('[case1] scenario', () => {
    when('[t0] hook runs', () => {
      then('expected outcome', () => {
        const result = runHook({ command: 'test' });
        expect(result.exitCode).toBe(0);
      });
    });
  });
});
```

**key elements:**
- use `genTempDir` with `{ git: true }` for isolated git repos
- create `.claude/` directory in temp dir for settings
- pass JSON via stdin using `spawnSync` with `input` option
- capture stdout, stderr, and exit code
- use `given/when/then` from test-fns

**action:** [REUSE] — use this pattern for PostCompact hook tests

### pattern.2: boot.yml integration test structure

**source:** `src/domain.roles/mechanic/boot.yml.integration.test.ts` [2]

**structure:**
```typescript
import { execSync } from 'child_process';
import { genTempDir } from 'test-fns';

const execBoot = (input: { args: string; cwd?: string }): string =>
  execSync(
    `npx rhachet roles boot --repo ehmpathy --role mechanic ${input.args}`,
    { encoding: 'utf-8', timeout: 30_000, cwd: input.cwd },
  );

const parseStats = (output: string): { briefs: { say: number; ref: number }; tokens: number } => {
  const sayMatch = output.match(/say = (\d+)/);
  const refMatch = output.match(/ref = (\d+)/);
  const tokensMatch = output.match(/tokens ≈ (\d+)/);
  // ...
};

describe.skip('boot.yml integration', () => {
  it('should boot with --subject code.test', () => {
    const output = execBoot({ args: '--subject code.test' });
    const stats = parseStats(output);
    expect(stats.briefs.say).toBeGreaterThan(0);
  });
});
```

**key elements:**
- use `execSync` for rhachet commands
- use `parseStats` helper to extract stats from output
- use `genTempDir` with symlinks to test without boot.yml
- tests are `describe.skip` (require CI setup)

**action:** [REUSE] — use this pattern for brief validation tests

### pattern.3: test file colocation

**source:** file structure observation [3]

**pattern:**
```
src/domain.roles/mechanic/inits/claude.hooks/
  ├─ pretooluse.check-permissions.sh
  ├─ pretooluse.check-permissions.integration.test.ts
  └─ ...
```

**colocation rule:**
- test files live next to production files
- naming: `{file}.integration.test.ts` for integration tests
- naming: `{file}.test.ts` for unit tests

**action:** [REUSE] — colocate `postcompact.trust-but-verify.integration.test.ts` with hook file

### pattern.4: hook stdin format

**source:** pretooluse.check-permissions.integration.test.ts [1]

