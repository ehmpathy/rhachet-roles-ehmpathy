# blackbox criteria: keyrack-powered border guard

## usecase.1 = mechanic uses WebFetch with keyrack unlocked

```
given keyrack is unlocked for ehmpath owner with env prep
  when mechanic runs WebFetch
    then border guard fetches XAI_API_KEY from keyrack
      sothat credential is available without hardcoded paths
    then border guard inspects content via grok
    then WebFetch proceeds if content is admissible
      sothat mechanic can research the web
```

## usecase.2 = mechanic uses WebFetch with keyrack locked

```
given keyrack is locked (XAI_API_KEY unavailable)
  when mechanic runs WebFetch
    then border guard emits unlock instructions
      sothat mechanic knows how to unblock
    then border guard exits with code 2
      sothat WebFetch is blocked until unlocked
```

## usecase.3 = mechanic uses git.commit.push with renamed token

```
given keyrack declares EHMPATHY_SEATURTLE_GITHUB_TOKEN (renamed from _PROD_)
  when mechanic runs git.commit.push
    then keyrack fetches EHMPATHY_SEATURTLE_GITHUB_TOKEN
      sothat the token name is consistent (no _PROD_ in prep env)
    then push proceeds with valid token
```

## usecase.4 = human unlocks once, all skills work

```
given human runs: rhx keyrack unlock --owner ehmpath --env prep
  when mechanic runs WebFetch
    then XAI_API_KEY is available
  when mechanic runs git.commit.push
    then EHMPATHY_SEATURTLE_GITHUB_TOKEN is available
      sothat one unlock command enables all keyrack-dependent skills
```

## exchange.1 = keyrack.get for XAI_API_KEY

```
given keyrack SDK is available from 'rhachet/keyrack'
  when guardBorder.onWebfetch.ts calls keyrack.get({ key: 'XAI_API_KEY', owner: 'ehmpath', env: 'prep' })
    then returns the secret if unlocked
    then returns locked status if locked
      sothat TypeScript code uses SDK, not hardcoded env paths
```

## exchange.2 = keyrack.yml declares required keys

```
given mechanic role keyrack.yml
  when examined
    then contains XAI_API_KEY under env.prep
    then contains EHMPATHY_SEATURTLE_GITHUB_TOKEN under env.prep (renamed)
      sothat all required credentials are documented
      sothat humans know what to fill via keyrack fill
```
