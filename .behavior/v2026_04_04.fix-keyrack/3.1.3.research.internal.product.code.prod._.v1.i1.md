# research: internal prod code patterns

## pattern.1: keyrack CLI fetch in shell skills

**[REUSE]**

`src/domain.roles/mechanic/skills/git.commit/keyrack.operations.sh` demonstrates the pattern for shell-based keyrack access:

```bash
# citation [1]: keyrack.operations.sh:32-36
keyrack_output=$("$repo_root/node_modules/.bin/rhachet" keyrack get \
  --key EHMPATHY_SEATURTLE_PROD_GITHUB_TOKEN \
  --env prep \
  --allow-dangerous \
  --json 2>&1) || keyrack_exit=$?
```

this pattern:
- uses CLI via node_modules/.bin/rhachet
- extracts secret via jq from JSON response
- has fallback to ehmpath owner unlock
