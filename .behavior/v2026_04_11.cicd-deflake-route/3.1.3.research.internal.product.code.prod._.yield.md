## pattern.2 = route init with bind

**[REUSE]** the init subskill creates route directory, copies templates, and binds to branch.

### citation.3 — route creation and bind

> source: `src/domain.roles/mechanic/skills/declapract.upgrade/init.sh:42-69`
>
> ```bash
> ISO_DATE=$(date +%Y_%m_%d)
> ROUTE_SLUG="v${ISO_DATE}.declapract.upgrade"
> ROUTE_PATH=".route/${ROUTE_SLUG}"
> TEMPLATES_DIR="$SKILL_DIR/declapract.upgrade/templates"
>
> # findsert route directory
> mkdir -p "$ROUTE_PATH"
>
> # copy only .stone and .guard files (no .sh files)
> for file in "$TEMPLATES_DIR"/*.stone "$TEMPLATES_DIR"/*.guard; do
>   if [[ -f "$file" ]]; then
>     cp -f "$file" "$ROUTE_PATH/"
>   fi
> done
>
> # bind route to branch
> CURRENT_BRANCH=$(git rev-parse --abbrev-ref HEAD)
> npx rhachet run --repo bhrain --skill route.bind.set --route "$ROUTE_PATH" > /dev/null 2>&1 || true
> ```

### citation.6 — gh api for workflow runs and jobs

> source: `src/domain.roles/mechanic/skills/show.gh.action.logs.sh:144-152,239-251`
>
> ```bash
> # build gh run list command
> LIST_CMD="gh run list --branch $BRANCH --limit 1 --json databaseId,workflowName,status,conclusion,createdAt"
> if [[ -n "$FLOW" ]]; then
>   LIST_CMD="$LIST_CMD --workflow $FLOW"
> fi
> RUNS_JSON=$(eval "$LIST_CMD")
> ...
> # get jobs for this run
> JOBS_JSON=$(gh api --method GET "repos/$REPO/actions/runs/$RUN_ID/jobs" -q '.jobs')
> # get only failed job ids
> JOB_IDS=$(echo "$JOBS_JSON" | jq -r '.[] | select(.conclusion == "failure") | .id')
> ```

**relation to wish**: evidence stone can reuse these patterns to query main-branch runs and identify flakes.

## pattern.5 = guard structure with self-review and peer-review

**[REUSE]** guards have structured sections: artifacts, reviews (self + peer), judges.

### citation.8 — self-review structure

> source: `src/domain.roles/mechanic/skills/declapract.upgrade/templates/3.1.repair.test.defects.guard:1-35`
>
> ```yaml
> reviews:
>   self:
>     - slug: has-zero-test-failures
>       say: |
>         do all tests now pass with zero failures?
>         ...
>
>     - slug: has-protected-test-intentions
>       say: |
>         were test intentions preserved?
>         we tolerate zero degradation of coverage. never skip, never drop.
>         ...
>         if any test was modified beyond infra fixes (imports, types, config),

> ```

**relation to wish**: execution guard needs similar pattern to detect failhide (test skip, test drop).

**relation to wish**: execution guard peer-review should use same pattern to detect failhide.

### citation.10 — judges
