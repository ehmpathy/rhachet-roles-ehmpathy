# blackbox criteria: cicd.deflake

## episode experience

### episode.1 = full deflake cycle

```
given(route not yet initialized)
  when(user runs `rhx cicd.deflake init`)
    then(route directory is created with all stones)
    then(route is bound to current branch)
    then(stdout shows files created and bind confirmation)
      sothat(user knows route is ready to drive)

given(route is initialized and bound)
  when(user runs `rhx route.drive`)
    then(next stone is shown with instructions)
      sothat(user knows what to do next)

given(driver completes a stone)
  when(driver runs `rhx route.stone.set --stone $name --as passed`)
    then(route advances to next stone)
    then(driver can continue via `rhx route.drive`)
```

### episode.2 = evidence to diagnosis flow

```
given(evidence stone is active)
  when(driver gathers flake evidence from main-branch ci)
    then(yield enumerates every flake: test name, frequency, error message)
    then(yield covers the lookback period, default 30 days)
      sothat(no flake is missed)

given(evidence is gathered)
  when(driver advances to diagnosis stone)
    then(guard ensures every flake from evidence appears in diagnosis)
      sothat(completeness is enforced)

given(diagnosis stone is active)
  when(driver diagnoses each flake)
    then(yield includes root cause hypotheses ranked by probability)
    then(yield includes verification approach for each hypothesis)
      sothat(diagnosis is actionable)
```

### episode.3 = plan to execution flow

```
given(diagnosis is complete)
  when(driver advances to plan stone)
    then(each flake has a concrete repair plan)
    then(each plan has verification criteria)
      sothat(repairs are verifiable)

given(plan is complete)
  when(driver advances to execution stone)
    then(repairs are made: code changes or infra changes)
    then(changes are released)
    then(ci passes at least once)
      sothat(repairs are validated before verification)
```

### episode.4 = verification with rewind

```
given(execution is complete and ci passed at least once)
  when(driver advances to verification stone)
    then(ci runs 3x consecutively)
    then(results are captured in yield)
      sothat(stability is proven)

given(verification runs and any test flakes)
  when(driver attempts to pass verification stone)
    then(guard blocks passage)
    then(guard ensures driver runs `--as rewound` to plan stone)
      sothat(driver revisits the plan with new information)

given(verification runs and no flakes for 3 consecutive runs)
  when(driver attempts to pass verification stone)
    then(guard allows passage)
      sothat(deflake is proven stable)
```

### episode.5 = itemize, reflect, institutionalize

```
given(verification passes)
  when(driver advances to itemize stone)
    then(each flake from evidence is matched to its repair)
    then(yield documents: true root cause, true fix, verification result)
      sothat(repairs are traceable)

given(itemization is complete)
  when(driver advances to reflection stone)
    then(systemic lessons are distilled)
    then(prevention strategies are documented)
    then(new briefs or upgrade handoffs proposed)
      sothat(institutional memory accumulates for future travelers)

given(reflection is complete and human approved)
  when(driver advances to institutionalize stone)
    then(approved lessons become briefs at .agent/repo=.this/role=any/briefs/)
    then(or lessons become upgrade handoffs if infra changes needed)
    then(yield confirms all lessons institutionalized)
      sothat(lessons persist beyond this deflake cycle)
```

## exchange experience

### exchange.1 = init

```
given(no route bound to branch)
  when(user runs `rhx cicd.deflake init`)
    then(route directory created: .behavior/v{date}.cicd-deflake/)
    then(8 stones created: 1.evidence through 8.institutionalize)
    then(5 guards created: diagnosis, plan, execution, verification, reflection)
    then(route bound to branch)
    then(stdout confirms: branch <-> route bind)

given(route already bound to branch)
  when(user runs `rhx cicd.deflake init`)
    then(error: route already bound)
    then(hint: use `rhx route.bind.del` to unbind first)
```

### exchange.2 = detect

```
given(route is initialized)
  when(user runs `rhx cicd.deflake detect --days 30 --into $route/1.evidence.yield.md`)
    then(gh api queries workflow runs from main branch)
    then(filters to failed runs that were retried and passed)
    then(extracts test names from failure logs)
    then(groups by test name with frequency count)
    then(samples error messages per test)
    then(writes structured flake inventory to --into file)
    then(stdout shows turtle vibes summary)

given(detect finds no flakes)
  when(command completes)
    then(stdout shows "zero flakes detected")
    then(file is written with empty inventory)
    then(exit 0)

given(detect finds flakes)
  when(command completes)
    then(stdout shows flake count and test names)
    then(file is written with full inventory)
    then(exit 0)

given(gh api rate limit hit)
  when(detect fails)
    then(error with hint to retry or extend timeout)
    then(exit 1)

given(--into path is outside route directory)
  when(user runs detect)
    then(error: --into must be within route directory)
    then(exit 2)
```

### exchange.3 = diagnosis

```
given(flake has clear error message)
  when(driver diagnoses)
    then(root cause hypothesis is formed)
    then(probability is assigned: high, medium, low)

given(flake has unclear error message)
  when(driver diagnoses)
    then(multiple hypotheses are formed)
    then(observability additions are proposed)
      sothat(next run can capture more data)

given(flake requires domain expertise)
  when(driver cannot diagnose)
    then(diagnosis marked as "escalate to expert")
    then(yield includes what is known and what is needed)
```

### exchange.4 = plan

```
given(diagnosis identifies root cause)
  when(driver plans repair)
    then(plan includes concrete steps)
    then(plan includes verification criteria)

given(diagnosis identifies multiple root causes, compound flake)
  when(driver plans repair)
    then(each root cause has its own repair steps)
    then(verification covers all causes)
```

### exchange.5 = execution

```
given(plan specifies code changes)
  when(driver executes)
    then(code is modified)
    then(commit is created)
    then(pr is merged to main)
    then(ci passes at least once)

given(plan specifies infra changes)
  when(driver executes)
    then(ci config is modified)
    then(or workflow is updated)
    then(changes are deployed)
    then(ci passes at least once)
```

### exchange.6 = verification

```
given(repairs are released and ci passed at least once)
  when(driver runs verification)
    then(ci triggered 3x on main)
    then(results captured: pass or fail per run)

given(any of 3 runs has a flake)
  when(driver checks verification)
    then(verification fails)
    then(driver must rewind to plan)

given(all 3 runs pass)
  when(driver checks verification)
    then(verification passes)
    then(driver can proceed to itemize)
```

### exchange.7 = guards

```
given(evidence yield has N flakes)
  when(diagnosis yield has fewer than N entries)
    then(guard blocks diagnosis stone passage)
      sothat(no flake is skipped)

given(repair removes test or adds skip)
  when(execution guard runs peer review)
    then(failhide pattern detected)
    then(guard blocks execution stone passage)
      sothat(test intent is preserved)

given(verification fails)
  when(driver attempts `--as passed` on verification)
    then(guard blocks)
    then(driver must run `--as rewound` to plan stone)
```

## edge cases

```
given(flake is infrastructure issue, not test issue)
  when(diagnosis distinguishes)
    then(infra flake documented separately)
    then(repair targets ci config, not test code)

given(test is inherently probabilistic)
  when(diagnosis identifies nondeterminism)
    then(plan includes time-isolation or retry pattern)
    then(test is made deterministic)

given(flake only reproduces under parallel load)
  when(diagnosis identifies parallelism issue)
    then(plan includes isolation or order fix)
    then(verification runs with parallelism enabled)

given(all flakes fixed but new flake appears in verification)
  when(verification fails)
    then(new flake added to evidence)
    then(rewind includes the new flake)
      sothat(comprehensive fix is achieved)
```
