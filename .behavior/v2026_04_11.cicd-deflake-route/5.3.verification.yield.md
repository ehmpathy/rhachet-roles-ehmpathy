### keyrack fix applied

the git.repo.test.sh skill was updated to export keyrack credentials as env vars for subprocesses:
```bash
eval "$(rhx keyrack source --owner ehmpath --env test --lenient 2>/dev/null)" || true
```

this fix ensures integration tests that need API keys (e.g., brief.compress with XAI_API_KEY) can access credentials from keyrack.
