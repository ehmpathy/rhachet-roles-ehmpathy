## phase 2: detect subcommand

**read before start:**
- `.behavior/v2026_04_11.cicd-deflake-route/3.3.1.blueprint.product.yield.md` (detect codepath)
- `.agent/repo=ehmpathy/role=mechanic/skills/show.gh.action.logs.sh` (gh api pattern)

### 2.1 create detect.sh

- [ ] create `src/domain.roles/mechanic/skills/cicd.deflake/detect.sh`
- [ ] parse arguments: --days, --into, --help
- [ ] validate --into path is within bound route directory
- [ ] fetch workflow runs from main via `gh api`
- [ ] filter to failed runs that were retried and passed (flake signal)
- [ ] extract test names from failure logs
- [ ] group by test name, count frequency
- [ ] sample error messages per test
- [ ] write JSON to --into file
- [ ] emit turtle vibes summary

**acceptance criteria:**
- `--into` must be within route directory (exit 2 otherwise)
- JSON output format: `{ "flakes": [{ "test": "...", "count": N, "errors": [...] }] }`
- stdout shows flake count and summary
- exit 0 even when no flakes found (just different message)

**verification:**
```sh
# with real repo that has flakes
rhx cicd.deflake detect --days 30 --into .behavior/v{date}.cicd-deflake/1.evidence.yield._.detected.json
# expect: JSON written, stdout shows summary

# outside route
rhx cicd.deflake detect --days 30 --into /tmp/outside.json
# expect: exit 2, path must be within route
```
