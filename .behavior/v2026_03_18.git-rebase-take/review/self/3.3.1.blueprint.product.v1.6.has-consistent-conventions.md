## reviewed: git.branch.rebase.take blueprint

set aside what i knew. looked at the codebase as if for the first time.

### what does "convention" mean here?

conventions are patterns that others can follow without instruction. if the pattern is consistent, new code writes itself.

i am not the author. i am the reviewer. i must find where the author diverged.

### convention 1: file names

examined `ls src/domain.roles/mechanic/skills/git.branch.rebase/`:

```
git.branch.rebase.abort.sh
git.branch.rebase.begin.sh
git.branch.rebase.continue.sh
git.branch.rebase.operations.sh
git.branch.rebase.sh
git.branch.rebase.take.sh
```

the pattern: `git.branch.rebase.<verb>.sh`

| verb | purpose |
|------|---------|
| begin | start a rebase |
| continue | resume after conflict |
| abort | cancel and restore |
| take | settle conflict (new) |

`take` fits the pattern. a verb. lowercase. `.sh` suffix.

**verdict**: no divergence.

### convention 2: dispatcher structure

opened `git.branch.rebase.sh` and found the switch:

```bash
case $SUBCOMMAND in
  begin)
    exec "$SCRIPT_DIR/git.branch.rebase.begin.sh" "$@"
    ;;
  continue)
    exec "$SCRIPT_DIR/git.branch.rebase.continue.sh" "$@"
    ;;
  abort)
    exec "$SCRIPT_DIR/git.branch.rebase.abort.sh" "$@"
    ;;
  take)
    exec "$SCRIPT_DIR/git.branch.rebase.take.sh" "$@"
    ;;
```

pattern: `exec "$SCRIPT_DIR/git.branch.rebase.$SUBCOMMAND.sh" "$@"`

the take case uses the exact same pattern. no special treatment.

**verdict**: no divergence.
