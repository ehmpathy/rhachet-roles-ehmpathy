### exit code discrepancy

**blueprint declares**:
```
├── 0 if all settled or partial success
└── non-zero if all failed or guard violation
```

**criteria declares** (usecase.9):
```
then(overall exit code reflects partial success or failure)
```

**implementation**:
```bash
if [[ $HAS_FAILED -gt 0 || $HAS_NOT_FOUND -gt 0 ]]; then
  exit 1  # includes partial success with errors
```

**analysis**: the blueprint says "0 if partial success" but implementation uses exit 1 for partial success. however, the criteria says exit code "reflects partial success or failure", which implementation satisfies via non-zero exit.

**is this a deviation?**

the blueprint phrasing was ambiguous. "partial success" could mean "some succeeded" (exit 0) or "not complete success" (exit non-zero). the criteria is clearer: exit code should "reflect" partial status.

the implementation follows criteria intent: if any file failed or was not found, exit non-zero. this is the safer choice — clones should know an issue arose.

**verdict**: ✓ adherant to criteria; blueprint had ambiguous phrasing.
