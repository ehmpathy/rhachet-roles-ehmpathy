# review: has-consistent-conventions (fifth pass)

## reviewed: git.branch.rebase.take blueprint

tea refilled. examined name conventions and patterns in the codebase.

### step 1: identified the extant patterns

ran `ls` on the git.branch.rebase directory to see the established convention:

```
git.branch.rebase.sh              # dispatcher
git.branch.rebase.begin.sh        # subcommand
git.branch.rebase.continue.sh     # subcommand
git.branch.rebase.abort.sh        # subcommand
git.branch.rebase.operations.sh   # shared logic
```

the pattern is clear:
- `git.branch.rebase.<action>.sh` for subcommands
- `git.branch.rebase.operations.sh` for shared functions

### step 2: verified take.sh follows the pattern

the new file is named `git.branch.rebase.take.sh`.

| convention | extant pattern | take.sh |
|------------|----------------|---------|
| namespace | git.branch.rebase | git.branch.rebase |
| action suffix | begin, continue, abort | take |
| file extension | .sh | .sh |
| dispatcher case | begin\|continue\|abort | take added |

the name fits. no divergence.

### step 3: examined the flag convention

looked at how begin.sh handles flags:

```bash
--mode)
  MODE="$2"

  ;;
```

take.sh uses the same pattern:

```bash
--whos)
  WHOS="$2"
  shift 2
  ;;
```

the `--` prefix and `shift 2` pattern are identical.

### step 5: examined the guard pattern

begin.sh guards:
```bash
if is_rebase_in_progress; then
  print_turtle_header "hold up dude..."
  ...
  exit 1
fi
```

take.sh guards:
```bash
if ! is_rebase_in_progress; then
  print_turtle_header "hold up dude..."
  ...
  exit 1
fi
```

note: the logic is inverted (begin requires NO rebase, take requires rebase) but the pattern is identical. correct behavior, consistent style.

### step 6: questioned the new term "--whos"

is `--whos` a divergent term?

searched the codebase for how side selection is expressed:
- no prior art for ours/theirs side selection
- this is a new capability

the term `--whos` is intentional (from vision document):
- `--who` reads as a query ("who did this?")
- `--whos` reads as possessive ("whose version?")

this is documented and justified. not a mistake.

### verdict

**no convention divergence detected.** the implementation follows all extant patterns:
- file name pattern
- flag pattern
- output pattern
- guard pattern

the only new element is `--whos`, which is justified in the vision document.
