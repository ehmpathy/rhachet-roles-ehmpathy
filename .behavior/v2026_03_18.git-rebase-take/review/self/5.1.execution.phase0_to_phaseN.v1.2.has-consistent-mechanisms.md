## examine: glob expansion

**what take.sh does** (lines 141):
```bash
matches=$(git ls-files "$pattern" 2>/dev/null || echo "")
```

**extant alternatives**:
- `compgen -G "$pattern"` — bash builtin, expands globs
- `find . -name "$pattern"` — filesystem search

**why `git ls-files`**:
- only matches tracked files (safety)
- respects gitignore
- consistent with git context

**is this duplicate?**: no. other skills don't do glob expansion on conflict files. this is new behavior unique to take.

**verdict**: not duplicate — unique to this use case.

## examine: dedup logic

**what take.sh does** (lines 166-174):
```bash
declare -A seen
for file in "${FILES_TO_SETTLE[@]}"; do
  if [[ -z "${seen[$file]:-}" ]]; then
    UNIQUE_FILES+=("$file")
    seen[$file]=1
  fi
done
```

**extant alternatives**:
- `sort -u` — unix dedup
- `uniq` — adjacent dedup

**why inline associative array**:
- preserves order (sort -u changes order)
- no subprocess spawn
- simple and readable

**is this duplicate?**: no. there's no shared `dedup_array()` function in operations.sh.

**verdict**: not duplicate — and a shared function would be premature abstraction for single use.
