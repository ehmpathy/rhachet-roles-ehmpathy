# blueprint: git.branch.rebase take

## summary

add `git.branch.rebase take` subcommand that settles rebase conflicts via selection of ours or theirs version for specified files.

**contract**:
```
rhx git.branch.rebase take --whos ours|theirs <paths...>
```

**guards**:
- must be in a rebase
- `--whos` must be `ours` or `theirs`
- paths required (use `.` for all)
- files must be in conflict (non-conflict files skipped with notice)

**behavior**:
- for each path: expand globs, filter to conflict files, `git checkout --{ours|theirs}`, `git add`
- best-effort: settle what can be settled, report errors for failures
- output: turtle vibes tree with status per file

### take subcommand

```
git.branch.rebase.take.sh
├── [←] source git.branch.rebase.operations.sh
├── [←] source output.sh
├── [+] parse args
│   ├── --whos → WHOS
│   ├── --skill|--repo|--role → skip (rhachet)
│   └── positional → PATHS[]
├── [+] guards
│   ├── [←] is_rebase_in_progress() || error "no rebase in progress"
│   ├── [+] WHOS in (ours, theirs) || error "whos must be ours or theirs"
│   └── [+] PATHS not empty || error "paths required"
├── [+] expand paths
│   ├── [←] CONFLICT_FILES=$(get_conflict_files)
│   ├── [+] if PATHS contains "." → FILES=$CONFLICT_FILES
│   └── [+] else → FILES = intersection(glob(PATHS), CONFLICT_FILES)
├── [+] settle files
│   ├── for FILE in FILES
│   │   ├── git checkout --$WHOS "$FILE"
│   │   ├── git add "$FILE"
│   │   └── track success/failure
│   └── collect results
├── [+] output tree
│   ├── print_turtle_header (righteous! or hold up dude...)
│   ├── print_tree_start "git.branch.rebase take"
│   ├── print_tree_branch "whos" "$WHOS"
│   ├── print_tree_branch "files"
│   │   └── for FILE: print success ✓ or error ✗
│   └── print_tree_leaf "done" or error summary
└── [+] exit code
    ├── 0 if all settled or partial success
    └── non-zero if all failed or guard violation
```

## implementation notes

1. **auto-stage**: `git checkout --theirs` does NOT auto-stage. explicit `git add` required after each checkout.

2. **glob expansion**: use bash glob on paths, then intersect with conflict files. skip non-conflict matches with notice.

3. **best-effort**: if some files settle and others fail, report both. exit non-zero only if ANY file failed.

4. **worktree support**: use `get_git_dir()` from operations.sh to handle `.git` as file (worktree case).

5. **ours/theirs semantics**: in a rebase, `ours` = base branch, `theirs` = commit in replay. counterintuitive but correct. document in output.
