# playtest: git.branch.rebase take

## prerequisites

- npm dependencies installed (`pnpm install --frozen-lockfile`)
- rhachet skills built (`npm run build`)
- rhachet roles linked (`npx rhachet roles link --role mechanic`)
- rhachet permissions initialized (`npx rhachet roles init --role mechanic`)

## sandbox

all file operations happen in `.temp/playtest-rebase-take/`. no repo pollution.

---

## setup: create rebase conflict scenario

```sh
# create sandbox
rm -rf .temp/playtest-rebase-take
mkdir -p .temp/playtest-rebase-take
cd .temp/playtest-rebase-take

# init repo
git init
git config user.email "test@example.com"
git config user.name "Test User"

# create initial file and commit
echo "original content" > conflict.txt
git add conflict.txt
git commit -m "initial commit"

# create main branch commit
echo "main branch content" > conflict.txt
git add conflict.txt
git commit -m "main change"

# create feature branch from initial
git checkout HEAD~1
git checkout -b feature
echo "feature branch content" > conflict.txt
git add conflict.txt
git commit -m "feature change"

# start rebase (this will conflict)
git rebase main || true
```

**expected**: rebase stops with conflict on `conflict.txt`.

---

## happy paths

### path 1: take theirs for single file

```sh
# still in .temp/playtest-rebase-take/
rhx git.branch.rebase take --whos theirs conflict.txt
```

**expected output**:
```
🐢 righteous!
git.branch.rebase take
   ├─ whos: theirs
   ├─ settled
   │  └─ conflict.txt
   └─ done
```

**verify**:
```sh
cat conflict.txt
# should show: main branch content

git diff --cached --name-only
# should show: conflict.txt (staged)
```

### path 2: continue rebase after take

```sh
git rebase --continue
```

**expected**: rebase completes successfully.

---

## edgey paths

### edge 1: no rebase in progress

```sh
# exit sandbox, go to repo root
cd ../..

# try take without rebase
rhx git.branch.rebase take --whos theirs pnpm-lock.yaml
```

**expected**: error message "no rebase in progress", exit non-zero.

### edge 2: file not in conflict

```sh
# setup new sandbox
rm -rf .temp/playtest-edge2
mkdir -p .temp/playtest-edge2
cd .temp/playtest-edge2

git init
git config user.email "test@example.com"
git config user.name "Test User"

echo "a" > a.txt
echo "b" > b.txt
git add .
git commit -m "initial"

echo "a2" > a.txt
git add a.txt
git commit -m "change a on main"

git checkout HEAD~1
git checkout -b feature
echo "a3" > a.txt
git add a.txt
git commit -m "change a on feature"

git rebase main || true
```

now `a.txt` conflicts but `b.txt` does not.

```sh
rhx git.branch.rebase take --whos theirs b.txt
```

**expected**: error about `b.txt` not in conflict or skipped.

### edge 3: invalid --whos value

```sh
rhx git.branch.rebase take --whos mine a.txt
```

**expected**: error "whos must be ours or theirs".

---

## pass/fail criteria

### ✓ pass if:

- [ ] `rhx git.branch.rebase take --whos theirs <file>` settles conflict
- [ ] settled file contains theirs (main branch) content
- [ ] settled file is staged
- [ ] `git rebase --continue` completes after take
- [ ] error shown when no rebase in progress
- [ ] error shown for invalid --whos value

### ✗ fail if:

- [ ] take command hangs or crashes
- [ ] settled file contains wrong content
- [ ] file not staged after take
- [ ] rebase cannot continue after take
- [ ] permission prompt appears (should be pre-approved)
