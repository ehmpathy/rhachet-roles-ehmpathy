# research: internal prod code patterns

## objective

identify extant patterns in `git.branch.rebase` skills to reuse, extend, or replace for the `take` subcommand.

---

## pattern.1: rebase state detection

**source**: `git.branch.rebase.operations.sh`

**what it does**: detects if a rebase is in progress by checking for `.git/rebase-merge` or `.git/rebase-apply` directories.

```bash
is_rebase_in_progress() {
  local git_dir
  git_dir=$(get_git_dir)
  [[ -d "$git_dir/rebase-merge" ]] || [[ -d "$git_dir/rebase-apply" ]]
}
```

**verdict**: [REUSE] — `take` needs this exact guard.

## pattern.3: conflict commit info

**source**: `git.branch.rebase.operations.sh`

**what it does**: gets the commit hash and message being replayed.

```bash
get_conflict_commit() {
  local git_dir
  git_dir=$(get_git_dir)
  if [[ -f "$git_dir/rebase-merge/stopped-sha" ]]; then
    cat "$git_dir/rebase-merge/stopped-sha"
  fi
}
```

**verdict**: [REUSE] — useful for output context, not critical for `take` but nice to have.

## pattern.4: file tree output

**source**: `git.branch.rebase.operations.sh`

**what it does**: prints files in tree format with proper indentation.

```bash
print_files_tree() {
  local files=("$@")
  local count=${#files[@]}
  local i=0
  for file in "${files[@]}"; do
    i=$((i + 1))
    if [[ $i -eq $count ]]; then
      echo "   │     └─ $file"
    else
      echo "   │     ├─ $file"
    fi
  done
}
```

**verdict**: [EXTEND] — `take` needs to show success/failure status per file (e.g., `file.ts ✓` or `file.ts ✗ not in conflict`).

## pattern.6: dispatcher

**source**: `git.branch.rebase.sh`

**what it does**: routes subcommands to their handlers via `exec`.

```bash
case "$SUBCOMMAND" in
  begin)   exec "$SKILL_DIR/git.branch.rebase.begin.sh" "$@" ;;
  continue) exec "$SKILL_DIR/git.branch.rebase.continue.sh" "$@" ;;
  abort)   exec "$SKILL_DIR/git.branch.rebase.abort.sh" "$@" ;;
  help)    show_help ;;
  *)       show_help; exit 1 ;;
esac
```

**verdict**: [EXTEND] — add `take)` case to dispatch to `git.branch.rebase.take.sh`.

## pattern.7: guard structure

**source**: `git.branch.rebase.begin.sh`

**what it does**: validates preconditions before action.

```bash
# guard: must not be on main
if [[ "$CURRENT_BRANCH" == "main" ]] || [[ "$CURRENT_BRANCH" == "master" ]]; then
  print_turtle_header "hold up dude..."
  echo "cannot rebase main/master"
  exit 1
fi

# guard: must have clean work tree
if ! git diff --quiet || ! git diff --cached --quiet; then
  print_turtle_header "hold up dude..."
  echo "work tree not clean"
  exit 1
fi
```

**verdict**: [REUSE] — `take` uses same guard pattern with different conditions (must be in rebase, files must be in conflict).

## pattern.8: arg parsing

**source**: `git.branch.rebase.begin.sh`

**what it does**: parses named args with shift pattern.

```bash
while [[ $# -gt 0 ]]; do
  case $1 in
    --mode)
      MODE="$2"
      shift 2
      ;;
    --skill|--repo|--role)
      shift 2
      ;;
    *)
      echo "unknown arg: $1"
      exit 1
      ;;
  esac
done
```

**verdict**: [EXTEND] — `take` adds `--whos` arg and collects positional paths.
