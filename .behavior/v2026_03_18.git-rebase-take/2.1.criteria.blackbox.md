# blackbox criteria: git.branch.rebase take

## usecase.1 = take theirs for a single file

```
given(rebase in progress with conflict on pnpm-lock.yaml)
  when(clone runs: rhx git.branch.rebase take --whos theirs pnpm-lock.yaml)
    then(file is replaced with theirs version)
      sothat(conflict is settled)
    then(file is staged)
      sothat(clone can continue rebase)
    then(output shows success tree with "done")
      sothat(clone knows it worked)
```

## usecase.2 = take ours for a single file

```
given(rebase in progress with conflict on .eslintrc.json)
  when(clone runs: rhx git.branch.rebase take --whos ours .eslintrc.json)
    then(file is replaced with ours version)
      sothat(conflict is settled)
    then(file is staged)
    then(output shows success tree with "done")
```

## usecase.3 = take theirs for multiple files via glob

```
given(rebase in progress with conflicts on package-lock.json, pnpm-lock.yaml, yarn.lock)
  when(clone runs: rhx git.branch.rebase take --whos theirs '*.lock' '*.yaml')
    then(all matched files in conflict are replaced with theirs version)
    then(all matched files are staged)
    then(output shows each file settled)
    then(files not in conflict are skipped with notice)
      sothat(clone knows what was skipped)
```

## usecase.4 = take theirs for all conflicts

```
given(rebase in progress with conflicts on 3 files)
  when(clone runs: rhx git.branch.rebase take --whos theirs .)
    then(all files in conflict are replaced with theirs version)
    then(all files are staged)
    then(output shows each file settled)
```

## usecase.5 = error: no rebase in progress

```
given(no rebase in progress)
  when(clone runs: rhx git.branch.rebase take --whos theirs pnpm-lock.yaml)
    then(error: "no rebase in progress")
    then(exit code non-zero)
      sothat(clone knows to start a rebase first)
```

## usecase.6 = error: file not in conflict

```
given(rebase in progress, but README.md has no conflict)
  when(clone runs: rhx git.branch.rebase take --whos theirs README.md)
    then(error: "file has no conflicts: README.md")
    then(exit code non-zero)
      sothat(clone knows the file doesn't need settlement)
```

## usecase.7 = error: invalid --whos value

```
given(rebase in progress with conflict)
  when(clone runs: rhx git.branch.rebase take --whos mine pnpm-lock.yaml)
    then(error: "whos must be ours or theirs")
    then(exit code non-zero)
```

## usecase.8 = error: no paths given

```
given(rebase in progress with conflict)
  when(clone runs: rhx git.branch.rebase take --whos theirs)
    then(error or help shown — paths required)
    then(exit code non-zero)
      sothat(clone knows to specify paths)
```

## usecase.9 = partial success (best-effort)

```
given(rebase in progress with conflicts on a.ts, b.ts, c.ts)
  when(clone runs: rhx git.branch.rebase take --whos theirs a.ts b.ts nonexistent.ts)
    then(a.ts and b.ts are settled and staged)
    then(nonexistent.ts shows error in output)
    then(overall exit code reflects partial success or failure)
      sothat(clone knows some files failed)
```

## usecase.10 = output format

```
given(rebase in progress with conflict on pnpm-lock.yaml)
  when(clone runs: rhx git.branch.rebase take --whos theirs pnpm-lock.yaml)
    then(output includes turtle header)
    then(output includes tree with whos value)
    then(output includes tree with files settled)
    then(output ends with "done")
      sothat(output is consistent with other git.branch.rebase skills)
```
