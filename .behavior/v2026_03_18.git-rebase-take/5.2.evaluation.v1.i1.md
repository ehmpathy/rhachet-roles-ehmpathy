# evaluation: git.branch.rebase take

## summary (as implemented)

added `git.branch.rebase take` subcommand that settles rebase conflicts via selection of ours or theirs version for specified files.

**contract**:
```
rhx git.branch.rebase take --whos ours|theirs <paths...>
```

**guards**:
- must be in a rebase
- `--whos` must be `ours` or `theirs`
- paths required (use `.` for all)
- files must be in conflict (non-conflict files skipped with notice)

**behavior**:
- for each path: expand globs via `git ls-files`, filter to conflict files, `git checkout --{ours|theirs}`, `git add`
- best-effort: settle what can be settled, report errors for failures
- output: turtle vibes tree with status per file
