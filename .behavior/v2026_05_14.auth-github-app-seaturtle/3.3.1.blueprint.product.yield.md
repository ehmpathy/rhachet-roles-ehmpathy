## notes

### what this blueprint does NOT include

1. **no prod code changes** — extant keyrack.operations.sh already fetches token
2. **no test code changes** — extant tests already cover token scenarios
3. **no keyrack.yml changes** — token already declared

### what this blueprint DOES include

1. **provision infra** — declastruct files to create github app
2. **readme** — usage docs for provisioner flow

### provision workflow (manual, one-time)

1. store admin token: `rhx keyrack set --key GITHUB_TOKEN --env sudo --vault 1password`
2. generate plan: `npx declastruct plan --wish ./provision/github.apps/resources.ts --into ./provision/github.apps/.temp/plan.json`
3. apply plan: `npx declastruct apply --plan ./provision/github.apps/.temp/plan.json`
4. store app creds: `rhx keyrack set --key EHMPATHY_SEATURTLE_GITHUB_TOKEN --env prep --vault os.secure`
5. store ci creds: `rhx keyrack set --key EHMPATHY_SEATURTLE_GITHUB_TOKEN --env prep --vault github.secrets`

### permissions granted

| permission | scope | operations |
|------------|-------|------------|
| `pull_requests: write` | repository | create PRs, enable automerge |
| `actions: read` | repository | view workflow runs and logs |
| `contents: read` | repository | read file contents |
| `metadata: read` | repository | required for all apps |

### app identity

- slug: `seaturtle-by-ehmpathy`
- name: `Seaturtle, by Ehmpathy`
- commit author: `seaturtle[bot] <seaturtle@ehmpath.com>`
- public: yes (any org can install)
