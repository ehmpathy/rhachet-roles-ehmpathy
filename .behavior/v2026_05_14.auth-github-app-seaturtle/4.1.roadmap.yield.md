## phase 1: provisioner workflow (manual, human-driven)

### read first

- `.behavior/v2026_05_14.auth-github-app-seaturtle/3.3.1.blueprint.product.yield.md` (notes: provision workflow)

### checklist

- [ ] 1.1 human stores github admin token in 1password
- [ ] 1.2 human runs `rhx keyrack unlock --owner ehmpath --env sudo`
- [ ] 1.3 human runs `npx declastruct plan` to preview
- [ ] 1.4 human runs `npx declastruct apply` to create app
- [ ] 1.5 human retrieves app credentials from github settings
- [ ] 1.6 human stores EHMPATHY_SEATURTLE_GITHUB_TOKEN in keyrack

### acceptance criteria

```
given human has admin token unlocked
  when human runs `npx declastruct plan --wish ./provision/github.apps/resources.ts`
    then plan shows app creation diff
      sothat human can review before apply

given plan looks correct
  when human runs `npx declastruct apply`
    then github app "seaturtle-by-ehmpathy" is created
      sothat app exists for installation

given app is created
  when human visits github.com/apps/seaturtle-by-ehmpathy
    then app page is visible
      sothat users can install it
```

### verification

```sh
# verify app exists after apply
gh api /apps/seaturtle-by-ehmpathy --jq '.name'
# expected: "Seaturtle, by Ehmpathy"
```
