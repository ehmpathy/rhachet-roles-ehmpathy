## pattern.1 = keyrack token fetch [REUSE]

### what

mechanic skills fetch github tokens via keyrack with fallback to ehmpath owner unlock.

### citation.1

file: `src/domain.roles/mechanic/skills/git.commit/keyrack.operations.sh`

```bash
fetch_github_token() {
  local token=""
  local keyrack_exit=0

  # try to get the token from keyrack
  keyrack_output=$("$repo_root/node_modules/.bin/rhachet" keyrack get \
    --key EHMPATHY_SEATURTLE_GITHUB_TOKEN \
    --env prep \
    --allow-dangerous \
    --json 2>&1) || keyrack_exit=$?

  if [[ $keyrack_exit -eq 0 ]]; then
    token=$(echo "$keyrack_output" | jq -r '.grant.key.secret // empty')
  fi

  # if empty, try to unlock ehmpath keyrack and retry
  if [[ -z "$token" ]]; then
    "$repo_root/node_modules/.bin/rhachet" keyrack unlock \
      --owner ehmpath \
      --env prep 2>/dev/null || true

    keyrack_output=$("$repo_root/node_modules/.bin/rhachet" keyrack get \
      --key EHMPATHY_SEATURTLE_GITHUB_TOKEN \
      --env prep \
      --allow-dangerous \
      --json 2>&1) || keyrack_exit=$?

    if [[ $keyrack_exit -eq 0 ]]; then
      token=$(echo "$keyrack_output" | jq -r '.grant.key.secret // empty')
    fi
  fi

  echo "$token"
}
```

### relation to wish

this pattern fetches the EHMPATHY_SEATURTLE_GITHUB_TOKEN that the seaturtle app will provide. once the app is provisioned and credentials stored in keyrack, this extant code will work without modification.

```bash
_gh_with_retry() {
  local max_attempts=3
  local attempt=1
  local exit_code=0

  while [[ $attempt -le $max_attempts ]]; do
    if GH_TOKEN="$EHMPATHY_SEATURTLE_GITHUB_TOKEN" gh "$@"; then
      return 0
    fi
    exit_code=$?
    # ... retry logic
  done
  return $exit_code
}
```

### relation to wish

this pattern works for any github token. once keyrack returns the seaturtle app token (via EPHEMERAL_VIA_GITHUB_APP mechanism), all extant gh cli calls will authenticate as seaturtle[bot].
