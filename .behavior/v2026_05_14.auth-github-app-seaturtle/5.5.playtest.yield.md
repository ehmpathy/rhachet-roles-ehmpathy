# playtest: seaturtle-by-ehmpathy github app

## prerequisites

- github token with `admin:org` scope via keyrack
  - one-time setup: `rhx keyrack set --owner admin --vault os.daemon --env prod --key GITHUB_TOKEN`
  - permissions: organization → administration → read and write
- declastruct cli installed: `npm install -g declastruct`
- access to the ehmpathy github org

## sandbox

all operations target the github org, not local filesystem. no local file ops needed.

## happy paths

### 1. declaration is type-safe

```sh
npm run test:types
```

**expected outcome:**
- exit code 0
- no type errors

**pass criteria:** type check passes, declaration conforms to github app schema

### 2. declastruct plan shows app will be created

```sh
rhx keyrack unlock --owner admin --env prod --key GITHUB_TOKEN
npx declastruct plan --wish ./provision/github.apps/resources.ts --into ./provision/github.apps/.temp/plan.json
```

**expected outcome:**
- plan shows seaturtle-by-ehmpathy app will be created
- permissions listed: pullRequests=write, actions=read, contents=read, metadata=read
- no unexpected resources

**pass criteria:** plan output shows exactly the declared app with declared permissions

### 3. declastruct apply creates the app

```sh
npx declastruct apply --plan ./provision/github.apps/.temp/plan.json
```

**expected outcome:**
- app is created in github
- app slug: seaturtle-by-ehmpathy
- app permissions match declaration

**pass criteria:** app appears in github org settings → developer settings → github apps

### 4. re-run is idempotent

```sh
npx declastruct plan --wish ./provision/github.apps/resources.ts --into ./provision/github.apps/.temp/plan.json
npx declastruct apply --plan ./provision/github.apps/.temp/plan.json
```

**expected outcome:**
- plan shows no changes (app already exists)
- apply is a no-op

**pass criteria:** repeated apply does not error or create duplicates

### 5. app can be installed to a repo

1. navigate to: github.com/organizations/ehmpathy/settings/apps/seaturtle-by-ehmpathy
2. click "install app"
3. select a test repo (e.g., this repo: ehmpathy/rhachet-roles-ehmpathy)
4. confirm installation

**expected outcome:**
- installation succeeds
- app appears in repo settings → integrations → github apps

**pass criteria:** app installs without error

## edgey paths

### no github token

```sh
unset GITHUB_TOKEN
declastruct plan
```

**expected behavior:** clear error about auth requirement

### insufficient permissions

(token without admin:org scope)

**expected behavior:** clear error about permission scope

## pass/fail criteria

- pass if:
  - type check passes (exit 0, no errors)
  - declastruct plan shows app with exact permissions:
    - pullRequests: write
    - actions: read
    - contents: read
    - metadata: read
  - declastruct apply creates app in github
  - re-run is idempotent (no changes on second apply)
  - app can be installed to a repo

- fail if:
  - type errors in declaration
  - plan shows unexpected resources or permissions
  - apply fails to create app
  - re-run creates duplicates or errors
  - permissions differ from: pullRequests=write, actions=read, contents=read, metadata=read
