# vision: stop /tmp permission noise

## the outcome world

### before

```
human: tail -50 /tmp/claude-1000/.../tasks/abc123.output
claude: [permission prompt] Allow reading /tmp/claude-1000/...?
human: yes
claude: [shows output]
human: cat /tmp/claude-1000/.../another-file
claude: [permission prompt again]
human: yes
human: (sighs)
```

mechanic wants to check agent task output. every read from claude's own temp directory triggers a permission prompt. friction accumulates. flow breaks.

### after

```
human: tail -50 /tmp/claude-1000/.../tasks/abc123.output
claude: [auto-allowed, shows output immediately]
human: cat /tmp/claude-1000/.../another-file
claude: [auto-allowed, shows immediately]
```

reads from `/tmp/claude*` flow without interruption. claude can inspect its own workspaces.

```
human: echo "data" > /tmp/scratch.txt
claude: [blocked]
        🛑 /tmp is not actually temporary
        /tmp persists indefinitely and never auto-cleans.
        use .temp/ instead - it's scoped to this repo and gitignored.
```

writes to `/tmp/*` are blocked with an explanation. /tmp is a misnomer - .temp/ is actually temporary.

### aha moment

the value clicks when: you're debugging an agent task failure, need to read 5 different output files from `/tmp/claude-1000/...`, and none of them prompt for permission. the investigation flows without friction.

## user experience

### usecases

| goal | before | after |
|------|--------|-------|
| read agent task output | permission prompt per file | auto-allowed |
| inspect claude session temp | permission prompt | auto-allowed |
| write scratch file to /tmp | allowed (but scatters outside repo) | blocked → nudged to .temp/ |
| write scratch file to .temp/ | works | works (preferred path) |

### contract inputs & outputs

**for reads:**
- input: `cat /tmp/claude-1000/...` or `tail /tmp/claude-*`
- output: file contents, no prompt

**for writes:**
- input: `echo x > /tmp/foo`
- output: blocked message with .temp/ alternative

### timeline

1. mechanic invokes read from `/tmp/claude*` → auto-allowed
2. mechanic invokes write to `/tmp/*` → blocked with guidance
3. mechanic uses `.temp/` instead → works, artifacts stay in repo

## mental model

### how users describe it

> "claude can read its own temp files without asking, but won't write outside the repo"

### analogies

- **your own room**: you can look in your own closet without asking. but you shouldn't leave your stuff in the hallway.
- **sandbox boundaries**: read from the sandbox, but write only within the repo.

### terms

| user says | we say |
|-----------|--------|
| "temp files" | `/tmp/claude*` paths |
| "scratch space" | `.temp/` directory |
| "outside the repo" | `/tmp/*` non-claude paths |

## evaluation

### how well does it solve the goals?

| goal | solved? |
|------|---------|
| stop permission prompts for claude temp reads | yes |
| prevent scattered writes to /tmp | yes |
| guide toward .temp/ | yes |

### pros

- eliminates repetitive permission prompts for claude's own temp
- keeps artifacts in-repo (trackable, cleanable)
- clear guidance when blocked

### cons

- slightly more restrictive than before (writes to /tmp blocked)
- .temp/ needs to exist or be auto-created

### edgecases

| edge | pit of success |
|------|----------------|
| read non-claude /tmp paths | still prompts (correct - external files) |
| write to .temp/ | allowed (preferred path) |
| write to /tmp/claude* | [answered] block tool writes - internal writes unaffected |

## what is awkward?

### what feels off

- the distinction between "claude's temp" and "general /tmp" requires pattern match
- two different mechanisms (permissions vs hooks) for related behavior

### where design fights mental model

- users might expect all /tmp to be blocked or all allowed, not conditional

### uncomfortable tradeoffs

- we add complexity (pattern match + permissions + hooks) to gain convenience

### why /tmp is wrong for scratch files

/tmp is a misnomer - it's not actually temporary:
- persists indefinitely on most systems
- never auto-cleans
- accumulates garbage forever
- mixes files from all projects/processes

.temp/ is actually temporary:
- scoped to the project
- cleaned when repo is deleted
- easy to clean manually
- gitignored by convention

block /tmp writes is not opinionated - it's correct.

## proposed behavior (updated after research)

**for auto-allow reads from /tmp/claude*:**
- mechanism: permission rules in settings.json
- Bash: command prefix patterns work, e.g., `Bash(cat /tmp/claude:*)`
- Read: unclear if path patterns work - need to test `Read(/tmp/claude*)`

**for auto-block writes to /tmp/*:**
- mechanism: PreToolUse hook with exit 2
- intercepts Write/Edit/Bash tools
- nudge message explains why:

```
🛑 BLOCKED: /tmp is not actually temporary

/tmp persists indefinitely and never auto-cleans.
use .temp/ instead - it's scoped to this repo and gitignored.

  echo "data" > .temp/scratch.txt
```

```
# settings.json permissions.allow additions:
Bash(cat /tmp/claude:*)
Bash(head /tmp/claude:*)
Bash(tail /tmp/claude:*)
Read(/tmp/claude*)  # test if this works

# new PreToolUse hook (matcher: Write|Edit|Bash):
if file_path or command targets /tmp/*:
  exit 2 with "use .temp/ instead"
```
