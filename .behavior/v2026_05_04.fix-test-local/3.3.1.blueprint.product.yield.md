### problem diagnosis

**why mock npm fails:**

1. `setupFixture` creates `.fakebin/npm` with correct stdout/stderr/exit
2. `setupFixture` prepends `.fakebin` to PATH in `env` object
3. `runGitRepoTest` passes `env` to `spawnSync`
4. `spawnSync` runs `bash git.repo.test.sh` with modified PATH
5. **BUT** the skill runs `npm run test:unit` — npm inherits PATH from spawnSync
6. npm subprocess does NOT see `.fakebin` in its PATH
7. real npm executes, calls mock jest, mock jest exits 0

**why mock jest hides the issue:**

the mock jest executable exits 0 for all non-listTests calls:
```bash
if [[ "$*" == *"--listTests"* ]]; then
  echo "${fileList}"
  exit 0
fi
# for non-listTests calls, just succeed
exit 0  # <-- this masks npm mock failure
```

### fix strategy

**option A: intercept npm differently**

instead of PATH injection (which npm subprocess ignores), create a mock `package.json` with `scripts` entries that directly echo the expected output:

```json
{
  "scripts": {
    "test:unit": "echo 'Test Suites: 1 passed' && exit 0"
  }
}
```

this approach:
- works because npm reads local package.json
- produces deterministic stdout/stderr
- can set any exit code via `exit N`

**option B: use pnpm instead of npm**

pnpm may propagate PATH correctly. needs verification.

**option C: mock at lower level**

mock `node` or `jest` binary directly instead of npm.

### recommended: option A

mock via package.json `scripts` field is most reliable:
- no PATH propagation issues
- full control over stdout, stderr, exit code
- matches how the real skill works

## execution plan

### phase 1: fix setupFixture

update `setupFixture` to write mock entries into `package.json` `scripts` field:

```typescript
// instead of .fakebin/npm, write package.json with mock scripts
const scripts: Record<string, string> = {};
if (config.mockNpm) {
  const exitCode = config.mockNpm.exitCode ?? 0;
  const stdout = config.mockNpm.stdout ?? '';
  const stderr = config.mockNpm.stderr ?? '';
  scripts[`test:${config.testType ?? 'unit'}`] =
    `echo '${stdout}' && echo '${stderr}' >&2 && exit ${exitCode}`;
}
```

### phase 2: update all test cases

each case that uses `mockNpm` needs its mock moved into package.json `scripts`.

### phase 3: run tests and capture snapshots
