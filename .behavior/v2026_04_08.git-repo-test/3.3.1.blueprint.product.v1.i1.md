# blueprint.product: git.repo.test

blueprint for git.repo.test skill extension.

---

## summary

extend `git.repo.test.sh` to support:
- `--what unit | integration | acceptance | all` (in addition to lint)
- `--what all` runs lint в†’ unit в†’ integration в†’ acceptance in sequence
- `--scope <pattern>` for jest test path filter
- `--resnap` for snapshot updates via `RESNAP=true`
- `--thorough` for full test runs via `THOROUGH=true`
- auto keyrack unlock for integration/acceptance
- log capture on success AND failure (not just failure like lint)
- namespaced log paths: `.log/.../what=${WHAT}/...` to avoid overlap
- jest output parse for stats (suites, tests passed/failed/skipped, time)
- **progressive output**: header appears immediately, timer lines stack, final summary after completion
- **scope preview**: show matched file count before test runs

deliverables:
1. extended `git.repo.test.sh` skill
2. `git.repo.test.play.integration.test.ts` journey tests
3. `howto.run-tests.[lesson].md` brief

### fixture pattern

each journey creates a temp git repo with:
- `package.json` with appropriate test commands
- `jest.config.js` (if needed)
- test files that pass/fail/skip as required
- mock `rhx keyrack` via PATH injection for journey 5 (hermetic test)

```typescript
const setupFixture = (config: {
  packageJson: object;
  jestConfig?: string;
  testFiles?: Record<string, string>;
  mockKeyrack?: boolean; // for journey 5: mock keyrack via PATH
}): { tempDir: string; env: NodeJS.ProcessEnv } => {
  const tempDir = genTempDir({ slug: 'git-repo-test', git: true });
  fs.writeFileSync(
    path.join(tempDir, 'package.json'),
    JSON.stringify(config.packageJson, null, 2),
  );
  if (config.jestConfig) {
    fs.writeFileSync(path.join(tempDir, 'jest.config.js'), config.jestConfig);
  }
  if (config.testFiles) {
    for (const [filePath, content] of Object.entries(config.testFiles)) {
      const fullPath = path.join(tempDir, filePath);
      fs.mkdirSync(path.dirname(fullPath), { recursive: true });
      fs.writeFileSync(fullPath, content);
    }
  }

  // mock keyrack for hermetic tests (journey 5)
  let env = { ...process.env };
  if (config.mockKeyrack) {
    const fakeBinDir = path.join(tempDir, '.fakebin');
    fs.mkdirSync(fakeBinDir);
    fs.writeFileSync(
      path.join(fakeBinDir, 'rhx'),
      `#!/bin/bash
if [[ "$1" == "keyrack" && "$2" == "unlock" ]]; then
  echo "unlocked ehmpath/test"
  exit 0
fi
exec "$(which rhx)" "$@"
`,
    );
    fs.chmodSync(path.join(fakeBinDir, 'rhx'), '755');
    env = { ...env, PATH: `${fakeBinDir}:${process.env.PATH}` };
  }

  return { tempDir, env };
};
```

## hook contract compatibility

the skill is used in hook context (`--when hook.onStop`). the extension must preserve:

- **exit code 2** on test failure вЂ” forces brain to address defects
- **summary output** вЂ” no raw jest stream to terminal, saves context tokens
- **consistent format** вЂ” turtle vibes output, same structure as lint
- **--what lint behavior unchanged** вЂ” extant hook usage continues to work

the new flags (`--scope`, `--resnap`, `--thorough`, `--what all`) are additive. the default behavior for `--what lint` remains unchanged.
