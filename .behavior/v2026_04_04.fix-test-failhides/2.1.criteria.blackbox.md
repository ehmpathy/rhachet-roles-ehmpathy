# blackbox criteria: failhide/failfast/failloud rules

---

**this is the most important rule you'll ever read.**

if it can't fulfill the contract, failfast.

failhides are broken promises — they break trust.

---

## usecase.1 = mechanic writes prod code

```
given(mechanic writes prod code)
  when(code hides error and continues as if ok)
    then(rule.forbid.failhide blocks)
      sothat(broken promises are caught before merge)
  when(code detects bad state but does not exit early)
    then(rule.require.failfast blocks)
      sothat(defects are caught at origin, not downstream)
  when(code throws error without proper class or full details)
    then(rule.require.failloud blocks)
      sothat(errors are diagnosable without debug sessions)
  when(code throws ConstraintError/MalfunctionError with full context)
    then(all rules pass)
      sothat(code fulfills its contract or fails loud)
```

## usecase.2 = mechanic writes test code

```
given(mechanic writes test code)
  when(test passes without verification on some code path)
    then(rule.forbid.failhide blocks)
      sothat(tests that lie about coverage are caught)
  when(test lacks required resource but silently passes)
    then(rule.forbid.failhide blocks)
      sothat(absent resource = unacceptable, always fail)
  when(test conditional has one branch that verifies and one that does not)
    then(rule.forbid.failhide blocks)
      sothat(every code path either verifies or explicitly fails)
  when(test throws ConstraintError on absent resource with clear message)
    then(all rules pass)
      sothat(test either runs and verifies, or fails loud)
```

## usecase.3 = behavior guard reviews code

```
given(behavior guard reviews pr)
  when(diff contains failhide pattern in prod code)
    then(guard emits blocker)
      sothat(failhides cannot merge)
  when(diff contains failhide pattern in test code)
    then(guard emits blocker)
      sothat(test failhides cannot merge)
  when(diff contains failfast/failloud violations)
    then(guard emits blocker)
      sothat(all error paths are explicit and loud)
  when(diff has no failhide/failfast/failloud violations)
    then(guard passes)
      sothat(clean code merges without friction)
```

## usecase.4 = session boots with mechanic role

```
given(session boots with mechanic role)
  when(boot.yml loads)
    then(all 6 failhide/failfast/failloud rules are in context)
      sothat(mechanic always knows the most important rules)
    then(rules are in say section, not ref)
      sothat(rules are always visible, never just referenced)
```

## usecase.5 = mechanic encounters error scenario

```
given(mechanic encounters error scenario)
  when(caller must fix the issue)
    then(throws ConstraintError or BadRequestError)
      sothat(caller knows they need to act)
  when(server must fix the issue)
    then(throws MalfunctionError or UnexpectedCodePathError)
      sothat(developer knows system failed)
  when(error is thrown)
    then(error includes full context about the defect)
      sothat(diagnosis requires no additional debug)
```

---

## failhide patterns (forbidden)

prod code:
- `try { ... } catch { /* swallow */ }`
- `try { ... } catch { return null }`
- `if (error) { /* ignore */ }`

test code:
- `if (!condition) { expect(true).toBe(true) }`
- `if (!hasResource) { return }` (silent skip)
- `expect([0, 1, 2]).toContain(exitCode)` (accepts errors as valid)
- `expect.any(Object)` (avoids verification)
- empty test body: `it('does x', () => {})`

## failfast patterns (required)

- early return/throw on bad state
- guard clauses at function start
- throw immediately when contract cannot be fulfilled

## failloud patterns (required)

- use ConstraintError/BadRequestError for caller-must-fix (exit 2)
- use MalfunctionError/UnexpectedCodePathError for server-must-fix (exit 1)
- include full context: what failed, why, what to do
- error message should make diagnosis immediate
