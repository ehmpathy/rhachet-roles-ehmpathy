# playtest: rhx commands auto-approved without prompt

## prerequisites

1. the branch has been built: `npm run build`
2. roles have been reinitialized: `npx rhachet roles init --role mechanic`
3. you are in a Claude Code session with the mechanic role active
   - verify: session start shows "SessionStart:compact hook success" with mechanic briefs

## what a permission prompt looks like

when Claude Code prompts for approval, you see:
- a gray box with "This command requires approval"
- the command text
- Allow/Deny buttons

if you do NOT see this box, the command ran without a prompt.

## sandbox

all file operations target files that already exist in the codebase. no new files need to be created.

## happy paths

### path 1: sedreplace with curly braces

**action:**
run this command in Claude Code:
```sh
rhx sedreplace --old '{ identity: x }' --new '{ identity: y }' --glob 'src/domain.roles/mechanic/getMechanicRole.ts'
```

**expected outcome:**
- command runs WITHOUT a permission prompt
- sedreplace shows turtle vibes output (🐢 header, plan summary)
- even "0 matches found" means success — the command ran
- no "this command requires approval" dialog appears

### path 2: sedreplace with parentheses

**action:**
run this command in Claude Code:
```sh
rhx sedreplace --old 'foo(bar)' --new 'baz()' --glob 'src/domain.roles/mechanic/getMechanicRole.ts'
```

**expected outcome:**
- command runs WITHOUT a permission prompt
- sedreplace shows plan output
- no "this command requires approval" dialog appears

### path 3: grepsafe with pipe in regex

**action:**
run this command in Claude Code:
```sh
rhx grepsafe --pattern 'onTool|onBoot' --glob 'src/**/*.ts'
```

**expected outcome:**
- command runs WITHOUT a permission prompt
- grepsafe shows matches
- no "this command requires approval" dialog appears

## edge paths

### edge 1: rhx with pipe operator prompts

**action:**
ask Claude Code to run:
```sh
rhx grepsafe --pattern 'foo' | cat
```

**expected outcome:**
- command SHOULD prompt for approval
- the hook detects the pipe outside quotes and passes through
- this verifies the security boundary works

## pass/fail criteria

### pass if:

1. happy paths 1-3 all run without permission prompts
2. edge path 1 triggers a permission prompt (or Claude Code refuses to run it)
3. the turtle vibes output from sedreplace/grepsafe appears

### fail if:

1. any happy path triggers a permission prompt with "suspicious syntax"
2. edge path 1 runs without any prompt (security bypass)
3. the hook causes errors or crashes

## notes

- if the hook is not registered, all paths will prompt (fail case)
- check `.claude/settings.json` to verify `allow-rhx-skills` hook is present
- the hook should be FIRST in the onTool list (before forbid-* hooks)
