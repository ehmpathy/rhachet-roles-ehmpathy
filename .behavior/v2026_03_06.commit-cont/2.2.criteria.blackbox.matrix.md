# blackbox criteria matrix: cont-prefixed continuation commits

## dimensions

### independent variables
1. **branch state**: whether branch has a behavioral commit
2. **commit type**: the prefix of the attempted commit

### dependent variables
1. **outcome**: SUCCEEDS or BLOCKED
2. **branch state after**: whether branch now has behavioral commit
3. **error context**: what info the error provides (if blocked)

---

## matrix: commit validation

| ind: branch state | ind: commit type | dep: outcome | dep: branch state after | dep: error context |
|-------------------|------------------|--------------|-------------------------|-------------------|
| no behavioral | `fix(scope):` | SUCCEEDS | has behavioral | — |
| no behavioral | `feat(scope):` | SUCCEEDS | has behavioral | — |
| no behavioral | `cont(scope):` | SUCCEEDS | no behavioral | — |
| no behavioral | `cont:` | SUCCEEDS | no behavioral | — |
| no behavioral | `chore:` | SUCCEEDS | no behavioral | — |
| no behavioral | `docs:` | SUCCEEDS | no behavioral | — |

| has behavioral | `feat(scope):` | BLOCKED | has behavioral | shows first behavioral, suggests `cont:` |
| has behavioral | `cont(scope):` | SUCCEEDS | has behavioral | — |
| has behavioral | `cont:` | SUCCEEDS | has behavioral | — |
| has behavioral | `chore:` | SUCCEEDS | has behavioral | — |
| has behavioral | `docs:` | SUCCEEDS | has behavioral | — |

---

## coverage analysis

### gap check
all 12 combinations (2 branch states × 6 commit types) are covered. no gaps.

### symmetry observations
1. **non-behavioral commits (`chore:`, `docs:`)**: always succeed regardless of branch state
2. **continuation commits (`cont:`, `cont(scope):`)**: always succeed regardless of branch state
3. **behavioral commits (`fix:`, `feat:`)**: succeed only when branch has no behavioral commit

### edge case: cont on fresh branch
the matrix shows `cont:` succeeds on a branch with no behavioral commits. this is technically allowed but semantically odd — what is continued?

decision: **allow** — the enforcement prevents pollution of changelogs with multiple fix/feat entries. a cont on fresh branch is harmless (won't appear in changelog anyway).

---

## decomposition assessment

**dimension count**: 2 independent variables
**combination count**: 12 rows

this is well within reasonable bounds. no decomposition needed.

the behavior is simple and focused:
- one clear rule: after first behavioral commit, block subsequent fix/feat
- one clear escape: use `cont:` prefix
- non-behavioral commits are exempt
