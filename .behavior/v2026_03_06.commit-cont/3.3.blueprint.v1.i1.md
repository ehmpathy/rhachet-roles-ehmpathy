# blueprint: cont-prefixed continuation commits

## summary

enforce that after the first behavioral commit (`fix:`/`feat:`) on a branch, subsequent fix/feat commits must use `cont:` or `cont(scope):` prefix. non-behavioral commits (`chore:`, `docs:`, etc.) are exempt.

## domain logic

### get_behavioral_commits_on_branch

returns:
- `"NO_COMMITS"` if repo has no commits (fail fast)
- `"NO_BASE"` if no base branch found: main/master/trunk (fail fast)
- `"ON_BASE"` if current branch is the base branch (fail fast)
- commit subjects (one per line) if behavioral commits found
- empty string if no behavioral commits on branch

```bash
get_behavioral_commits_on_branch() {
  # fail fast: repo has no commits
  if ! git rev-parse HEAD >/dev/null 2>&1; then
    echo "NO_COMMITS"
    return 0
  fi

  # find base branch (main, master, trunk)
  local base_branch=""
  for candidate in main master trunk; do
    if git rev-parse --verify "$candidate" >/dev/null 2>&1; then
      base_branch="$candidate"
      break
    fi
  done

  # fail fast: no known base branch found
  if [[ -z "$base_branch" ]]; then
    echo "NO_BASE"
    return 0
  fi

  # fail fast: current branch IS the base branch
  local current_branch
  current_branch=$(git rev-parse --abbrev-ref HEAD 2>/dev/null || echo "")
  if [[ "$current_branch" == "$base_branch" ]]; then
    echo "ON_BASE"
    return 0
  fi

  # list commits on branch since divergence from base (oldest first)
  # filter to fix: or feat: prefixed commits
  git log --reverse --format="%s" "$base_branch..HEAD" 2>/dev/null | \
    grep -E "^(fix|feat)(\([^)]+\))?:" || true
}
```

### validation logic placement

insert after bound level constraint (line ~302), before quota check (line ~304):

```bash
# guard: continuation commit enforcement
BEHAVIORAL_COMMITS=$(get_behavioral_commits_on_branch)
if [[ -n "$BEHAVIORAL_COMMITS" ]]; then
  # branch already has a behavioral commit
  if [[ "$COMMIT_PREFIX" == "fix" || "$COMMIT_PREFIX" == "feat" ]]; then
    FIRST_BEHAVIORAL=$(echo "$BEHAVIORAL_COMMITS" | head -n1)

    # extract scope from header for helpful suggestion
    COMMIT_SCOPE=""
    if [[ "$HEADER" =~ ^[a-z]+\(([^)]+)\): ]]; then
      COMMIT_SCOPE="${BASH_REMATCH[1]}"
    fi

    print_turtle_header "bummer dude..."
    print_tree_start "git.commit.set"
    print_tree_error "branch already has a behavioral commit"
    echo ""
    echo "   first behavioral commit: $FIRST_BEHAVIORAL"
    echo "   attempted: $HEADER"
    echo ""
    if [[ -n "$COMMIT_SCOPE" ]]; then
      echo "   use \`cont:\` or \`cont($COMMIT_SCOPE):\` prefix"
    else
      echo "   use \`cont:\` prefix"
    fi
    exit 2
  fi
fi
```

## risks

### rebase reorder
if commits are reordered via rebase, the "first" behavioral commit changes. mitigation: use topological order (`git log base..HEAD`), not timestamp.

### orphan branches
branches without a merge-base to main/master. mitigation: if no base found, skip validation (allow all).

### detached HEAD
similar to orphan — no branch context. mitigation: skip validation in detached HEAD state.
