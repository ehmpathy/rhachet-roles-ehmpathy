# blackbox criteria: cont-prefixed continuation commits

## definitions

- **behavioral commit**: `fix:` or `feat:` prefixed (changes behavior)
- **first behavioral commit**: first fix/feat on branch
- **continuation commit**: subsequent fix/feat (must use `cont:`)
- **non-behavioral commit**: `chore:`, `docs:`, etc. (exempt)

---

# usecase.1 = first behavioral commit on branch

given(branch has no behavioral commits yet)
  when(mechanic commits with `fix(scope): message`)
    then(commit succeeds)
      sothat(the first behavioral commit defines the PR's deliverable)
  when(mechanic commits with `feat(scope): message`)
    then(commit succeeds)
      sothat(the first behavioral commit defines the PR's deliverable)

# usecase.2 = continuation commit after first behavioral

given(branch already has a behavioral commit)
  when(mechanic commits with `fix(scope): message`)
    then(commit is BLOCKED)
    then(error message says: "branch already has a behavioral commit. use `cont:` or `cont(scope):` prefix")
      sothat(mechanic knows exactly what to do)
  when(mechanic commits with `feat(scope): message`)
    then(commit is BLOCKED)
    then(error message says: "branch already has a behavioral commit. use `cont:` or `cont(scope):` prefix")
      sothat(mechanic knows exactly what to do)
  when(mechanic commits with `cont(scope): message`)
    then(commit succeeds)
      sothat(continuation commits are clearly labeled)
  when(mechanic commits with `cont: message`)
    then(commit succeeds)
      sothat(scope is optional for continuation commits)

# usecase.3 = non-behavioral commits are exempt

given(branch already has a behavioral commit)
  when(mechanic commits with `chore: message`)
    then(commit succeeds)
      sothat(non-behavioral commits don't require `cont:` prefix)
  when(mechanic commits with `docs: message`)
    then(commit succeeds)
      sothat(non-behavioral commits don't require `cont:` prefix)

given(branch has no behavioral commits yet)
  when(mechanic commits with `chore: message`)
    then(commit succeeds)
    then(branch still has no behavioral commits)
      sothat(chore doesn't count as "first behavioral commit")
  when(mechanic commits with `chore: setup` then `fix(api): actual fix`)
    then(both commits succeed)
    then(the fix is the "first behavioral commit")
      sothat(chore before fix doesn't trigger cont: requirement)

# usecase.4 = error message clarity

given(branch has behavioral commit `fix(api): validate input`)
  when(mechanic attempts `fix(api): add tests`)
    then(commit is BLOCKED)
    then(error shows the first behavioral commit: `fix(api): validate input`)
    then(error suggests: use `cont:` or `cont(api):` prefix)
      sothat(mechanic has full context to fix the issue)

# usecase.5 = fail fast on invalid repo state (never failhide)

given(repo has no commits)
  when(mechanic attempts any commit)
    then(commit is BLOCKED)
    then(error says: "repo has no commits")
      sothat(mechanic knows the repo is not properly initialized)

given(repo has no base branch: main/master/trunk)
  when(mechanic attempts any commit)
    then(commit is BLOCKED)
    then(error says: "no base branch found")
      sothat(mechanic knows to create a proper branch structure)

given(mechanic is on the base branch: main/master/trunk)
  when(mechanic attempts any commit)
    then(commit is BLOCKED)
    then(error says: "cannot commit to base branch")
    then(error suggests: create a feature branch first)
      sothat(mechanic never commits directly to main)
