# playtest: declapract.upgrade

## prerequisites

- this repo checked out at branch `vlad/fix-declapract-upgrade`
- skill built: `npm run build`
- skill linked: `npx rhachet roles link --role mechanic`

## sandbox

all playtest operations occur within `.temp/playtest.declapract.upgrade/`.

create the sandbox:
```sh
mkdir -p .temp/playtest.declapract.upgrade
cd .temp/playtest.declapract.upgrade
```

create a mock declapract repo:
```sh
git init
git config user.email "test@test.com"
git config user.name "Test"
echo "declared:" > declapract.use.yml
echo "  - package: declapract-typescript-ehmpathy" >> declapract.use.yml
git add .
git commit -m "init"
git checkout -b test-branch
```

return to repo root between tests:
```sh
cd ../..
```

## happy paths

### 1. init creates route with turtle vibes

**action:**
```sh
cd .temp/playtest.declapract.upgrade
npx rhachet run --skill declapract.upgrade init
```

**expected outcome:**
- output shows turtle emoji and "radical!"
- output shows route path like `.route/v2026_03_16.declapract.upgrade/`
- output shows "hang ten!"

**pass criterion:**
- turtle vibes visible in output
- route directory created at `.route/v$DATE.declapract.upgrade/`

**verify:**
```sh
ls .route/ | grep declapract.upgrade
ls .route/*declapract.upgrade*/
```

- should list `.stone` and `.guard` files
- should NOT list any `.sh` files

### 3. exec starts upgrade (prereq check passes)

**action:**
```sh
cd .temp/playtest.declapract.upgrade
SKIP_PNPM_CHECK=1 npx rhachet run --skill declapract.upgrade exec
```

note: `SKIP_PNPM_CHECK=1` is used because the sandbox has no real pnpm. the test verifies the prereq check passes and the upgrade starts.

**expected outcome:**
- output shows "upgrade time" or similar start message
- skill proceeds past prereq checks

**pass criterion:**
- no "pnpm not found" error
- no "not a declapract repo" error
- upgrade flow starts (may fail later at fnm, but that's expected in sandbox)

## pass/fail criteria

### pass if:
- all happy paths produce expected output
- all edgey paths produce expected errors
- no `.sh` files present in route after init
- exec works standalone without init
- init is idempotent

### fail if:
- any `.sh` file appears in route
- exec requires init to run
- error messages are unclear or absent
- turtle vibes absent from init output
