## question 2: should `exec` require the route to exist?

**can this be answered via logic now?**
partially.

**rationale:**
- if `exec` works without a route, why have `init` at all?
- the route provides: progress tracker, workflow structure, reflection prompts
- however, some users might want a quick one-off upgrade without ceremony

**counterpoint:**
- the wisher might prefer simplicity: just run the upgrade, no route needed
- we could default to "require route" but allow `--force` to bypass

**verdict: [wisher]**
- this is a UX decision the wisher should confirm
- my recommendation: require route by default, fail with clear message

## question 3: should we add `--skip-init` flag to `exec`?

**can this be answered via logic now?**
no — this is the same question as #2 in different form.

**if we add --skip-init:**
- user can do `rhx declapract.upgrade exec --skip-init`
- skips route creation, runs upgrade directly
- loses structured workflow

**if we don't add --skip-init:**
- user must do `init` then `exec`
- structured workflow preserved
- slight ceremony cost

**verdict: [wisher]**
- recommend NOT to add --skip-init for v1
- the two-command dance is intentional
- can add later if users complain
