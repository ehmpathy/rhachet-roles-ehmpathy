## is this a fallback convention?

yes. the repo convention for skills is `.integration.test.ts`.

the `declapract.upgrade.integration.test.ts` file tests the full journey:
- case1: init happy path
- case2: init error path
- case3: exec happy path (mocked)
- case4: exec error path (no config)
- case5: exec error path (no pnpm)
- case6: init idempotency
- case7: exec standalone
- case8: invalid subcommand
- case9: absent subcommand

these are journey tests — they test the skill from user input to user output. but they use the repo's fallback convention (`.integration.test.ts`) rather than `.play.test.ts`.

## should I rename to `.play.`?

the guide says:
> "if not supported, is the fallback convention used?"

the fallback convention is supported and used consistently across the repo. a rename of this one file to `.play.` would break consistency with the other 14 skill tests.

I follow the repo convention, not introduce a new pattern for one behavior.

## conclusion
