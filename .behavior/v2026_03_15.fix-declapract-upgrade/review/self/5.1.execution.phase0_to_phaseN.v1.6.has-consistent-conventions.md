# self-review: has-consistent-conventions (round 6)

## security and idempotency conventions

### file copy safety convention

| skill | copy pattern |
|-------|--------------|
| init.sh | `cp -f "$file" "$ROUTE_PATH/"` with explicit whitelist |
| cpsafe.sh | validates paths within git repo |

**analysis:** init.sh uses explicit whitelist:
```bash
for file in "$TEMPLATES_DIR"/*.stone "$TEMPLATES_DIR"/*.guard; do
```

this pattern only copies .stone and .guard files. impossible to copy .sh files even if malicious actor adds them to templates.

**verdict: secure.** whitelist copy prevents accidental executable inclusion.

### mkdir idempotency convention

| skill | mkdir pattern |
|-------|---------------|
| init.sh | `mkdir -p "$ROUTE_PATH"` |
| cpsafe.sh | `mkdir -p "$(dirname "$DEST_PATH")"` |
| mvsafe.sh | `mkdir -p "$(dirname "$DEST_PATH")"` |

**verdict: consistent.** all skills use `mkdir -p` for idempotent directory creation.

### test environment skip convention

| file | skip pattern |
|------|--------------|
| init.sh | `if [[ "${SKIP_ROUTE_BIND:-}" != "1" ]]` |
| exec.sh | `if [[ -z "${SKIP_PNPM_CHECK:-}" ]]` |

**observation:** init.sh uses `!= "1"` check, exec.sh uses `-z` check.

**verdict: minor inconsistency, acceptable.** both achieve same goal (skip in test). `!= "1"` is more explicit about expected value.

### error suppression convention

| file | suppression pattern |
|------|---------------------|
| init.sh | `> /dev/null 2>&1 \|\| true` for route.bind.set |
| exec.sh | none (all errors propagate) |

**analysis:** init.sh suppresses route.bind.set errors because:
1. route bind is optional (route works without bind)
2. test environments may not have rhachet installed

exec.sh does not suppress errors because all operations are critical.

**verdict: intentional asymmetry.** correct for each context.
