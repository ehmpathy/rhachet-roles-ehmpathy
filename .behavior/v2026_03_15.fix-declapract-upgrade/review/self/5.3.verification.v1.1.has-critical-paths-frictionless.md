# self-review: has-critical-paths-frictionless (round 1)

## critical paths identified

from vision/criteria:

1. **init happy path**: `rhx declapract.upgrade init` creates route
2. **exec happy path**: `rhx declapract.upgrade exec` runs upgrade
3. **exec standalone**: exec works without init

## friction analysis

| path | steps | friction points | mitigated? |
|------|-------|-----------------|------------|
| init | 1 command | none | yes |
| exec | 1 command | requires pnpm, requires declapract.use.yml | yes (clear errors) |
| standalone exec | 1 command | none | yes |

## error path friction

| error | message clarity | actionable? |
|-------|-----------------|-------------|
| no declapract.use.yml | "not a declapract repo" | yes |
| no pnpm | "pnpm not found" | yes |
| invalid subcommand | shows usage | yes |
| no subcommand | shows usage | yes |

## conclusion

**holds because:** all critical paths are single-command. error paths provide clear, actionable messages.
