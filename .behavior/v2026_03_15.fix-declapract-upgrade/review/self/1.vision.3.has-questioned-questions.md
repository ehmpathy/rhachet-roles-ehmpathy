**final answer:**
- recommend: require route by default
- but the wisher should confirm, because they might prefer simplicity over workflow
- status stays [wisher]

## deeper reflection: why these questions matter

**the core tension:**
the wish wants simplicity and security. the route wants structure and ceremony.
these goals pull in opposite directions.

**simplicity says:** run one command, get upgraded.
**structure says:** run init, inspect route, run exec, follow stones.

**the vision tries to balance both:**
- security: exec logic stays in skill (not copied to route)
- simplicity: two commands instead of three
- structure: route preserved for workflow

**does this balance work?**

let me imagine a user's journey:

1. user runs `rhx declapract.upgrade init`
   - route created
   - user sees turtle vibes
   - user feels confident

2. user runs `rhx declapract.upgrade exec`
   - packages upgraded
   - declapract applied
   - user sees output

3. user runs `rhx route.drive`
   - guided to detect hazards
   - guided to repair test defects
   - guided to reflect

this is a good journey. the two-command dance (init + exec) is light, and the route provides the guardrails.

**what if a user skips step 1?**

current vision says: fail with clear message.
this is correct — explicit is better than implicit.

**what if a user never runs step 3?**

the upgrade still works. they just don't get the structured workflow.
this is acceptable — the route is opt-in ceremony, not required.

**conclusion:**
the vision's question triage is sound. the balance is appropriate.
the [wisher] questions are genuinely for the wisher to decide.
