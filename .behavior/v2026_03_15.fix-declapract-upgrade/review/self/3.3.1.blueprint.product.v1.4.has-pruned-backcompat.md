### re-examined: should we keep old command as alias?

**old command:** `rhx init.declapract.upgrade`
**new command:** `rhx declapract.upgrade init`

**did wisher request backcompat?** no — wish explicitly asks for new structure
**evidence from wish:**
> "call it `declapract.upgrade`"
> "invoked via `declapract.upgrade init`"

**should we add alias anyway?**
- adds maintenance burden (two entry points)
- no evidence of external callers
- this is an internal ehmpathy skill, not a public API
- clean break is simpler

**verdict:** no alias. intentional clean break.

**verdict:** removal is the explicit ask. no backcompat needed.
