## deeper assumption: two commands is better than one

we assumed users prefer `init` then `exec` over a single command.

**what if one command were better?**

```sh
# single command option
rhx declapract.upgrade
# → creates route AND runs upgrade
```

**why two commands is better:**
1. user can inspect the route before exec
2. user can abort after init if something looks wrong
3. user can re-run exec without re-init
4. separates setup from action (terraform pattern)

**why single command might be better:**
1. fewer keystrokes
2. less to remember
3. more "just do it"

**the wish explicitly asks for two commands.** we follow the wish.

**verdict: two commands, as requested** ✓
