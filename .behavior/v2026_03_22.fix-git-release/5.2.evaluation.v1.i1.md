# 5.2.evaluation.v1.i1

## summary (as implemented)

refactored `git.release` skill to emit uniform stdout across all 3 release transports (feature-branch, release-branch, release-tag) via reusable domain operations.

**key changes implemented:**
1. decomposed operations into separate `git.release._.{operation}.sh` files (6 files)
2. replaced `--to` flag with `--into` for destination clarity (fully removed, not aliased per wish line 219)
3. added `--apply` alias for `--mode apply`
4. created p3 journey tests with 215 snapshots
5. created .test/infra/ with test utilities (5 files)
6. updated spec documentation

### decomposed operations

all `git.release._.*.sh` files are source-only with no shebang. main `git.release.sh` sources them.

```
git.release._.get_one_goal_from_input.sh

│     ├─ input: current_branch, default_branch, flag_from, flag_into
│     ├─ output: from=, into= (to stdout)
│     └─ throws: exit 2 if from=main && into=main
└─ logic as declared in blueprint
```

#### git.release._.get_all_flags_from_input.sh
```
git.release._.get_all_flags_from_input.sh

│     ├─ input: argv
│     ├─ output: watch=, apply=, retry=, dirty= (to stdout)
│     └─ --apply implies watch=true
└─ logic as declared in blueprint
```

#### git.release._.get_one_transport_status.sh
```
git.release._.get_one_transport_status.sh
├─ [+] get_one_transport_status()
│     ├─ input: transport_type, transport_ref
│     └─ output: check_status=, automerge_status=, rebase_status=, title= (to stdout)
├─ [+] _get_pr_transport_status()   # adapter for PR transports
└─ [+] _get_tag_transport_status()  # adapter for tag transport
```

#### git.release._.emit_transport_status.sh
```
git.release._.emit_transport_status.sh
├─ [+] emit_transport_status()
│     ├─ input: transport_type, transport_ref, flag_apply, flag_retry, flag_watch, status_json
│     ├─ output: uniform 🌊 release: shape to stdout
│     └─ side effects: enable_automerge(), rerun_failed_workflows()
└─ sources output.sh for print_* functions
```

#### git.release._.emit_transport_watch.sh
```
git.release._.emit_transport_watch.sh
├─ [+] emit_transport_watch()
│     ├─ input: transport_type, transport_ref
│     ├─ output: poll cycles to stdout
│     ├─ polls: every 10s until terminal or 15min timeout
│     └─ supports mock sequences via GIT_RELEASE_MOCK_SEQUENCE
└─ sources output.sh for print_watch_* functions
```
