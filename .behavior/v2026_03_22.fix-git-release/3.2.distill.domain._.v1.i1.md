# 3.2.distill.domain._.v1.i1

## context

this is a bash skill. domain objects are represented as bash variables/functions that return structured data. domain operations are bash functions that transform or emit.

---

## domain objects

### goal (literal)

represents the inferred release goal from flags and branch context.

```bash
# shape
GOAL_FROM="feat" | "main" | "<branch-name>"
GOAL_INTO="main" | "prod"
```

| field | type | description |
|-------|------|-------------|
| from | feat \| main \| branch | source branch type |
| into | main \| prod | target destination |

constraints:
- from=main && into=main → ConstraintError

### flags (literal)

represents the parsed CLI flags.

```bash
# shape
FLAG_WATCH="true" | "false"
FLAG_APPLY="true" | "false"
FLAG_RETRY="true" | "false"
FLAG_DIRTY="block" | "allow"
```

| field | type | default | description |
|-------|------|---------|-------------|
| watch | bool | false | poll until completion |
| apply | bool | false | enable automerge (implies watch) |
| retry | bool | false | rerun failed workflows |
| dirty | block \| allow | block | dirty work dir behavior |

constraints:
- apply=true → watch=true (implied)

### transport (entity)

represents a release transport (PR or tag workflows).

```bash
# transport type
TRANSPORT_TYPE="feature-branch" | "release-branch" | "release-tag"

# transport identity
TRANSPORT_PR_NUMBER="123"  # for PR transports
TRANSPORT_TAG="v1.33.0"    # for tag transport
```

| field | type | description |
|-------|------|-------------|
| type | feature-branch \| release-branch \| release-tag | transport variant |
| pr_number | string \| null | PR number (for PR types) |
| tag | string \| null | tag name (for tag type) |

### transport_status (literal)

represents the current state of a transport.

```bash
# shape
STATUS_CHECK="unfound" | "inflight" | "passed" | "failed"
STATUS_AUTOMERGE="unfound" | "found" | "added" | "merged" | "n/a"
STATUS_REBASE="clean" | "behind" | "dirty"
```

| field | type | description |
|-------|------|-------------|
| check | unfound \| inflight \| passed \| failed | check status |
| automerge | unfound \| found \| added \| merged \| n/a | automerge state |
| rebase | clean \| behind \| dirty | rebase state |

note: release-tag transport has automerge=n/a (no PR).

### get_one_transport_status

```bash
# contract
get_one_transport_status() {
  # input: $TRANSPORT_TYPE, $TRANSPORT_PR_NUMBER or $TRANSPORT_TAG
  # output: sets STATUS_CHECK, STATUS_AUTOMERGE, STATUS_REBASE
  # calls: gh pr view, gh run list (via adapters)
}
```

adapters:
- `_get_pr_transport_status()` — for feature-branch, release-branch
- `_get_tag_transport_status()` — for release-tag

### emit_transport_status

```bash
# contract
emit_transport_status() {
  # input: $TRANSPORT_TYPE, STATUS_*, FLAG_APPLY, FLAG_RETRY
  # output: prints uniform status tree to stdout
  # side effects: if FLAG_APPLY, enables automerge; if FLAG_RETRY, triggers rerun
}
```

output format (uniform across all transports):
```
🌊 release: {title}
   ├─ {check status}
   ├─ {automerge status}
   └─ {hints}
```

### emit_transport_watch

```bash
# contract
emit_transport_watch() {
  # input: $TRANSPORT_TYPE, $TRANSPORT_PR_NUMBER or $TRANSPORT_TAG
  # output: prints watch progress to stdout
  # polls: until terminal state or 15min timeout
  # guarantees: at least 3 poll cycles visible when inflight
}
```

output format:
```
🥥 let's watch
   ├─ 💤 N left, Xs in action, Xs watched
   ├─ 💤 N left, Xs in action, Xs watched
   └─ {terminal: 👌 passed | ⚓ failed | ⏰ timeout}
```

### emit_one_transport_status_exitcode

```bash
# contract
emit_one_transport_status_exitcode() {
  # input: $STATUS_CHECK
  # output: exits process with appropriate code
  # exit 0: passed, merged
  # exit 2: unfound, inflight, failed (constraint error)
}
```

## relationships

### composition hierarchy

```
git.release (main flow)
├── get_one_goal_from_input
├── get_all_flags_from_input
│
├── [if goal.from=feat]
│   ├── emit_transport_status(feature-branch)
│   ├── emit_transport_watch(feature-branch) [if flags.watch]
│   └── emit_one_transport_status_exitcode [if not merged, stop]
│
├── [if goal.into=prod OR goal.from=main]
│   ├── emit_transport_status(release-branch)
│   ├── emit_transport_watch(release-branch) [if flags.watch]
│   └── emit_one_transport_status_exitcode [if not merged, stop]
│
└── [if goal.into=prod OR goal.from=main]
    ├── emit_transport_status(release-tag)
    ├── emit_transport_watch(release-tag) [if flags.watch]
    └── emit_one_transport_status_exitcode
```

### adapter pattern

```
emit_transport_status
├── uses get_one_transport_status
│   ├── _get_pr_transport_status (feature-branch, release-branch)
│   └── _get_tag_transport_status (release-tag)
└── uses uniform output functions
    ├── print_release_header
    ├── print_check_status
    ├── print_automerge_status
    └── print_hints
```

### dependency flow

```
goal ← get_one_goal_from_input ← (branch, --from, --into)
flags ← get_all_flags_from_input ← (argv)
status ← get_one_transport_status ← (transport)
output ← emit_transport_status ← (transport, status, flags)
watch ← emit_transport_watch ← (transport)
exit ← emit_one_transport_status_exitcode ← (status)
```

## how domain supports wish

| wish requirement | domain solution |
|------------------|-----------------|
| consistent stdout | emit_transport_status uses uniform output functions |
| reusable operations | each operation has single responsibility |
| 3+ poll cycles | emit_transport_watch guarantees minimum polls |
| --to → --into | get_all_flags_from_input parses new flag |
| --apply alias | get_all_flags_from_input maps --apply to apply=true |
| test composition | each operation testable in isolation |
