## matrix.2: transport status display

| ind: check state | ind: automerge state | dep: check stdout | dep: automerge stdout |
|------------------|----------------------|-------------------|----------------------|
| unfound | n/a | 🫧 no open branch pr | (n/a) |
| inflight | unfound | 🐢 N check(s) in progress | 🌴 automerge unfound |
| inflight | found | 🐢 N check(s) in progress | 🌴 automerge enabled [found] |

## decomposition analysis

the matrices show clean orthogonal dimensions:

1. **flag inference** — branch × from × into (3 dims) ✓
2. **transport status** — check state × automerge state (2 dims) ✓
3. **watch behavior** — initial × final (2 dims) ✓
4. **mode combinations** — watch × apply × retry (3 dims) ✓
5. **multi-transport flow** — from × into (2 dims) ✓

no matrix exceeds 3 independent dimensions. the design decomposes well.

**key insight**: the wish mentioned 7×7×7 = 343 combinations for --from feat --into prod. but via decomposition:
- transport status (11 rows) + watch behavior (6 rows) + mode combinations (8 rows) = 25 atomic rows
- these compose into the 343 combinations without need to enumerate each one
