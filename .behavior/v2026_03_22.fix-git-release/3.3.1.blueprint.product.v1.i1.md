# 3.3.1.blueprint.product.v1.i1

## summary

refactor `git.release` skill to emit uniform stdout across all 3 release transports (feature-branch, release-branch, release-tag) via reusable domain operations.

**key changes:**
1. decompose operations into separate `git.release._.{operation}.sh` files
2. cover each operation with dedicated integration tests
3. replace `--to` flag with `--into` for destination clarity
4. add `--apply` alias for `--mode apply`
5. ensure at least 3 watch poll cycles are visible in tests
6. create p3 journey tests for full combinations per wish (215 snapshots)

## codepath tree

### decomposed operations

**note:** all `git.release._.*.sh` files are source-only — they define functions but have no shebang and are NOT directly invocable. the main `git.release.sh` sources them and calls the functions. tests invoke them by source + function call, not direct execution.

#### git.release._.get_one_goal_from_input.sh

```
git.release._.get_one_goal_from_input.sh
├─ input: CURRENT_BRANCH, FLAG_FROM, FLAG_INTO
├─ output: GOAL_FROM, GOAL_INTO (to stdout as key=value)
├─ throws: exit 2 (ConstraintError) if from=main && into=main
└─ logic:
   ├─ if FLAG_FROM omitted && on main → GOAL_FROM=main
   ├─ if FLAG_FROM omitted && on feat → GOAL_FROM=feat
   ├─ if FLAG_INTO omitted && GOAL_FROM=main → GOAL_INTO=prod
   ├─ if FLAG_INTO omitted && GOAL_FROM=feat → GOAL_INTO=main
   └─ if GOAL_FROM=main && GOAL_INTO=main → exit 2
```

#### git.release._.get_all_flags_from_input.sh

```
git.release._.get_all_flags_from_input.sh
├─ input: argv (passed as arguments)
├─ output: FLAG_WATCH, FLAG_APPLY, FLAG_RETRY, FLAG_DIRTY (to stdout as key=value)
├─ logic:
│  ├─ parse --watch → FLAG_WATCH=true
│  ├─ parse --mode apply OR --apply → FLAG_APPLY=true, FLAG_WATCH=true
│  ├─ parse --retry → FLAG_RETRY=true
│  ├─ parse --dirty allow|block → FLAG_DIRTY=allow|block
│  └─ parse --into, --from → pass through (handled by goal)
└─ defaults: FLAG_WATCH=false, FLAG_APPLY=false, FLAG_RETRY=false, FLAG_DIRTY=block
```

#### git.release._.get_one_transport_status.sh

```
git.release._.get_one_transport_status.sh
├─ input: TRANSPORT_TYPE, TRANSPORT_REF (PR number or tag name)
├─ output: STATUS_CHECK, STATUS_AUTOMERGE, STATUS_REBASE (to stdout as key=value)
├─ adapters:
│  ├─ _get_pr_transport_status()   # for feature-branch, release-branch
│  └─ _get_tag_transport_status()  # for release-tag (workflow runs)
└─ return values:
   ├─ STATUS_CHECK: unfound | inflight | passed | failed | merged
   ├─ STATUS_AUTOMERGE: unfound | found | added | merged | n/a
   └─ STATUS_REBASE: needed | clean
```

#### git.release._.emit_transport_status.sh

```
git.release._.emit_transport_status.sh
├─ input: TRANSPORT_TYPE, TRANSPORT_REF, TRANSPORT_TITLE, STATUS_*, FLAG_APPLY, FLAG_RETRY
├─ output: stdout with uniform 🌊 release: shape
├─ side effects:
│  ├─ if FLAG_APPLY && STATUS_AUTOMERGE=unfound → enable_automerge()
│  └─ if FLAG_RETRY && STATUS_CHECK=failed → rerun_failed_workflows()
└─ sources: output.sh for print_* functions
```

#### git.release._.emit_transport_watch.sh

```
git.release._.emit_transport_watch.sh
├─ input: TRANSPORT_TYPE, TRANSPORT_REF
├─ output: stdout with poll cycles (at least 3 when inflight)
├─ polls: every 10s until terminal state or 15min timeout
├─ terminal states: passed, failed, merged, timeout
└─ sources: output.sh for print_watch_* functions
```

#### git.release._.emit_one_transport_status_exitcode.sh

```
git.release._.emit_one_transport_status_exitcode.sh
├─ input: STATUS_CHECK
├─ output: exits process with semantic code
└─ exit codes:
   ├─ exit 0: passed, merged
   ├─ exit 1: malfunction (gh failed, network error)
   └─ exit 2: constraint (unfound, inflight, failed, needs action)
```

## state tree (sequential dependency)

**see:** `refs/state-tree.exact.md` for row-by-row enumeration with exact counts.

the three entities are processed in sequence, not in parallel. you can only reach entity N+1 if entity N is merged. this eliminates the cross product (not 8×8×4=256, but 7+7+4=18 leaf nodes).

```
feat PR (8 states)
├─ unfound       → exit 2 (release PR never exists)
├─ inflight      → watch → exit per result (release PR never exists)
├─ passed:wout   → exit 0 or apply (release PR never exists)
├─ passed:with   → watch → merge → continue to release PR
├─ failed        → exit 2 (release PR never exists)
├─ rebase:behind → exit 2 (release PR never exists)
├─ rebase:dirty  → exit 2 (release PR never exists)
└─ merged        → release PR (8 states)
                   ├─ unfound       → exit 0 (no release PR found)
                   ├─ inflight      → watch → exit per result
                   ├─ passed:wout   → exit 0 or apply
                   ├─ passed:with   → watch → merge → continue to tags
                   ├─ failed        → exit 2
                   ├─ rebase:behind → exit 2
                   ├─ rebase:dirty  → exit 2
                   └─ merged        → tags (4 states)
                                      ├─ unfound  → exit 0 (no tag workflows)
                                      ├─ inflight → watch → exit per result
                                      ├─ passed   → exit 0
                                      └─ failed   → exit 2
```

**leaf count: 7 + 7 + 4 = 18 state rows** (for feat → prod flow)

this tree must be declared in `git.release.spec.tree.md` next to the tests.

## contracts

### emit_transport_status output shape

```
🌊 release: {title}
   ├─ {check_status}
   ├─ {automerge_status}
   └─ {hint or watch}
```

where:
- `{check_status}` = `👌 all checks passed` | `🐢 N check(s) in progress` | `⚓ N check(s) failed`
- `{automerge_status}` = `🌴 automerge unfound` | `🌴 automerge enabled [found]` | `🌴 automerge enabled [added]` | `🌴 already merged`

### emit_transport_watch output shape

```
🥥 let's watch
   ├─ 💤 N left, Xs in action, Xs watched
   ├─ 💤 N left, Xs in action, Xs watched
   ├─ 💤 N left, Xs in action, Xs watched
   └─ {terminal}
```

where:
- `{terminal}` = `✨ done!` | `⚓ N check(s) failed` | `⏰ timeout`

1. **phase 0**: update flags (`--to` → `--into`, add `--apply` alias)
2. **phase 1**: extract uniform output functions to output.sh
3. **phase 2**: extract domain operations to git.release.operations.sh
4. **phase 3**: refactor main flow to use new operations
5. **phase 4**: update extant tests for new flags
6. **phase 5**: add journey tests with snapshot coverage
7. **phase 6**: update spec documentation

## risks and mitigations

| risk | mitigation |
|------|------------|
| break extant users of `--to` | add deprecation alias that maps `--to` → `--into` |
| watch tests flaky | use SEQUENCE mock with explicit poll arrays |
| snapshot drift | use time placeholders (`Xs in action`) |
