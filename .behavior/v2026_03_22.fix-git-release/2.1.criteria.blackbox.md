# 2.1.criteria.blackbox: git.release

## usecase.1 = flag inference

the system infers `--from` and `--into` based on current branch and explicit flags.

### scene.1: on feat branch, no flags
```
given('on feature branch turtle/add-surfboards')
  given('--from is omitted')
    given('--into is omitted')
      when('rhx git.release')
        then('infers --from feat --into main')
          sothat('feature branches default to main target')
```

### scene.2: on feat branch, --into prod
```
given('on feature branch turtle/add-surfboards')
  given('--from is omitted')
    given('--into prod')
      when('rhx git.release --into prod')
        then('infers --from feat --into prod')
          sothat('can release feature all the way to prod')
```

### scene.3: on feat branch, --from main
```
given('on feature branch turtle/add-surfboards')
  given('--from main')
    given('--into is omitted')
      when('rhx git.release --from main')
        then('infers --from main --into prod')
          sothat('can skip feature branch and release from main')
```

### scene.4: --from main --into main (invalid)
```
given('any branch')
  given('--from main')
    given('--into main')
      when('rhx git.release --from main --into main')
        then('ConstraintError: cannot merge main into main')
          sothat('prevents nonsensical self-merge')
```

### scene.5: on main, no flags
```
given('on main branch')
  given('--from is omitted')
    given('--into is omitted')
      when('rhx git.release')
        then('infers --from main --into prod')
          sothat('main branch defaults to prod release')
```

### scene.6: on main, --from main
```
given('on main branch')
  given('--from main')
    given('--into is omitted')
      when('rhx git.release --from main')
        then('infers --from main --into prod')
          sothat('explicit --from main is redundant but valid')
```

### scene.7: on main, --from feat-branch
```
given('on main branch')
  given('--from turtle/add-surfboards')
    given('--into is omitted')
      when('rhx git.release --from turtle/add-surfboards')
        then('infers --from turtle/add-surfboards --into main')
          sothat('can release a specific feature branch from main')
```

## usecase.2 = uniform transport status

each transport shows identical stdout structure regardless of type.

### transport status: unfound
```
given('transport has no open PR')
  when('status is checked')
    then('shows 🫧 no open branch pr')
    then('shows branch name')
    then('shows hint to push')
```

### transport status: inflight, wout-automerge
```
given('transport PR has checks in progress')
  given('automerge is not enabled')
    when('status is checked')
      then('shows 🌊 release: {title}')
      then('shows 🐢 N check(s) in progress')
      then('shows 🌴 automerge unfound (use --apply to add)')
```

### transport status: passed, wout-automerge
```
given('transport PR has all checks passed')
  given('automerge is not enabled')
    when('status is checked')
      then('shows 🌊 release: {title}')
      then('shows 👌 all checks passed')
      then('shows 🌴 automerge unfound (use --apply to add)')
      then('shows hint: use --apply to enable automerge and watch')
```

### transport status: failed, wout-automerge
```
given('transport PR has failed checks')
  given('automerge is not enabled')
    when('status is checked')
      then('shows 🌊 release: {title}')
      then('shows ⚓ N check(s) failed')
      then('shows failed check details with links')
      then('shows 🌴 automerge unfound (use --apply to add)')
      then('shows hint: use --retry to rerun failed workflows')
```

### transport status: inflight, with-automerge
```
given('transport PR has checks in progress')
```

### transport status: passed, with-automerge
```
given('transport PR has all checks passed')

```

### transport status: failed, with-automerge
```
given('transport PR has failed checks')

    when('status is checked')
      then('shows 🌊 release: {title}')
      then('shows ⚓ N check(s) failed')

      then('shows 🌴 automerge enabled [found]')
```

## usecase.3 = watch behavior

watch polls until completion or timeout.

### watch: inflight to passed
```
given('transport checks are in progress')
  when('--watch flag is provided')
    then('shows 🥥 lets watch')
    then('shows at least 3 poll cycles')
    then('each poll shows 💤 N left, Xs in action, Xs watched')
    then('final poll shows 👌 all checks passed')
      sothat('user sees progress over time')
```

### watch: inflight to failed
```
given('transport checks are in progress')
  given('a check will fail')
    when('--watch flag is provided')
      then('shows 🥥 lets watch')
      then('shows poll cycles until failure')
      then('final poll shows ⚓ N check(s) failed')
      then('shows failed check details with links')
        sothat('user sees which check failed')
```

### watch: timeout
```
given('transport checks are in progress')
  given('checks do not complete within 15 minutes')
    when('--watch flag is provided')
      then('shows poll cycles for 15 minutes')
      then('exits with error')
      then('shows partial progress')
        sothat('user knows what state was reached')
```

### watch: already passed
```
given('transport checks already passed')
  when('--watch flag is provided')
    then('shows 👌 all checks passed')
    then('does not poll')
      sothat('no wasted time on completed work')
```

### watch: already merged
```
given('transport is already merged')
  when('--watch flag is provided')
    then('shows 🌴 already merged')
    then('proceeds to next transport if applicable')
```

## usecase.4 = apply behavior

apply enables automerge and watches.

### apply: automerge not found
```
given('transport PR does not have automerge')
  when('--apply flag is provided')
    then('enables automerge on the PR')
    then('shows 🌴 automerge enabled [added]')
    then('watches until completion')
      sothat('single command drives to merge')
```

### apply: automerge already found
```
given('transport PR already has automerge')
  when('--apply flag is provided')
    then('does not modify automerge')
    then('shows 🌴 automerge enabled [found]')
    then('watches until completion')
      sothat('idempotent behavior')
```

### apply: merged immediately
```
given('transport PR passes checks and merges')
  when('--apply flag is provided')
    then('shows 🌴 automerge enabled [added] -> and merged already')
    then('shows ✨ done!')
      sothat('user knows merge completed')
```

## usecase.5 = retry behavior

retry reruns failed workflows.

### retry: failed checks
```
given('transport has failed checks')
  when('--retry flag is provided')
    then('triggers rerun of failed workflows')
    then('shows 👌 rerun triggered for each failed check')
    then('shows hint: use --watch to monitor rerun progress')
      sothat('user can fix transient failures')
```

### retry: no failed checks
```
given('transport has no failed checks')
  when('--retry flag is provided')
    then('shows status as normal')
    then('does not trigger any reruns')
      sothat('no wasted CI cycles')
```

### retry + apply: combined
```
given('transport has failed checks')
  when('--retry --apply flags are provided')
    then('triggers rerun of failed workflows first')
    then('watches until checks pass or fail')
    then('if passed, enables automerge')
    then('continues to next transport if applicable')
      sothat('single command recovers and releases')
```

## usecase.6 = multi-transport flow

--into prod chains through multiple transports.

### flow: feat to prod (all succeed)
```
given('on feature branch')
  given('--into prod')
    given('all transports will pass')
      when('rhx git.release --into prod --apply')
        then('shows feature-branch transport status')
        then('enables automerge, watches to merge')
        then('shows 🫧 and then...')
        then('shows release-branch transport status')
        then('enables automerge, watches to merge')
        then('shows 🫧 and then...')
        then('shows release-tag transport status')
        then('watches tag workflows to completion')
        then('exits 0')
          sothat('full release in one command')
```

### flow: feature fails, stops early
```
given('on feature branch')
  given('--into prod')
    given('feature-branch checks will fail')
      when('rhx git.release --into prod --apply')
        then('shows feature-branch transport status')
        then('watches to failure')
        then('exits non-zero')
        then('does not proceed to release-branch')
          sothat('fail-fast prevents partial releases')
```

### flow: from main to prod
```
given('--from main')
  given('--into prod')
    when('rhx git.release --from main --into prod --apply')
      then('skips feature-branch transport')
      then('shows release-branch transport status')
      then('enables automerge, watches to merge')
      then('shows release-tag transport status')
      then('watches tag workflows to completion')
        sothat('can release from main directly')
```

## usecase.7 = edge cases

### needs rebase
```
given('transport PR needs rebase')
  when('status is checked')
    then('shows 🐚 needs rebase')
    then('shows hint: rhx git.branch.rebase begin')
      sothat('user knows how to proceed')
```

### multiple release PRs (ambiguous)
```
given('multiple open release PRs for same version')
  when('status is checked')
    then('ConstraintError: ambiguous release PRs')
      sothat('user must clean up before proceed')
```

### dirty work directory
```
given('uncommitted changes in work directory')
  given('--dirty is not allow')
    when('rhx git.release')
      then('fails fast with error')
      then('shows hint about uncommitted changes')
        sothat('prevents accidental release of incomplete work')
```

### dirty allowed
```
given('uncommitted changes in work directory')
  given('--dirty allow')
    when('rhx git.release --dirty allow')
      then('proceeds normally')
        sothat('user can override when intentional')
```

## usecase.8 = alias behavior

### --apply alias
```
given('--apply flag')
  when('rhx git.release --apply')
    then('behaves same as --mode apply')
      sothat('shorter flag for common action')
```

```
given('--into flag')
  when('rhx git.release --into prod')
    then('targets prod release')
      sothat('clearer destination semantics')
```

## summary: experience boundaries

| input | experience |
|-------|------------|
| no flags on feat | show feature-branch status to main |
| no flags on main | show release-branch + tag status to prod |
| --watch | poll until complete, show cycles |
| --apply | enable automerge, watch to merge |
| --retry | rerun failed checks |
| --into prod | chain through all transports |
| --from main | skip feature-branch |
| --from <branch> | release specific branch |

| transport state | stdout |
|-----------------|--------|
| unfound | 🫧 no open branch pr |
| inflight | 🐢 N check(s) in progress |
| passed | 👌 all checks passed |
| failed | ⚓ N check(s) failed |
| merged | 🌴 already merged |

| automerge state | stdout |
|-----------------|--------|
| unfound | 🌴 automerge unfound |
| found | 🌴 automerge enabled [found] |
| added | 🌴 automerge enabled [added] |
