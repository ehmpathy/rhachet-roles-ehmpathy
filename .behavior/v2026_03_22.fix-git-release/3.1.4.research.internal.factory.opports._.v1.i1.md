## purpose

identify THE bottleneck that, if fixed, would unblock the most future work.

---

## the constraint

**singular bottleneck: watch behavior mock setup**

the wish demands "at least 3 poll cycles" in watch tests [0.wish.md:142-143]. current SEQUENCE mock pattern requires manual array construction for each watch scenario:

```typescript
'pr view': 'SEQUENCE:' + JSON.stringify([
  inflight1, inflight2, inflight3, passed  // manual 4-element array
])
```

this is tedious and error-prone for the 6 watch scenarios × 3 transports = 18 watch test cases.

---

## flow and queues

no external queues. tests run locally, fully mocked. no CI wait.

---

## waste

### repeated mock construction

each watch test manually constructs the poll sequence. a reusable generator would eliminate this repetition:

```typescript
// current: manual
'pr view': 'SEQUENCE:' + JSON.stringify([inflight, inflight, inflight, passed])

// opportunity: generator
'pr view': genWatchSequence({ polls: 3, terminal: 'passed' })
```

### no other significant waste

- setupTestEnv is reusable
- cleanup is automatic
- no handoffs
