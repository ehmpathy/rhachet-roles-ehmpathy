wish =

folks call `--from main` too often when they shouldnt, which breaks expectations and halts releases

when someone calls `--from main --into prod` without a really good reason, typiclaly they just want to release `--into prod` and dont realize how to do it correctly

the issue with `--from main` is that if after `--into main` runs and merges their pr, if the please-release github action hasnt created their release pr yet (takes ~30s), then `--from main` will target the prior release pr - the old one not related to their current one

---

so, we should

1. make sure the briefs and skill headers make it super clear that --into prod is the default, --from main should only be used in special circumstances

2. if someone's on a branch and runs --from main and they're not human, then they're required to articulate --why they need to and cant do just --into prod (i.e., throw a constraint error and ask them can they just do --into prod? its safer and if they just want ot release their current pr, then that's actually what they should do. if not, explain why they cant use it via --why)


---

that way, we keep folks in a pit of success
