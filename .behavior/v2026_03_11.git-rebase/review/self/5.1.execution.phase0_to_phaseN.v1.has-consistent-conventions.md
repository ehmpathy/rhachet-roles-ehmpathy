## directory variable names

| skill | variable | line |
|-------|----------|------|
| git.commit.push.sh | *_DIR | 24 |
| git.commit.set.sh | *_DIR | 33 |
| git.branch.rebase.begin.sh | SKILL_DIR | 21 |
| git.branch.rebase.continue.sh | SKILL_DIR | 19 |

**finding**: git.commit uses a generic `*_DIR` pattern, git.branch.rebase uses `SKILL_DIR`.

**analysis**:
- extant pattern = generic "where this executable lives"
- new pattern = specific "where this skill lives"

both are semantically correct. SKILL_DIR is more domain-aligned since these are skills. however, this diverges from extant convention.

**decision**: the divergence is acceptable because:
1. var is local, not exported
2. no external consumers rely on the name
3. both names are clear and self-documenting

if strict consistency is required, I can rename. flagging for awareness.

## robot identity

| skill | how identity is set | why |
|-------|---------------------|-----|
| git.commit.set.sh | ROBOT_NAME, ROBOT_EMAIL as local vars, passed to git commit --author | git commit accepts --author flag |
| git.commit.push.sh | ROBOT_NAME as local var for comparison | check if HEAD author matches |
| git.branch.rebase.continue.sh | export GIT_COMMITTER_NAME/EMAIL | git rebase --continue reads env vars |

**finding**: different mechanisms for same identity.

**analysis**: git commit accepts `--author` flag. git rebase reads `GIT_COMMITTER_*` env vars. the different mechanisms are required by git's design, not a convention choice.

**verdict**: different mechanisms are intentional and required.

## argument parsing conventions

**git.commit.push.sh** (lines 47-60):
- does not handle --skill/--repo/--role

**git.branch.rebase.begin.sh** (lines 33-56):
- explicitly handles and ignores --skill/--repo/--role

**finding**: git.branch.rebase explicitly ignores rhachet args, git.commit does not.

**analysis**: git.commit skills predate the rhachet passthrough pattern. git.branch.rebase is newer and handles them explicitly. this is an improvement, not a divergence.

**verdict**: acceptable difference. git.branch.rebase pattern is more robust.
