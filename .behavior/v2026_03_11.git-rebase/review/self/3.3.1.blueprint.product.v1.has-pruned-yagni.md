## default branch detection (added in review)

| question | answer |
|----------|--------|
| explicitly requested? | no |
| why added? | "main" is hardcoded; fails on repos that use "master" |
| is it YAGNI? | **questionable** |

**analysis:**

the wish says "rebase onto main". it does not say "rebase onto main or master".

**option 1: keep detection (current)**
- pro: works on both "main" and "master" repos
- pro: more robust
- con: not explicitly requested
- con: adds complexity

**option 2: revert to hardcoded "main"**
- pro: simpler
- pro: matches wish exactly
- con: fails on "master" repos
- con: user must rename branch or update skill

**open question for wisher:**
> should git.rebase support repos that use "master" as default branch?
> or should we require "main" and fail fast on "master" repos?

**temporary verdict:** keep (robustness > simplicity here) but flag for wisher review
