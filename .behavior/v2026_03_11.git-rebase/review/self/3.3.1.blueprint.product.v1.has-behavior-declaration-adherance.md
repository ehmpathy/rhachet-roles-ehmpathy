### continue output deviation analysis

**vision shows (lines 122-127):**
```
🐚 git.rebase continue
   ├─ rebase
   │  ├─ target: origin/main
   │  ├─ commits: 3 ✓
   │  └─ status: complete
```

**implementation shows:**
```
🐚 git.rebase continue
   ├─ rebase
   │  ├─ commits: 3 ✓
   │  └─ status: complete
```

**why omit target?**
- continue.sh runs AFTER begin.sh
- continue.sh does not know which branch begin.sh targeted
- to store target in state file would add complexity
- user already saw target in begin.sh output

**verdict:** intentional simplification, documented. acceptable deviation.
