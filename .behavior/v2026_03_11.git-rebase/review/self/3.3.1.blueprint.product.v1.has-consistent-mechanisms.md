### default branch detection

| question | answer |
|----------|--------|
| does codebase have this? | yes - git.commit.operations.sh line 32-37 |
| do we duplicate? | partially |

**comparison:**

git.commit.operations.sh (line 32-37):
```bash
for candidate in origin/main origin/master origin/trunk main master trunk; do
  if git rev-parse --verify "$candidate" >/dev/null 2>&1; then
    base_branch="$candidate"
    break
  fi
done
```

git.rebase.begin.sh (current):
```bash
if git rev-parse --verify origin/main &>/dev/null; then
  DEFAULT_BRANCH="main"
elif git rev-parse --verify origin/master &>/dev/null; then
  DEFAULT_BRANCH="master"
else
  # error
fi
```

**difference:**
- git.commit checks 6 variants including trunk
- git.rebase only checks main and master
- git.rebase returns branch NAME ("main"), git.commit returns full REF ("origin/main")

**options:**
1. align with git.commit pattern (add trunk)
2. keep simpler pattern (main/master only)

**decision:** add origin/trunk for consistency

### default branch detection — intentional divergence

**git.commit.operations.sh checks:**
- origin/main, origin/master, origin/trunk
- main, master, trunk (local fallbacks)

**git.rebase.begin.sh checks:**
- origin/main, origin/master, origin/trunk only

**why different?**
- git.commit: finds closest ancestor for stacked branches, may include local branches
- git.rebase: explicitly targets remote for `git pull origin $BRANCH --rebase`

**verdict:** intentional divergence — rebase targets remote, commit handles more scenarios

### argument parse pattern

**git.commit.set.sh (lines 112-168):**
```bash
while [[ $# -gt 0 ]]; do
  case $1 in
    --message|-m)
      ...
      shift 2
      ;;
    --push)
      DO_PUSH=true
      shift
      ;;
    --repo|--role|--skill)
      # rhachet passthrough args - ignore
      shift 2
      ;;
    --)
      shift
      ;;
    --*)
      echo "error: unknown option: $1"
      ...
      exit 2
      ;;
    *)
      shift
      ;;
  esac
done
```

**git.rebase.begin.sh (lines 33-56):**
```bash
while [[ $# -gt 0 ]]; do
  case $1 in
    # rhachet passes these - ignore them
    --skill|--repo|--role)
      shift 2
      ;;
    --mode)
      MODE="$2"
      shift 2
      ;;
    --help|-h)
      ...
      exit 0
      ;;
    *)

      ;;

```

**patterns aligned:**
- both use while + case
- both handle --skill|--repo|--role from rhachet
- both handle --help|-h

**patterns diverged:**
- git.commit handles `--` separator
- git.commit has --* catch-all before * catch-all
- error messages use different words

**verdict:** core pattern consistent, minor variations acceptable
