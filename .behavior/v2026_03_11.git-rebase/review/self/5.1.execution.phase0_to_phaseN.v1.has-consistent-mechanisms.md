## key question: could I have reused git.commit.operations.sh?

**extant mechanism**: `get_closest_ancestor_branch()` (lines 23-88)
- detects base branch (main/master/trunk)
- returns "ON_BASE" if current branch is the base branch
- handles stacked branches (finds closest ancestor)

**my mechanism**: `is_base_branch()` (lines 128-135)
- checks if current branch is main/master
- simple boolean return

**why I chose not to reuse**:

1. **trunk support mismatch**: git.commit.operations.sh line 32 includes trunk. the git.rebase blueprint explicitly says "main/master" only. if I sourced that file, I'd inherit trunk support that was not requested.

2. **stacked branch complexity**: `get_closest_ancestor_branch()` is 65 lines and handles stacked branches. rebase only needs to know "am I on main/master?" — a 7-line function. the simpler mechanism fits the simpler requirement.

3. **fail-fast signal mismatch**: `get_closest_ancestor_branch()` returns strings like "NO_COMMITS", "NO_BASE", "ON_BASE". my guards need booleans. I'd need wrapper logic.

**decision**: intentionally created simpler mechanism that matches blueprint scope.

## mechanism comparison

### 1. push permission check

**new**: `is_push_allowed()` in git.rebase.operations.sh
```bash
is_push_allowed() {
  local meter_file=".meter/git.commit.uses.jsonc"
  if [[ ! -f "$meter_file" ]]; then
    return 1
  fi
  local push_status
  push_status=$(jq -r '.push // "block"' "$meter_file" 2>/dev/null)
  if [[ "$push_status" == "allow" ]]; then
    return 0
  fi
  return 1
}
```

**extant**: git.commit.push.sh lines 125-137 (inline)
```bash
PUSH_ALLOWED=""
if [[ -f "$STATE_FILE" ]]; then
  PUSH_ALLOWED=$(jq -r '.push' "$STATE_FILE")
fi
if [[ "$PUSH_ALLOWED" != "allow" ]]; then
  emit_error "push not allowed"
  exit 2
fi
```

**verdict**: **no duplication** — I extracted common logic into a reusable function. the extant code does inline reads. my function is a DRY improvement, not duplication.

### 4. dirty worktree check

**new**: `is_worktree_dirty()` in git.rebase.operations.sh
```bash
is_worktree_dirty() {
  local changes
  changes=$(git diff --name-only 2>/dev/null)
  if [[ -n "$changes" ]]; then
    return 0
  fi
  return 1
}
```

**extant**: no similar function found in git.commit skills

**verdict**: **new functionality** — not duplication, this is rebase-specific guard logic
