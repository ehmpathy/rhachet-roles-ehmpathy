| in progress | "rebase already in progress" | yes — continue or abort |

### idempotency analysis

**why git.rebase is inherently non-idempotent:**
```
initial state: HEAD at commit A
after git rebase: HEAD at commit A' (different hash)
if run again: undefined (rebase not in progress)
```

**how blueprint handles this:**
1. begin.sh detects if rebase in progress → guides to continue/abort
2. continue.sh detects if rebase idle → rejects
3. abort.sh detects if rebase idle → rejects

**this is the correct approach:** instead of attempts to force idempotency on a non-idempotent operation, we detect state and guide appropriately.
