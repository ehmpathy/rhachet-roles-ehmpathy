### edgecases (1.vision.md:222-231)

```markdown
| edgecase | how we handle it |
|----------|------------------|
| no push permission | reject: "rebase requires push permission" |
| main/master branch | reject: "cannot rebase main/master" |
| dirty work tree | reject: "unstaged changes exist" |
| no commits to rebase | report: "branch is up to date, no rebase needed" |
| network unavailable for fetch | report fetch error, suggest retry |
| rebase already in progress | detect and guide to continue/abort |
| conflicts in rebase | mechanic sees them, fixes them, continues |
| force push to shared branch | use `--force-with-lease` (safer) |
```

## gap analysis

### network unavailable (1.vision.md:228)

**status:** not tested in playtest

**rationale:**
- transient error condition
- requires network mock or disconnection
- error message is simple: "fetch failed, try again"
- no ux complexity to verify byhand
- automated integration test verifies error path

**disposition:** acceptable gap

### force-with-lease (1.vision.md:231)

**status:** not explicitly tested

**rationale:**
- this is behavior of `git push` after rebase
- covered by git.commit.push skill
- playtest focuses on rebase skills, not push

**disposition:** out of scope for this playtest
