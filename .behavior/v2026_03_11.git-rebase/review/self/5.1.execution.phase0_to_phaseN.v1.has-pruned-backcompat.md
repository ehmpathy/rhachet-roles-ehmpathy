## backwards compatibility decisions found

### 1. main/master support for rebase target (lines 105-116 in begin.sh)

**location**: git.branch.rebase.begin.sh lines 105-116

```bash
if git rev-parse --verify origin/main &>/dev/null; then
  DEFAULT_BRANCH="main"
elif git rev-parse --verify origin/master &>/dev/null; then
  DEFAULT_BRANCH="master"
elif git rev-parse --verify origin/trunk &>/dev/null; then
  DEFAULT_BRANCH="trunk"
```

**was main/master requested?**:
- blueprint says "rebase onto origin/main" in summary
- vision says "origin/main" in stdout demos
- BUT: blueprint codepath tree line 44 says "reject if main/master" — implying both exist

**was trunk requested?**: no mention of trunk anywhere in vision, criteria, or blueprint

**verdict**:
- main/master: **keep** — blueprint implies both exist ("reject if main/master")
- trunk: **REMOVE** — not requested, YAGNI

### 2. main/master/trunk support in guard (lines 128-135 in operations.sh)

**location**: git.branch.rebase.operations.sh lines 128-135

```bash
is_base_branch() {
  local branch
  branch=$(get_current_branch)
  if [[ "$branch" == "main" || "$branch" == "master" || "$branch" == "trunk" ]]; then
    return 0
  fi
  return 1
}
```

**was main/master requested?**: yes — blueprint codepath tree says "reject if main/master"

**was trunk requested?**: no

**verdict**:
- main/master: **keep** — explicitly requested
- trunk: **REMOVE** — not requested, YAGNI
