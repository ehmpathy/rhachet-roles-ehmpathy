### pitofsuccess.errors

**rule: require-fail-fast**
- all guards use early exit pattern

example from continue.sh (lines 61-77):
```bash
# guard: conflicts must be settled
CONFLICT_FILES=$(get_conflict_files)
if [[ -n "$CONFLICT_FILES" ]]; then
  print_turtle_header "hold up dude..."
  print_tree_start "git.rebase continue"
  echo "   ├─ error: conflicts not settled"
  echo "   └─ files"
  ...
  exit 1
fi
```

pattern: check condition, print error context, exit 1. no fallthrough.
- **verdict**: compliant

**rule: forbid-failhide**
- no try/catch patterns in shell
- `set -euo pipefail` at top of each file (line 14-17 in all skills)
- errors propagate via exit codes
- no `2>/dev/null` that hides meaningful errors

example from operations.sh (line 161):
```bash
push_status=$(jq -r '.push // "block"' "$meter_file" 2>/dev/null)
```

the `2>/dev/null` here is acceptable - it silences jq parse errors but returns "block" as fallback. fail-safe, not failhide.
- **verdict**: compliant

### readable.narrative

**rule: require-narrative-flow**
- guards use early return pattern (exit 1)
- no nested if/else structures
- code flows top to bottom

example from begin.sh (lines 63-71):
```bash
# guard: push permission required
if ! is_push_allowed; then
  print_turtle_header "hold up dude..."
  print_tree_start "git.rebase begin"
  echo "   └─ error: rebase requires push permission"
  ...
  exit 1
fi
```

each guard exits early. no nested logic.
- **verdict**: compliant

**rule: forbid-else-branches**
ran grep for "else" in all skill files:

```
$ grep -n "else" begin.sh continue.sh abort.sh operations.sh
begin.sh:107:elif git rev-parse --verify origin/master &>/dev/null; then
```

found one `elif` for main/master detection. this is condition branching, not else-branch anti-pattern. it's the shell equivalent of:
```bash
if main_exists then main
elif master_exists then master
else error
```

this is acceptable - it's selecting between mutually exclusive conditions, not nested else logic.

all other logic uses if-then-exit pattern.
- **verdict**: compliant
