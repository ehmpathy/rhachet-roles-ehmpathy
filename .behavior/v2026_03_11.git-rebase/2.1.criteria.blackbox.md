# blackbox criteria: git.rebase skills

## usecase.1 = rebase onto main

### episode: plan then apply

```
given a feature branch with commits ahead of origin/main
  given push permission is granted
    when git.rebase begin (plan mode)
      then show preview of commits to rebase
      then show how many commits behind main
      then show meter status
        sothat mechanic sees what will happen before apply

    when git.rebase begin --mode apply
      given no conflicts
        then rebase completes
        then show "righteous!" with status clean
        then show "ready to push" prompt
          sothat mechanic knows next step

      given conflicts occur
        then show "hold up dude..." with conflict status
        then show which commit caused conflict
        then show which files have conflicts
        then show "settle conflicts, then run git.rebase continue"
          sothat mechanic knows what to do next
```

### episode: conflict settlement

```
given a rebase in progress with conflicts
  when mechanic edits files to settle conflicts
    when mechanic stages settled files
      when git.rebase continue
        given more commits remain
          then show settled files with checkmarks
          then show how many commits left
            sothat mechanic sees progress

        given all commits done
          then show "righteous!" with status complete
          then show "ready to push" prompt
            sothat mechanic can proceed to push
```

### episode: abort

```
given a rebase in progress
  when git.rebase abort
    then restore branch to pre-rebase state
    then show "no worries dude" with restored status
      sothat mechanic can try again or take different approach
```

## usecase.2 = permission enforcement

```
given no push permission granted
  when git.rebase begin
    then reject with "rebase requires push permission"
      sothat mechanic knows to ask human for permission

given push permission granted
  when git.rebase begin
    then allow rebase to proceed
      sothat mechanic can rebase autonomously
```

## usecase.3 = edgecase guards

### guard: cannot rebase main

```
given current branch is main or master
  when git.rebase begin
    then reject with "cannot rebase main/master"
      sothat base branch is protected
```

### guard: dirty work tree

```
given unstaged changes exist
  when git.rebase begin
    then reject with "unstaged changes exist"
      sothat work is not lost
```

### guard: already up to date

```
given branch has no commits ahead of main
  when git.rebase begin
    then report "branch is up to date, no rebase needed"
      sothat mechanic knows no action required
```

### guard: rebase already in progress

```
given a rebase is already in progress
  when git.rebase begin
    then detect in-progress state
    then guide to continue or abort
      sothat mechanic recovers from partial state
```

### guard: network unavailable

```
given network is unavailable
  when git.rebase begin --mode apply
    then report fetch error
    then suggest retry
      sothat mechanic knows issue is transient
```
