# research: internal product code patterns (test)

## summary

the git.commit test patterns provide all infrastructure for git.branch.rebase tests. key patterns to reuse: genTempDir with git setup, spawnSync execution, bdd structure, and fixture utils.

```typescript

  symlink: [{ at: 'node_modules', to: 'node_modules' }],
```

## pattern.8: commit creation for push tests

**source**: `src/domain.roles/mechanic/skills/git.commit/git.commit.push.integration.test.ts` [8]

```typescript
// create commits
if (args.commits) {
  const author = args.commitAuthor ?? {
    name: 'Test Human',
    email: 'human@test.com',
  };
  for (const [i, message] of args.commits.entries()) {
    const fileName = `file-${i}.txt`;
    fs.writeFileSync(path.join(tempDir, fileName), `content ${i}`);
    spawnSync('git', ['add', fileName], { cwd: tempDir });
    spawnSync(
      'git',
      [
        'commit',
        `--author=${author.name} <${author.email}>`,
        '-m',
        message,
      ],
      { cwd: tempDir },
    );
  }
}
```

**relation to wish**: git.branch.rebase tests need commits on feature branch to rebase

**decision**: [REUSE] — same commit creation pattern
