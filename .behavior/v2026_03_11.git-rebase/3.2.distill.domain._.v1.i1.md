# distill: domain model for git.rebase

## summary

git.rebase is a shell skill, not a typescript domain library. the "domain" here is the git rebase workflow state and operations. this document maps the conceptual domain to shell operations.

---

## usecases and contract

### usecase.1: rebase onto main

```bash
# plan mode - preview what will happen
rhx git.rebase begin                    # → preview commits, behind count, meter

# apply mode - execute the rebase
rhx git.rebase begin --mode apply       # → rebase runs
```

### usecase.2: continue after conflict

```bash
# after mechanic settles conflicts and stages files
rhx git.rebase continue                 # → continue or complete
```

### usecase.3: abort a stuck rebase

```bash
rhx git.rebase abort                    # → restore to pre-rebase state
```

## domain objects (conceptual)

### literals (immutable values)

| name | shape | description |
|------|-------|-------------|
| CommitRef | `{ hash: string; header: string }` | a commit to rebase |
| ConflictFile | `{ path: string }` | a file in conflict |
| MeterState | `{ uses: number; push: 'allow' \| 'block' }` | permission state |

### entities (stateful)

| name | unique | description |
|------|--------|-------------|
| RebaseState | singleton per repo | tracks in-progress vs idle |
| WorkTree | singleton per repo | tracks clean vs dirty |
| Branch | name | tracks current branch |

---

## domain operations

### getters (pure queries)

| operation | input | output | description |
|-----------|-------|--------|-------------|
| getRebaseState | cwd | 'idle' \| 'in_progress' | check .git/rebase-merge |
| getConflictFiles | cwd | ConflictFile[] | git diff --name-only --diff-filter=U |
| getCommitsToRebase | cwd | CommitRef[] | git log --oneline origin/main..HEAD |
| getBehindCount | cwd | number | git rev-list --count HEAD..origin/main |
| getCurrentBranch | cwd | string | git rev-parse --abbrev-ref HEAD |
| getWorkTreeState | cwd | 'clean' \| 'dirty' | git diff --name-only |
| getMeterPushPermission | cwd | 'allow' \| 'block' | read .meter/git.commit.uses.jsonc |

### setters (state mutations)

| operation | input | effect | description |
|-----------|-------|--------|-------------|
| setRebaseBegin | mode | starts rebase | git pull origin main --rebase |
| setRebaseContinue | — | continues rebase | git rebase --continue |
| setRebaseAbort | — | aborts rebase | git rebase --abort |

---

## relationships

```
MeterState
    │
    └── gates ── RebaseBegin
                      │
                      ├── on success ── ready to push
                      │
                      └── on conflict ── RebaseState (in_progress)
                                              │
                                              ├── RebaseContinue ── loop until done
                                              │
                                              └── RebaseAbort ── restore to idle
```

## shell operations map

| conceptual operation | shell implementation |
|---------------------|---------------------|
| getRebaseState | `[[ -d .git/rebase-merge ]] \|\| [[ -d .git/rebase-apply ]]` |
| getConflictFiles | `git diff --name-only --diff-filter=U` |
| getCommitsToRebase | `git log --oneline origin/main..HEAD` |
| getBehindCount | `git rev-list --count HEAD..origin/main` |
| getCurrentBranch | `git rev-parse --abbrev-ref HEAD` |
| getWorkTreeState | `git diff --name-only` returns non-empty = dirty |
| getMeterPushPermission | `jq -r '.push // "block"' .meter/git.commit.uses.jsonc` |
| setRebaseBegin | `git pull origin main --rebase` |
| setRebaseContinue | `git rebase --continue` |
| setRebaseAbort | `git rebase --abort` |

## composition for wish fulfillment

### git.rebase begin

```
1. getMeterPushPermission → if block, reject
2. getCurrentBranch → if main/master, reject
3. getWorkTreeState → if dirty, reject
4. getCommitsToRebase → if empty, report "up to date"
5. getRebaseState → if in_progress, guide to continue/abort
6. if mode == plan: preview commits, behind count, meter
7. if mode == apply: setRebaseBegin
   - on success: "righteous! ready to push"
   - on conflict: show conflict files, guide to continue
```

### git.rebase continue

```
1. getRebaseState → if idle, reject "no rebase in progress"
2. getConflictFiles → if non-empty, reject "conflicts not settled"
3. setRebaseContinue
   - if more commits: show progress, left count
   - if done: "righteous! ready to push"
```

### git.rebase abort

```
1. getRebaseState → if idle, reject "no rebase in progress"
2. setRebaseAbort → "no worries dude, restored"
```
