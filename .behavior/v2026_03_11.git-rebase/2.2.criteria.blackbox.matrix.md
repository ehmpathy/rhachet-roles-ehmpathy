## coverage analysis

### dimensions identified

1. **push permission**: none, granted
2. **branch type**: main/master, feature
3. **work tree state**: clean, dirty
4. **commits ahead**: yes, no
5. **mode**: plan, apply
6. **conflicts**: none, yes
7. **rebase state**: not in progress, in progress
8. **conflicts settled**: yes, no
9. **commits left**: yes, no

### decomposition

the matrices above are well-scoped:
- matrix.1 covers **begin preconditions** (4 dimensions)
- matrix.2 covers **apply execution** (1 dimension)
- matrix.3 covers **continue states** (3 dimensions)
- matrix.4 covers **abort states** (1 dimension)

no decomposition needed — each matrix has ≤4 independent dimensions.

### gaps found

none. all meaningful combinations are covered:
- permission checks before any action
- branch protection for main/master
- work tree guards before rebase
- conflict handle after apply
- progress track in continue
- clean abort from any in-progress state
