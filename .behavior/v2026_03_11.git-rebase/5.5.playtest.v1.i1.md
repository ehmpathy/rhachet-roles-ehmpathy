# playtest: git.branch.rebase skills

## prerequisites

- repo with git.commit skills operational
- feature branch with commits ahead of main
- push permission granted via `git.commit.uses set --quant N --push allow`
- npm dependencies installed (`npm ci` or `pnpm install --frozen-lockfile`)
- skills built and linked (`npm run build && npx rhachet roles link --role mechanic`)

---

## happy paths

### 1. plan mode preview

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin
```

**expected outcome:**
- output shows turtle vibes header
- tree structure with:
  - target: origin/main
  - commits: list of commits to rebase
  - behind: how many commits main is ahead
  - meter: push permission status
- ends with "run with --mode apply to execute"

**pass if:** tree output renders with all expected branches
**fail if:** errors, absent data, or no output

---

### 2. apply mode success (no conflicts)

**setup:**
create a branch that will rebase cleanly:
```bash
git checkout main
git pull origin main
git checkout -b test-rebase-clean
echo "clean test" > testfile.txt
git add testfile.txt
git commit -m "feat(test): clean rebase test"
```

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin --mode apply
```

**expected outcome:**
- output shows "righteous!" turtle phrase
- status: clean
- shows "ready to push" guidance

**pass if:** rebase completes without conflict, output matches vision format
**fail if:** errors or unexpected conflict

---

### 3. continue after conflict settlement

**setup:**
create a conflict scenario:
```bash
git checkout main
git pull origin main
# create file with conflict potential on main
echo "main content" > conflict.txt
git add conflict.txt
git commit -m "feat: main version"
git push origin main

# create feature branch with conflict
git checkout -b test-rebase-conflict HEAD~1
echo "feature content" > conflict.txt
git add conflict.txt
git commit -m "feat: feature version"
```

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin --mode apply
```

**expected outcome:**
- output shows "hold up dude..." turtle phrase
- shows conflict status
- lists conflict files
- shows guidance to settle and continue

**then settle conflict:**
```bash
# edit conflict.txt to settle
echo "settled content" > conflict.txt
git add conflict.txt
```

**action:**
```bash
npx rhachet run --skill git.branch.rebase.continue
```

**expected outcome:**
- output shows "righteous!" turtle phrase
- status: complete
- shows "ready to push" guidance

**pass if:** full conflict flow works end-to-end
**fail if:** errors at any step, unclear guidance

---

### 4. abort rebase

**setup:**
start a conflicted rebase (from step 3)

**action:**
```bash
npx rhachet run --skill git.branch.rebase.abort
```

**expected outcome:**
- output shows "no worries dude" turtle phrase
- status: restored to pre-rebase state
- branch returns to original state

**pass if:** branch restored, output matches vision
**fail if:** errors or incomplete restore

---

## edgey paths

### 5. no push permission

**setup:**
```bash
npx rhachet run --skill git.commit.uses set --quant 0
```

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin
```

**expected outcome:**
- rejects with error about push permission required

**pass if:** clear error message about permission
**fail if:** proceeds without permission or unclear error

---

### 6. on main branch

**action:**
```bash
git checkout main
npx rhachet run --skill git.branch.rebase.begin
```

**expected outcome:**
- rejects with error about not rebase main/master

**pass if:** clear rejection message
**fail if:** allows rebase on main

---

### 7. dirty work tree

**setup:**
```bash
echo "uncommitted" > dirty.txt
```

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin
```

**expected outcome:**
- rejects with error about unstaged changes

**pass if:** clear error about dirty work tree
**fail if:** proceeds with dirty work tree

---

### 8. already up to date

**setup:**
create branch from latest main with no divergence:
```bash
git checkout main
git pull origin main
git checkout -b test-uptodate
```

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin
```

**expected outcome:**
- reports branch is up to date, no rebase needed

**pass if:** clear message about no rebase needed
**fail if:** attempts rebase or errors

---

### 9. rebase already in progress

**setup:**
start a conflicted rebase but do not abort or continue

**action:**
```bash
npx rhachet run --skill git.branch.rebase.begin
```

**expected outcome:**
- detects in-progress state
- guides to continue or abort

**pass if:** clear guidance about extant rebase
**fail if:** attempts new rebase or unclear error

---

## pass/fail criteria

### overall pass

- all happy path scenarios work as documented
- all edge cases produce clear, helpful error messages
- turtle vibes output format matches vision demos
- no crashes or unclear errors

### overall fail

- any happy path fails
- edge cases do not guard properly (e.g., allows rebase on main)
- output format does not match vision
- unclear or absent error messages
