# blueprint: product for git.branch.rebase

## summary

build three shell skills for git rebase workflow:
1. `git.branch.rebase.begin.sh` — start a rebase onto origin/main (plan/apply mode)
2. `git.branch.rebase.continue.sh` — continue after conflict settlement
3. `git.branch.rebase.abort.sh` — abort and restore pre-rebase state

plus shared utils:
- `git.branch.rebase.operations.sh` — shared shell operations for rebase state detection

## codepath tree

### git.branch.rebase.begin.sh

```
git.branch.rebase.begin
├── [←] source output.sh (from git.commit)
├── [←] source git.branch.rebase.operations.sh
├── [+] check_push_permission
│   └── [←] read .meter/git.commit.uses.jsonc
├── [+] guard_branch
│   └── reject if main/master
├── [+] guard_worktree
│   └── reject if dirty (unstaged changes)
├── [+] guard_rebase_state
│   └── if in_progress, guide to continue/abort
├── [+] get_commits_to_rebase
│   └── git log --oneline origin/main..HEAD
├── [+] get_behind_count
│   └── git rev-list --count HEAD..origin/main
├── [+] plan_mode
│   ├── fetch origin main
│   ├── show preview (commits, behind, meter)
│   └── exit with "run with --mode apply"
└── [+] apply_mode
    ├── git pull origin main --rebase
    ├── on success: "righteous! ready to push"
    └── on conflict: show conflict files, guide to continue
```

### git.branch.rebase.continue.sh

```
git.branch.rebase.continue
├── [←] source output.sh (from git.commit)
├── [←] source git.branch.rebase.operations.sh
├── [+] guard_rebase_state
│   └── reject if idle
├── [+] guard_conflicts_settled
│   └── reject if conflicts still present
├── [+] set_seaturtle_identity
│   └── export GIT_COMMITTER as seaturtle[bot] (preserves original author)
├── [+] execute_continue
│   └── git rebase --continue
├── [+] check_more_commits
│   └── if more: show progress
│   └── if done: "righteous! ready to push"
└── [+] on_conflict
    └── show conflict files, guide to fix
```

### git.branch.rebase.abort.sh

```
git.branch.rebase.abort
├── [←] source output.sh (from git.commit)
├── [←] source git.branch.rebase.operations.sh
├── [+] guard_rebase_state
│   └── reject if idle
└── [+] execute_abort
    ├── git rebase --abort
    └── "no worries dude, restored"
```

### git.branch.rebase.operations.sh

```
git.branch.rebase.operations
├── [+] is_rebase_in_progress
│   └── check .git/rebase-merge or .git/rebase-apply
├── [+] get_conflict_files
│   └── git diff --name-only --diff-filter=U
├── [+] get_commits_left
│   └── read from .git/rebase-merge/end - done count
└── [+] get_current_commit
    └── read from .git/rebase-merge/msgnum
```

## test coverage

### integration tests

| test file | coverage |
|-----------|----------|
| git.branch.rebase.begin.integration.test.ts | plan mode, apply mode, permission guard, branch guard, dirty guard, up-to-date, conflict detection |
| git.branch.rebase.continue.integration.test.ts | not-in-progress guard, conflicts-not-settled guard, progress report, completion |
| git.branch.rebase.abort.integration.test.ts | not-in-progress guard, successful abort |

### test scenarios per skill

**git.branch.rebase.begin.sh:**
- [case1] plan mode shows preview
- [case2] apply mode succeeds
- [case3] apply mode with conflicts
- [case4] no push permission
- [case5] on main branch
- [case6] dirty work tree
- [case7] already up to date
- [case8] rebase already in progress

**git.branch.rebase.continue.sh:**
- [case1] not in progress
- [case2] conflicts not settled
- [case3] continue with more commits left
- [case4] continue to completion

**git.branch.rebase.abort.sh:**
- [case1] not in progress
- [case2] successful abort
