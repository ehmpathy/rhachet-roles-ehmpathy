● nice sets process priority (how much CPU time it gets relative to others).                                                 
                                                                                                                             
  ┌────────────┬──────────┬────────────────────────────┐                                                                     
  │ nice value │ priority │          meaning           │                                                                     
  ├────────────┼──────────┼────────────────────────────┤                                                                     
  │ -20        │ highest  │ hogs CPU, starves others   │
  ├────────────┼──────────┼────────────────────────────┤
  │ 0          │ normal   │ default                    │                                                                     
  ├────────────┼──────────┼────────────────────────────┤
  │ +19        │ lowest   │ only runs when CPU is idle │                                                                     
  └────────────┴──────────┴────────────────────────────┘                                                                     
   
  nice -n 10 npm test    # run tests at lower priority                                                                       
  nice -n 19 npm test    # run tests only when nothing else needs CPU                                                        
   
  It doesn't limit CPU — it just makes the process yield to others. Your acceptance tests would still use 12 cores, but only 
  when nothing else wants them.
                                                                                                                             
  For hard limits, you'd use cpulimit or --maxWorkers.
