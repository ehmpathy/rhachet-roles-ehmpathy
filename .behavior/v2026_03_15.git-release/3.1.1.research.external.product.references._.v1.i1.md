## key patterns from references

### from gh cli manual
- `gh pr merge --auto --squash` enables automerge with squash strategy
- `gh pr view --json statusCheckRollup` retrieves check status
- `gh run list --branch` filters workflow runs by branch/tag
- `gh run rerun --failed` reruns only failed jobs with dependencies
- `GH_TOKEN` environment variable for auth

### from terraform docs
- plan-then-apply pattern for safe operations
- plan shows what would happen without side effects
- apply executes the plan

### from cli design guides
- show progress for long operations
- use tree structure for hierarchical data
- provide actionable hints on errors

### from internal sources
- retry with exponential backoff (3 attempts, 5s delay)
- poll 5s for first 60s, then 15s
- 5-minute watch timeout
- keyrack for token management
