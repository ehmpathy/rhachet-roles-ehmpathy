# playtest: git.release skill

## prerequisites

- authenticated `gh` CLI (`gh auth status` should show logged in)
- keyrack unlocked (`rhx keyrack unlock --owner ehmpath --prikey ~/.ssh/ehmpath --env all`)
- a feature branch with an open PR (or ability to create one)
- clean worktree (no uncommitted changes)

## sandbox

the playtest operates on the current repo. no file changes are made — all operations are read-only in plan mode. apply mode modifies GitHub state (automerge, workflow reruns) but not local files.

---

## happy paths

### path 1: preview feature branch PR status

**setup:** checkout a feature branch with an open PR

```sh
git checkout turtle/your-feature-branch
```

**action:**

```sh
npx rhachet run --skill git.release -- --to main
```

**expected outcome:**

```
🐢 heres the wave...

🐚 git.release --to main --mode plan

🌊 release: turtle/your-feature-branch → main
   ├─ 👌 all checks passed
   ├─ 🌴 automerge unfound (use --apply to add)
   └─ hint: use --mode apply to enable automerge and watch
```

**pass criteria:**
- shows turtle vibe header
- shows PR status with check results
- shows hint to use `--mode apply`

## edgey paths

### edge 1: no PR for branch

**setup:** checkout a branch without an open PR

**action:**

```sh
npx rhachet run --skill git.release -- --to main
```

**expected:**

```
🐢 crickets...

🫧  no open branch pr
   ├─ turtle/your-branch
   └─ hint: use git.commit.push to push and findsert PR
```

**pass criteria:** shows crickets vibe, hints to push

---

### edge 2: failed CI checks

**setup:** PR with failed checks

**action:**

```sh
npx rhachet run --skill git.release -- --to main --mode apply
```

**expected:**

```
🐢 bummer dude...

🌊 release: turtle/your-branch → main
   ├─ ⚓  N check(s) failed
```

**pass criteria:** shows bummer vibe, surfaces failures

---

### edge 3: keyrack locked

**setup:** lock keyrack (or use a fresh session)

**action:**

```sh
npx rhachet run --skill git.release -- --to main --mode apply
```

**expected:**

```
🐢 bummer dude...

🔐 github token not found
   ├─ run: rhx keyrack unlock --owner ehmpath --prikey ~/.ssh/ehmpath --env all
   └─ then retry this command
```

**pass criteria:** shows unlock instructions

---

### edge 4: invalid arguments

**action:**

```sh
npx rhachet run --skill git.release -- --to invalid
```

**expected:** error message, exit code 2

```sh
npx rhachet run --skill git.release
```

**expected:** error about `--to is required`, exit code 2

**pass criteria:** clear error messages, exit 2

---

### edge 5: branch needs rebase

**setup:** PR that is behind main branch (main has commits not in your branch)

**action:**

```sh
npx rhachet run --skill git.release -- --to main
```

**expected:**

```
🐢 hold up...

🌊 release: turtle/your-branch → main
   ├─ 🐚 needs rebase
   └─ hint: rebase onto main before release
```

**pass criteria:** shows hold up vibe, indicates rebase needed

---

### edge 6: retry failed workflows

**setup:** PR with failed CI checks (same as edge 2)

**action:**

```sh
npx rhachet run --skill git.release -- --to main --mode apply --retry
```

**expected:**

```
🐢 lets try again...

🌊 release: turtle/your-branch → main
   ├─ 🔄 rerun triggered for 2 workflow(s)
   └─ 🥥 let's watch
```

**pass criteria:** shows rerun confirmation, then watches

## pass/fail criteria

### pass if

- ✅ path 1 shows PR status with check results
- ✅ path 2 enables automerge and shows watch progress
- ✅ path 3 shows release PR or latest tag status
- ✅ path 4 merges release PR and watches tag workflows
- ✅ edge 1-4 show appropriate vibes (crickets, bummer, hold up)
- ✅ edge 5 shows needs rebase with hint
- ✅ edge 6 shows rerun confirmation for --retry
- ✅ invalid args return exit code 2 with clear errors

### fail if

- ❌ any path shows unhandled errors or stack traces
- ❌ skill hangs without progress output
- ❌ automerge fails to enable (check GitHub PR page)
- ❌ output is cryptic or lacks turtle vibes
