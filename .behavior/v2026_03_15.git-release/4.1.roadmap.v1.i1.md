### checklist

- [ ] **2.1** create `git.release.sh` skeleton
  - add shebang and set -euo pipefail
  - add SKILL_DIR detection
  - source output.sh and operations.sh
- [ ] **2.2** implement argument parse
  - parse `--to main|prod` (required)
  - parse `--mode plan|apply` (default: plan)
  - parse `--retry` (optional flag)
  - ignore rhachet passthrough args (--repo, --role, --skill)
- [ ] **2.3** implement guards
  - validate --to value (main or prod)
  - validate --mode value (plan or apply)
  - ensure in git repo
  - detect tty for output mode
- [ ] **2.4** implement keyrack token fetch (apply mode only)
  - fetch EHMPATHY_SEATURTLE_PROD_GITHUB_TOKEN
  - fallback to ehmpath unlock
  - fail-fast if token empty
- [ ] **2.5** implement `--to main` dispatch
  - find PR for current branch
  - show PR status (checks, automerge)
  - plan mode: show hint to use --mode apply
  - apply mode: enable automerge, start watch
- [ ] **2.6** implement `--to prod` dispatch
  - find release PR (chore(release): pattern)
  - show release PR status
  - plan mode: show what would happen
  - apply mode: enable automerge, watch tag workflows
- [ ] **2.7** implement watch loop
  - poll every 5s for first 60s
  - poll every 15s after 60s
  - timeout at 5 minutes
  - exit on all checks pass or fail
- [ ] **2.8** implement error output
  - show failed checks with links
  - show hint to use --retry
- [ ] **2.9** implement retry behavior
  - `gh run rerun --failed` for each failed workflow
