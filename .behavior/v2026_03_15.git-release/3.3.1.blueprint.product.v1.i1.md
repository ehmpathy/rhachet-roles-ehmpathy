# 3.3.1.blueprint.product: git.release skill

## summary

build a `git.release` skill that wraps the extant `git release` alias from `~/.bash_aliases`. the skill provides:

- `git.release --to main` — merge feature branch PR to main, watch CI
- `git.release --to prod` — full release cycle: merge → watch test CI → tag → watch deploy/publish CI
- both with `--mode plan|apply` pattern (plan is default, apply executes)
- `--retry` flag to rerun failed workflows

the skill reuses:
- keyrack token fetch from git.commit.push
- plan/apply pattern from sedreplace
- turtle vibe output from output.sh
- tree output structure from git.commit skills

---

## extant alias base

the skill wraps the extant `git release` alias from `~/.bash_aliases` (function `git_alias_release`, lines 384-814). the alias is the source of truth for all behavior.

### key functions to reimplement

| alias function | purpose | skill location |
|----------------|---------|----------------|
| `_gh_with_retry` | retry gh commands on transient network errors (3x, 5s delay) | git.release.operations.sh |
| `_git_release_this` | handle feature branch release to main | git.release.sh dispatch |
| `_git_release_main` | handle main branch (show tag status) | git.release.sh dispatch |
| `_git_release_pr` | check PR status, enable automerge via `gh pr merge --auto --squash` | git.release.operations.sh |
| `_git_release_watch` | poll CI status until complete or 5-minute timeout | git.release.sh watch loop |
| `_git_release_report_failed_checks` | show failures with links and retry hint | output.sh |
| `_git_release_tag_runs` | watch tag workflows (publish.yml or deploy.yml) | git.release.sh --to prod |

### poll intervals from alias

```
first 60s: poll every 5s
after 60s: poll every 15s
timeout: 5 minutes total
```

### gh commands from alias

| command | purpose |
|---------|---------|
| `gh pr list --head "$branch" --state open --json number` | find PR for branch |
| `gh pr view "$pr" --json statusCheckRollup,autoMergeRequest,mergeStateStatus` | get PR status |
| `gh pr merge "$pr" --auto --squash` | enable automerge |
| `gh run list --branch "$tag" --json name,conclusion,status,url` | list tag workflows |
| `gh run view "$run" --json jobs` | get failed job details |
| `gh run rerun "$run" --failed` | retry failed workflows |

## contracts

### input contract

```
git.release --to <target> [--mode plan|apply] [--retry]

--to main       merge feature branch PR to main, watch test CI
--to prod       full release: merge → watch test CI → tag → watch deploy/publish

--mode plan     show what would happen (default)
--mode apply    execute the release

--retry         rerun failed workflows before watch
```

**vibe headers** (per mode/outcome):

| vibe | header | when |
|------|--------|------|
| plan | "heres the wave..." | plan mode |
| apply success | "cowabunga!" | apply mode, checks pass |
| apply fail | "bummer dude..." | apply mode, checks fail |
| no PR | "crickets..." | no PR found |
| needs rebase | "hold up..." | PR needs rebase |
| prod release | "radical!" | --to prod apply |

|-------|-------|

### mock strategy: PATH injection

tests mock `gh` CLI via PATH injection. the mock executable returns canned responses per gh subcommand:

```typescript
import { spawnSync } from 'child_process';
import * as fs from 'fs';
import * as path from 'path';

// create fake bin directory
const fakeBinDir = path.join(tempDir, '.fakebin');
fs.mkdirSync(fakeBinDir, { recursive: true });

// write mock gh executable
fs.writeFileSync(
  path.join(fakeBinDir, 'gh'),
  `#!/bin/bash
# mock gh CLI for test
case "$1 $2" in
  "pr list")
    echo '[{"number": 42, "headRefName": "turtle/feature-x"}]'
    ;;
  "pr view")
    echo '{"statusCheckRollup": [{"conclusion": "SUCCESS"}], "autoMergeRequest": null}'
    ;;
  "pr merge")
    echo 'auto-merge enabled'
    ;;
  "run list")
    echo '[{"name": "test", "conclusion": "success", "status": "completed"}]'
    ;;
  "run rerun")
    echo 'rerun triggered'
    ;;
  *)
    echo "mock: unhandled gh $*" >&2
    exit 1
    ;;
esac
`,
);
fs.chmodSync(path.join(fakeBinDir, 'gh'), '755');

// run skill with mock gh in PATH
const result = spawnSync('bash', [skillPath, '--to', 'main'], {
  cwd: tempDir,
  env: { ...process.env, PATH: `${fakeBinDir}:${process.env.PATH}`, TERM: 'dumb' },
});

// snapshot the output
expect(result.stdout).toMatchSnapshot();
expect(result.status).toEqual(0);
```
