# blueprint: product

## summary

extend `git.commit.uses` skill with:
1. **`block` / `allow` commands** — symmetric pair for local and global quota control
2. **`--global` flag** — targets global blocker at `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/`
3. **global blocker check** in `git.commit.set` — rejects commits when global blocker active

## codepath tree

### git.commit.uses.sh (dispatcher)

```
git.commit.uses.sh
├─ parse --global or --local flag (default: --local)
├─ if --global → exec git.commit.uses.global.sh "$@"
└─ else → exec git.commit.uses.local.sh "$@"
```

**note**: `--local` is implicit (backwards compatible). explicit `--local` flag accepted but optional.

git.commit.uses.global.sh

### git.commit.set.sh

```
git.commit.set.sh
├─ [+] check global blocker (before local quota)
│  ├─ read global blocker file
│  │  ├─ if file absent → not blocked, continue
│  │  ├─ if file is invalid json → BLOCKED (fail safe)
│  │  │  ├─ print_turtle_header "bummer dude..."
│  │  │  ├─ print_tree_error "global blocker file corrupt"
│  │  │  ├─ print_instruction "check the file:" "  $ cat $GLOBAL_METER_FILE"
│  │  │  └─ exit 2
│  │  └─ if { blocked: true }
│  │     ├─ print_turtle_header "bummer dude..."
│  │     ├─ print_tree_error "commits blocked globally"
│  │     ├─ print_instruction "ask your human to lift:" "  $ git.commit.uses allow --global"
│  │     └─ exit 2
│  └─ else continue to local quota check
└─ [○] retain extant commit flow
```

## test coverage

### integration tests (git.commit.uses.integration.test.ts)

| case | scenario | expected |
|------|----------|----------|
| block --global | in git repo | creates global blocker, stdout confirms |
| allow --global | blocker active | deletes global blocker, stdout confirms |
| allow --global | no blocker | no-op, stdout confirms |
| get --global | blocker active | shows "blocked" |
| get --global | no blocker | shows "not blocked" |
| block (local) | in git repo | sets quota to 0 |
| allow (local) | in git repo | sets unlimited quota |
| get | local quota + global blocker | shows local + "global: blocked" |
| get | local quota, no blocker | shows local only |

### integration tests (git.commit.set.integration.test.ts)

| case | scenario | expected |
|------|----------|----------|
| commit | global blocker active | exit 2, error message, instruction to lift |
| commit | global blocker active + local quota | exit 2, error message (global wins) |
| commit | no global, has local quota | commit succeeds |
| commit | global lifted after block | commit succeeds |
| commit | global blocker file corrupt (invalid json) | exit 2, "global blocker file corrupt" |

## contract summary

| command | scope | effect |
|---------|-------|--------|
| `block` | local | `del` alias — sets quota to 0 |
| `allow` | local | `set --quant 999999 --push allow` shorthand |
| `block --global` | global | creates `~/.rhachet/.../git.commit.uses.jsonc` with `{ blocked: true }` |
| `allow --global` | global | deletes global blocker file |
| `get --global` | global | shows blocker state |
| `get` | local + global | shows local quota; if global blocked, shows "global: blocked" |
