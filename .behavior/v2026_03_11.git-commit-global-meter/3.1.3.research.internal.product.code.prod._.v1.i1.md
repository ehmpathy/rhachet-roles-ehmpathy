## pattern.1 = local meter state storage

**[REUSE]** — the local `.meter/` directory pattern

### citations

[1] `git.commit.uses.sh:29-31`
```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
METER_DIR="$REPO_ROOT/.meter"
STATE_FILE="$METER_DIR/git.commit.uses.jsonc"
```

[2] `git.commit.uses.sh:146-150`
```bash
# findsert .meter dir and .gitignore
mkdir -p "$METER_DIR"
if [[ ! -f "$METER_DIR/.gitignore" ]]; then
  echo "*" > "$METER_DIR/.gitignore"

```

### relation to wish

the global blocker will use the same `.meter/` subdirectory convention, but at `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/` instead of `$REPO_ROOT/.meter/`.

[3] `git.commit.uses.sh:152-158`
```bash
# write state file
```

## pattern.5 = git repo requirement

**[EXTEND]** — local commands require git repo; global commands do not

### citations

[9] `git.commit.uses.sh:23-27`

## pattern.6 = command parsing (subcommand + flags)

**[EXTEND]** — add `block`, `allow` subcommands and `--global` flag

### citations

[10] `git.commit.uses.sh:45-94`
```bash
while [[ $# -gt 0 ]]; do
  case $1 in
    set|get|del)
      COMMAND="$1"
      shift
      ;;
    --quant)
      QUANT="$2"
      shift 2
      ;;
    --push)
      PUSH="$2"
      shift 2
      ;;
    # ... more options
  esac
done
```

### relation to wish

extend this parser to handle:
- `block` and `allow` as new subcommands
- `--global` flag to switch between local and global scope

## pattern.7 = del as shortcut for set

**[REUSE]** — `del` falls through to `set` logic

### citations

[11] `git.commit.uses.sh:106-112`
```bash
case "$COMMAND" in
  del)
    # del = revoke shortcut (quant=0, push=block)
    QUANT=0
    PUSH="block"
    VIA_DEL=true
    ;& # fall through to set logic
```

### relation to wish

`block` (local) will reuse this pattern — it's semantically equivalent to `del`.
