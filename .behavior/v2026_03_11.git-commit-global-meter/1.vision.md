# vision: global git.commit meter circuit breaker

## the outcome world

### before

a human grants a mechanic commit quota in repo A:
```sh
$ git.commit.uses set --quant 5 --push allow
```

later, the human realizes they need to pause all mechanic activity across their machine — maybe they step away, review work, or troubleshoot. they must:
1. remember every repo where they granted quota
2. visit each repo individually
3. run `git.commit.uses del` in each one
4. hope they didn't miss any

if they miss one, the mechanic can still commit there. no global kill switch exists.

### after

the human can now flip one switch:
```sh
$ git.commit.uses block --global
```

every repo on this machine immediately respects the global blocker. even if repo A has `--quant 5 --push allow` locally, the global blocker overrides it. the mechanic cannot commit anywhere until the human lifts the global blocker.

when ready to resume:
```sh
$ git.commit.uses allow --global

# local repo quotas are respected again
```

### the "aha" moment

> "i don't need to remember where i granted quota. one command, and every repo on my machine stops. one command, and they're all back."

the value clicks when you juggle 5 repos, step away for lunch, and realize you don't want any surprise commits while you're gone. `block --global` — done.

## user experience

### usecases

#### local (per-repo)

| goal | command | what happens |
|------|---------|--------------|
| grant quota | `git.commit.uses set --quant 3 --push allow` | writes to `$GIT_REPO/.meter/git.commit.uses.jsonc` |
| grant unlimited | `git.commit.uses allow` | shorthand for `set --quant @any --push allow` |
| revoke quota | `git.commit.uses block` | same as `del` — sets quota to 0 |
| check quota | `git.commit.uses get` | shows local quota + global state + effective |

#### global (all repos)

| goal | command | what happens |
|------|---------|--------------|
| pause all mechanics | `git.commit.uses block --global` | writes blocker to `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/git.commit.uses.jsonc` |
| resume all mechanics | `git.commit.uses allow --global` | removes global blocker file |
| check global state | `git.commit.uses get --global` | shows if global blocker is active |

### contract flow

```
mechanic wants to commit
  │
  ├─ check: global blocker exists at ~/.rhachet/storage/repo=$ROLE_REPO/role=$ROLE_SLUG/.meter/?
  │   └─ yes → BLOCKED (global circuit breaker active)
  │
  └─ no → check: local quota at $GIT_REPO/.meter/?
      ├─ has quota → ALLOWED
      └─ no quota → BLOCKED
```

**path clarification:**
- `$ROLE_REPO` = the repo that defines the role (e.g., `ehmpathy`)
- `$ROLE_SLUG` = the role name (e.g., `mechanic`)
- `$GIT_REPO` = the git repo where the mechanic operates (e.g., `~/git/ehmpathy/declastruct-aws`)

**rhachet global storage pattern:**
`~/.rhachet/storage/repo=$ROLE_REPO/role=$ROLE_SLUG/` is the standard rhachet global storage path for role-specific state. this feature uses it for `repo=ehmpathy/role=mechanic`:

- global blocker path: `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/git.commit.uses.jsonc`
- local quota path: `$GIT_REPO/.meter/git.commit.uses.jsonc` (e.g., `~/git/ehmpathy/declastruct-aws/.meter/git.commit.uses.jsonc`)

### timeline: roundtrip global block + allow

**9:00 AM — normal use, grant quota in two repos**
```sh
# in ~/repos/api

🐢 radical! let's ride!

🐚 git.commit.uses set
   ├─ granted: 3
   └─ push: allowed

🐢 gnarly! thanks human!

🐚 git.commit.uses set
   ├─ granted: 2
   └─ push: blocked
```

**10:30 AM — mechanic commits, quota decrements**
```sh
# mechanic commits in api repo
# quota: 3 → 2

# mechanic commits in web repo
# quota: 2 → 1
```

**11:00 AM — human needs to step away, activates global blocker**
```sh
$ git.commit.uses block --global
🐢 groovy, bond fire time

🐚 git.commit.uses block --global
   └─ commits blocked globally

# state at ~/.rhachet/storage/repo=$ROLE_REPO/role=$ROLE_SLUG/.meter/git.commit.uses.jsonc:
# { "blocked": true }
```

**11:05 AM — mechanic tries to commit (blocked)**
```sh
# mechanic in api repo attempts commit
🐢 bummer dude...

🐚 git.commit.set
   └─ error: commits blocked globally

ask your human to lift:
  $ git.commit.uses allow --global

# commit rejected
```

**12:30 PM — human returns, lifts global blocker**
```sh
$ git.commit.uses allow --global
🐢 shell yeah, back in the water!

🐚 git.commit.uses allow --global
   └─ commits resumed globally
```

**12:35 PM — mechanic can commit again**
```sh
# mechanic in api repo attempts commit
🐢 lets check the meter...

🐚 git.commit.uses
   └─ meter
      ├─ left: 2
      └─ push: allowed

# commit succeeds, quota: 2 → 1
```

**key observations from roundtrip:**
- local quota preserved through global blocker (2 uses in api, 1 in web)
- `block --global` sets the blocker; `allow --global` removes it
- mechanic sees clear status with `effective` field

### local block/allow (non-global)

**`block` (local) — revoke quota in current repo**
```sh
$ git.commit.uses block
🐢 groovy, break time

🐚 git.commit.uses del
   └─ revoked
```

**`allow` (local) — grant unlimited quota in current repo**
```sh
$ git.commit.uses allow
🐢 radical! let's ride!

🐚 git.commit.uses set
   ├─ granted: unlimited
   └─ push: allowed
```

## mental model

### how would users describe this to a friend?

> "it's like a master switch. each repo has its own thermostat, but there's a main breaker. flip the breaker, all stops. flip it back, each room goes back to its own set point."

### analogies

| concept | analogy |
|---------|---------|
| local quota | room thermostat |
| global blocker | main circuit breaker |
| override behavior | breaker > thermostat |

### terminology

| user might say | we say |
|----------------|--------|
| "kill switch" | global blocker |
| "pause all" | `block --global` |
| "resume" | `allow --global` |
| "per-repo quota" | local quota |

## evaluation

### how well does it solve the goals?

| goal | solved? | notes |
|------|---------|-------|
| pause all repos at once | ✅ yes | single command |
| resume without re-grant | ✅ yes | local quotas preserved |
| prevent accidental global allow | ✅ yes | no global allows, only global denies |
| simple mental model | ✅ yes | blocker overrides allow, always |

### pros

- **simplicity**: one command to pause all
- **safety**: global blocker is opt-in, explicit
- **predictability**: global blocker always wins (no complex precedence)
- **preservation**: local quotas untouched, just overridden
- **convention**: `~/.rhachet/storage/` is a dedicated state directory (note: not XDG compliant)

### cons

- **new concept**: users must learn global vs local
- **file location**: `~/.rhachet/storage/` is a new directory to manage
- **no per-repo global**: can't say "block only repo-a globally" (but this keeps it simple)

### edgecases

| edgecase | how we handle | pit of success |
|----------|---------------|----------------|
| global blocker + local allow | global wins | mechanic blocked, human must lift global |
| no global blocker + no local quota | local wins (no quota = blocked) | mechanic must ask human |
| global blocker file absent | no global blocker active | defaults to local-only behavior |
| corrupt global file | fail-fast, treat as blocked | mechanic blocked until human fixes |

## open questions & assumptions

### assumptions

1. **one global meter per user** — stored in `$HOME`, so per-user not per-machine
2. **`~/.rhachet/storage/` is writable** — standard home directory
3. **global blocker = blocked: true** — simplest model
4. **no global allow** — by design, to keep security tight
5. **block --global sets blocker** — `allow --global` removes it

### questions for wisher

1. **how to lift global blocker?** ✅ DECIDED: `allow --global`
   - new `block`/`allow` commands added as symmetric pair
   - `block --global` sets global blocker
   - `allow --global` lifts global blocker
   - cleaner than `del`/`lift` — natural opposites

2. **should `get` show both local and global?** ✅ DECIDED: yes.
   - users need to see WHY effective state differs from local
   - if local has quota but global denies, user must understand
   ```sh
   $ git.commit.uses get
   🐢 lets check the meter...

   🐚 git.commit.uses
      └─ meter
         ├─ left: 3
         ├─ push: allowed
         └─ global: blocked   ← blocks all
   ```

3. **storage path format?** ✅ DECIDED: namespaced by role repo.
   - pattern: `~/.rhachet/storage/repo=$ROLE_REPO/role=$ROLE_SLUG/.meter/`
   - example: `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/git.commit.uses.jsonc`

4. **can human lift global from anywhere?** ✅ DECIDED: must be in *some* git repo.
   - `--global` flag changes storage target, not execution context
   - human can run from any git repo to affect global state
   - simpler: no need for special "skip git repo check" logic

5. **is global blocker per-role or all-roles?** ✅ N/A — only mechanic supplies `git.commit.uses`.
   - no other role uses this skill
   - path includes `role=mechanic` for consistency with rhachet storage convention
   - the blocker is simply global

## what is awkward?

### ~~"del" creates a file~~ ✅ RESOLVED via `block`/`allow`

initial concern: `del --global` felt awkward because it creates a file.

**resolution**: new `block`/`allow` commands replace `del` for this use case:
- `block` (local) = same as `del` — sets quota to 0
- `block --global` = sets global blocker
- `allow` (local) = grants quota
- `allow --global` = lifts global blocker

the `block`/`allow` pair are natural opposites — cleaner semantics.

### ~~lift mechanism unclear~~ ✅ RESOLVED

decided: `allow --global` command.
- deletes the global blocker file
- returns system to local-only behavior
- symmetric with `block --global`

## summary

the vision: **a global circuit breaker for git.commit quota that lets humans pause all mechanics across all repos with one command, without loss of per-repo quota state.**

key design choices:
- global blocker only (no global allow)
- global blocker overrides local allow
- stored at `~/.rhachet/storage/repo=$ROLE_REPO/role=$ROLE_SLUG/.meter/`
- simple, predictable, safe
