# distillation: factory upgrades

## decision

**no factory work required** — extant factory is sufficient.

the test infrastructure already supports:
- `genTempDir({ git: true })` for temp repos
- `spawnSync` with custom `env` for `$HOME` injection
- BDD structure with `given`/`when`/`then` from `test-fns`
- snapshot assertions for stdout verification
- state file verification patterns

---

## blockers to address

| blocker | what to build | priority |
|---------|---------------|----------|
| none | — | — |

the `$HOME` isolation for global storage tests is solved via:
```typescript
spawnSync('bash', [skillPath, ...args], {
  env: { ...process.env, HOME: tempHomeDir },
});
```

this is a standard pattern already supported by `spawnSync`.

## improvements to make

| opportunity | what to build | benefit | decision |

## summary

proceed to implementation with extant test infrastructure. no upgrades needed.
