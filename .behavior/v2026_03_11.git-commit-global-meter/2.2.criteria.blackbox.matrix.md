# blackbox criteria matrix: global git.commit meter

## matrix.1 = commit attempt outcomes

the core behavior: what happens when mechanic tries to commit?

| ind: global blocker | ind: local quota | dep: commit result | dep: quota change |
|---------------------|------------------|--------------------|--------------------|
| active | has quota | BLOCKED | no change |
| active | no quota | BLOCKED | no change |
| none | has quota | ALLOWED | decrements |
| none | no quota | BLOCKED | no change |

**key insight**: global blocker wins over local quota. when global is active, local state is irrelevant for commit permission.

## matrix.3 = global command behavior

global commands require git repo context (same as local), but affect global storage.

| ind: in git repo | ind: command | dep: succeeds |
|------------------|--------------|---------------|
| yes | `block --global` | yes |
| yes | `allow --global` | yes |
| yes | `get --global` | yes |
| no | `block --global` | no (error) |
| no | `allow --global` | no (error) |
| no | `get --global` | no (error) |

**key insight**: `--global` changes storage target, not execution context. all commands require git repo.

## matrix.5 = blocker file state

what does `block`/`allow` do to file state?

| ind: command | ind: scope | dep: file action | dep: file location |
|--------------|------------|------------------|---------------------|
| `block` | local | writes `{uses: 0, push: "block"}` | `$GIT_REPO/.meter/` |
| `block` | global | writes `{blocked: true}` | `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/` |
| `allow` | local | writes `{uses: @any, push: "allow"}` | `$GIT_REPO/.meter/` |
| `allow` | global | deletes file | `~/.rhachet/storage/repo=ehmpathy/role=mechanic/.meter/` |

**key insight**: global `allow` deletes the file; global `block` creates it.
