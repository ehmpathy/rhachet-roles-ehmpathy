## check: code.prod/readable.narrative/rule.forbid.else-branches

### rule says (rule.forbid.else-branches.md)
> never use elses
> never use if elses
> use explicit ifs and early returns

### blueprint check (lines 62-77)

```
git.commit.set.sh
├─ [+] check global blocker (before local quota)
│  ├─ read global blocker file
│  │  ├─ if file absent → not blocked, continue
│  │  ├─ if file is invalid json → BLOCKED
│  │  │  └─ exit 2
│  │  └─ if { blocked: true }
│  │     └─ exit 2
│  └─ else continue to local quota check    ← CONCERN
└─ [○] retain extant commit flow
```

**concern**: line 75 says "else continue"

**analysis**:
- the "else" in the tree notation describes the fall-through path
- in actual code, this should be implemented WITHOUT an else keyword:

```bash
# check global blocker
if [[ -f "$GLOBAL_METER_FILE" ]]; then
  content=$(cat "$GLOBAL_METER_FILE")
  if ! echo "$content" | jq empty 2>/dev/null; then
    print_turtle_header "bummer dude..."
    print_tree_error "global blocker file corrupt"
    exit 2
  fi
  if echo "$content" | jq -e '.blocked == true' >/dev/null; then
    print_turtle_header "bummer dude..."
    print_tree_error "commits blocked globally"
    exit 2
  fi
fi
# fall-through: continue (NO ELSE KEYWORD)
```

**verdict**: ⚠️ **clarification added** — blueprint tree uses "else" as description; implementation MUST use fall-through without else keyword

## clarification: "else continue" in blueprint tree

### the concern

line 75 of blueprint says:
```
│  └─ else continue to local quota check
```

### rule.forbid.else-branches says
> never use elses
> use explicit ifs and early returns

### resolution

the "else" in the tree notation is **descriptive** (describes the fall-through path after early exits), not **prescriptive** (does not mean use else keyword).

**correct implementation pattern:**
```bash
# file absent check
if [[ ! -f "$GLOBAL_METER_FILE" ]]; then
  # not blocked, fall through
  :
fi

# corrupt file check
if [[ -f "$GLOBAL_METER_FILE" ]]; then
  content=$(cat "$GLOBAL_METER_FILE")
  if ! echo "$content" | jq empty 2>/dev/null; then
    print_tree_error "global blocker file corrupt"
    exit 2
  fi
  if echo "$content" | jq -e '.blocked == true' >/dev/null; then
    print_tree_error "commits blocked globally"
    exit 2
  fi
fi

# fall-through continues here (NO ELSE KEYWORD)
check_local_quota
```

**verdict**: blueprint tree notation is acceptable; implementation MUST NOT use else keyword
