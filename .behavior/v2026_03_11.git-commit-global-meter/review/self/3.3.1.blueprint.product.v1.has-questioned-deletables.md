# self-review: has-questioned-deletables

## components questioned

### 1. `block` command (local)

**question**: can this be removed?

**answer**: it is functionally identical to `del` — both set quota to 0. however, the wish explicitly uses "block" semantics, and `block`/`allow` form a cleaner mental model than `del`/`set`.

**decision**: keep as fall-through alias to `del` logic. no new code path — reuse extant `del` implementation.

---

### 2. `allow` command (local)

**question**: can this be removed?

**answer**: it is shorthand for `set --quant @any --push allow`. user could achieve the same with verbose `set` command.

**consideration**: `@any` sentinel is new — does it add complexity?

**simplification**: instead of `@any`, could use a large number like 999999. but `@any` is more explicit and prevents arbitrary limit.

**decision**: keep. symmetric pair with `block`. implementation is minimal (fall-through to `set` with preset values).

---

### 3. `--global` flag

**question**: can this be removed?

**answer**: no. this is the core feature requested. without it, no global circuit breaker.

**decision**: keep.

---

### 4. `get --global`

**question**: can this be removed?

**answer**: user could check global state via `ls ~/.rhachet/...` or infer from `get` output in a repo.

**consideration**: being able to check global state from any directory (non-git) is the usecase. this is valuable.

**decision**: keep.

---

### 5. `get` with global awareness

**question**: can this be simplified?

**answer**: the enhancement is minimal — one extra line "global: blocked" when blocker is active.

**decision**: keep. no simplification needed.

---

### 6. global blocker check in `git.commit.set`

**question**: can this be removed?

**answer**: no. this is how the global blocker actually prevents commits. without it, `block --global` would have no effect.

**decision**: keep.

---

## simplifications applied

1. **`block` (local) reuses `del` logic** — no new code path, just fall-through alias
2. **`allow` (local) reuses `set` logic** — no new code path, just preset values
3. **no `@any` complexity** — could use literal large number, but `@any` is clearer. keeping it simple.

---

## what could be deleted entirely?

**none.** the feature is minimal as designed:
- global blocker requires both the command (`block --global`) and the enforcement (check in `git.commit.set`)
- local `block`/`allow` are sugar but align with the vision's UX
- `get --global` enables checking state from anywhere

---

## final assessment

the blueprint is as simple as it can be while still delivering the wish. no components can be removed without breaking the feature.
