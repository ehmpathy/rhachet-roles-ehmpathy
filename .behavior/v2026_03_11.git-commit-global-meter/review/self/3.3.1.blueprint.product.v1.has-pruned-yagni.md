## deviation from vision

### `@any` vs `999999`

**vision says**: `allow` = `set --quant @any --push allow`
**blueprint says**: `allow` = `set --quant 999999 --push allow`

**why I changed it**: the extant code only accepts numeric quant. `@any` would require new string logic.

**is `999999` minimum viable?**: yes. it achieves "unlimited" without new logic.

**is `@any` minimum viable?**: no. it requires adding string detection and special decrement logic.

**decision**: `999999` is more minimum viable than `@any`. keep `999999`.
