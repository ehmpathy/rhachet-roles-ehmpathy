## question: should global blocker logic go in git.commit.operations.sh?

**searched**: `git.commit.operations.sh` for shared patterns

**found**: this file contains operations shared between `git.commit.set.sh` and `git.commit.push.sh`:
- `get_closest_ancestor_branch()` — used by both
- `get_behavioral_commits_on_branch()` — used by both
- `get_first_behavioral_commit_hash()` — used by both

**question**: should global blocker check be a shared operation?

**analysis**:
- `git.commit.set.sh` needs to check global blocker (line ~391, before local quota)
- `git.commit.uses.sh` needs to read/write global blocker (for block/allow --global)
- `git.commit.push.sh` does NOT check quota — it delegates that responsibility

**decision**: no. the global blocker check is NOT shared between set and push:
- `git.commit.push.sh` only checks if HEAD was authored by seaturtle[bot] (line 65-85)
- `git.commit.push.sh` does NOT decrement uses (documented at line 22: "does NOT decrement uses")
- the quota/blocker check is `git.commit.set`'s responsibility

therefore, global blocker logic should NOT go in `git.commit.operations.sh`.
