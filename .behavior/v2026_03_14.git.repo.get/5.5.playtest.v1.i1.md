# playtest: git.repo.get

## prerequisites

- rhachet-roles-ehmpathy installed and built (`npm run build`)
- at least one local clone in ~/git/ (e.g., ~/git/ehmpathy/domain-objects)
- gh cli authenticated (`gh auth status` should show logged in)

## sandbox

all playtest steps operate read-only on repos. no file operations needed.

---

## happy paths

### repos subcommand

**step 1: list repos by org**

```sh
rhx git.repo.get repos --repos 'ehmpathy/*'
```

expected outcome:
- header shows `far out`
- output shows repos grouped by org
- local repos show `(local)` indicator
- output ends with `found: N repos`

**step 2: list repos by name pattern**

```sh
rhx git.repo.get repos --repos 'sql-*'
```

expected outcome:
- matches repos with sql- prefix from any org
- repos with local clones show `(local)`

---

### files subcommand

**step 3: list files in local repo**

```sh
rhx git.repo.get files --in ehmpathy/domain-objects
```

expected outcome:
- header shows `far out`
- output shows file tree structure
- shows `(local)` indicator
- ends with `found: N files`

**step 4: list files with path filter**

```sh
rhx git.repo.get files --in ehmpathy/domain-objects --paths '**/*.test.ts'
```

expected outcome:
- only shows files that match the glob pattern
- file count is smaller than full tree

---

### lines subcommand (search mode)

**step 5: search for pattern**

```sh
rhx git.repo.get lines --in ehmpathy/domain-objects --words 'DomainEntity'
```

expected outcome:
- header shows `far out`
- matches show with `>` prefix on match lines
- context lines shown around each match (21 lines default)
- ends with `found: N matches in M files`

**step 6: search with custom radius**

```sh
rhx git.repo.get lines --in ehmpathy/domain-objects --words 'DomainEntity' --radius 3
```

expected outcome:
- context limited to 3 lines before/after each match
- fewer lines shown per match than step 5

---

### lines subcommand (read mode)

**step 7: read file content**

```sh
rhx git.repo.get lines --in ehmpathy/domain-objects --paths 'package.json'
```

expected outcome:
- shows full file content (no `--words` = read mode)
- line numbers shown
- ends with `found: 1 file`

---

## edgey paths

### no matches

**step 8: repos with no matches**

```sh
rhx git.repo.get repos --repos 'nonexistent-org-xyz/*'
```

expected outcome:
- header shows `crickets...`
- output shows `found: 0 repos`

**step 9: search with no matches**

```sh
rhx git.repo.get lines --in ehmpathy/domain-objects --words 'XYZNONEXISTENTPATTERN123'
```

expected outcome:
- header shows `crickets...`
- output shows `found: 0 matches`

### error cases

**step 10: unknown repo**

```sh
rhx git.repo.get files --in ehmpathy/unknown-repo-xyz
```

expected outcome:
- header shows `bummer dude`
- shows error message about repo not found

**step 11: repos default behavior**

```sh
rhx git.repo.get repos
```

expected outcome:
- defaults to `--repos '*/*'`
- lists all local repos
- header shows `far out`

---

## pass/fail criteria

- ✓ pass if: all 11 steps produce expected outcomes described above
- ✓ pass if: local repos show `(local)` indicator consistently
- ✓ pass if: error cases show appropriate turtle vibes headers
- ✗ fail if: any command prompts for permission (should be pre-approved)
- ✗ fail if: search results lack context lines
- ✗ fail if: read mode shows truncated content
