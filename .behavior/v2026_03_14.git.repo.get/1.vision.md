# vision: git.repo.get

## skill header

```sh
######################################################################
# git.repo.get — search and browse repos without permission prompts
#
# .what
#   access any repo you can reach (local clones or github)
#   local-first, cloud fallback, no permission prompts
#
# .when
#   - "how did another repo implement this pattern?"
#   - "what repos/files exist in this org?"
#   - "show me that file from another repo"
#
# .how
#   repos  — list repos
#     rhx git.repo.get repos --repos 'ehmpathy/*'
#     rhx git.repo.get repos --repos 'sql-*'
#
#   files  — list files in repo
#     rhx git.repo.get files --in ehmpathy/domain-objects
#     rhx git.repo.get files --in ehmpathy/domain-objects --paths '**/*.test.ts'
#     rhx git.repo.get files --in ehmpathy/domain-objects --words 'DomainEntity'
#     rhx git.repo.get files --in ehmpathy/domain-objects --paths 'src/**/*.ts' --words 'export'
#
#   lines  — search or read file contents
#     search: rhx git.repo.get lines --in ehmpathy/domain-objects --words 'DomainEntity'
#     read:   rhx git.repo.get lines --in ehmpathy/domain-objects --paths 'src/index.ts'
#
#   --words triggers search mode (21 lines radius by default)
#   no --words triggers read mode (full file)
#   --ref specifies branch/tag/sha (default: origin/main)
#
# .scope
#   --repos 'ehmpathy/*'             all ehmpathy repos
#   --repos 'ehmpathy/*-roles-*'     ehmpathy *-roles-* repos
#   --repos 'sql-*'                  sql-* in any org
#   --repos '@local/ehmpathy/*'      local only
#   --repos '@cloud/ehmpathy/*'      cloud only
#
######################################################################
```

---

## the outcome world

### before

- mechanic prompts permission for every `gh` or `git` command that accesses external repos
- searches fail silently or return incomplete results
- unclear which repos mechanic can access — trial and error
- context switch: "let me check if i have access to that repo..."
- local clones may be stale; github api may be rate-limited

### after

- mechanic runs `rhx git.repo.get lines --repos 'ehmpathy/*' --words 'DomainEntity'`
- no permission prompts — skill is pre-approved
- results come from local clones first, cloud fallback
- mechanic runs `repos --repos 'ehmpathy/*'` to see what's available
- reliable, fast, consistent experience

### the "aha" moment

> "wait, you just searched this repo instantly without a prompt?"

the value clicks when mechanic seamlessly pulls code context from a repo to inform a decision — no friction, no prompts, no failed searches.

**note**: `repos` lists all repos; `files` and `lines` operate on single repo. mechanic can invoke multiple times for cross-repo work. multi-repo `lines` is a stretch goal.

## user experience

### usecases

| goal | command | result |
|------|---------|--------|
| list repos | `rhx git.repo.get repos --repos 'ehmpathy/*'` | ehmpathy repos (local-first) |
| list repos by name | `rhx git.repo.get repos --repos 'sql-*'` | sql-* in any org |
| list files | `rhx git.repo.get files --in ehmpathy/domain-objects` | file tree for repo |
| find files by name | `rhx git.repo.get files --in ehmpathy/domain-objects --paths '**/*.test.ts'` | files that match glob |
| find files by content | `rhx git.repo.get files --in ehmpathy/domain-objects --words 'DomainEntity'` | files with pattern |
| find files (both) | `rhx git.repo.get files --in ehmpathy/domain-objects --paths 'src/**/*.ts' --words 'export'` | ts files with exports |
| search lines | `rhx git.repo.get lines --repos 'ehmpathy/*' --words 'DomainEntity'` | grep across repos |
| search lines | `rhx git.repo.get lines --in ehmpathy/domain-objects --words 'DomainEntity'` | matches with 21 lines around (default) |
| less context | `rhx git.repo.get lines --in ehmpathy/domain-objects --words 'DomainEntity' --radius 5` | matches with 5 lines around |
| read file | `rhx git.repo.get lines --in ehmpathy/domain-objects --paths 'src/DomainEntity.ts'` | all lines (no --words = read mode) |
| search + filter | `rhx git.repo.get lines --repos 'sql-*' --paths 'src/**/*.ts' --words 'export'` | exports in sql-* repos |

### source scopes

| scope | source | when to use |
|-------|--------|-------------|
| `@local` | local clones in ~/git/ | fast, no network |
| `@cloud` | github via gh cli | no local clone |
| `@all` | local-first, cloud fallback | best of both (default) |

note: `--repos glob` without `@scope/` prefix defaults to `@all`.

### filters

| filter | effect |
|--------|--------|
| `--paths <glob>` | limit to paths that match glob |
| `--words <pattern>` | limit to items with pattern |
| `--radius <N>` | show N lines before/after each match (default: 21) |

**note**: `lines` has two modes:
- **search mode**: `--words 'pattern'` → returns matches with 21 lines context (default)
- **read mode**: no `--words` → returns all lines in file (like cat/read)

### contract shape

```sh
# repos - enumerate repos
rhx git.repo.get repos \
  --repos <[@scope/]glob> \
  [--paths <glob>] \
  [--words <pattern>]

# files - enumerate files
rhx git.repo.get files \
  --in <owner/name> | --repos <[@scope/]glob> \
  [--paths <glob>] \
  [--words <pattern>] \
  [--ref <branch|tag|sha>]  # default: origin/main

# lines - get lines
# - with --words: search mode (returns matches + 21 lines context)
# - without --words: read mode (returns all lines in file)
rhx git.repo.get lines \
  --in <owner/name> | --repos <[@scope/]glob> \
  [--paths <glob>] \
  [--words <pattern>]       # omit for read mode (all lines)
  [--radius <N>]           # search mode only: context lines (default: 21)
  [--ref <branch|tag|sha>]       # default: origin/main

# @scope = @local | @cloud | @all (default: @all)
```

### --in vs --repos

**common usage (no @scope = @all = local-first, cloud fallback):**

| flag | scope | example |
|------|-------|---------|
| `--in owner/name` | single repo | `--in ehmpathy/domain-objects` |
| `--repos 'org/*'` | all repos in org | `--repos 'ehmpathy/*'` |
| `--repos 'name-*'` | repos by name in any org | `--repos 'sql-*'` |
| `--repos 'org/name-*'` | repos by name in org | `--repos 'ehmpathy/domain-*'` |

**explicit scope (when you need it):**

| flag | scope | example |
|------|-------|---------|
| `--repos '@local/glob'` | local only | `--repos '@local/ehmpathy/*'` |
| `--repos '@cloud/glob'` | cloud only | `--repos '@cloud/*/domain-*'` |

**glob patterns for --repos:**

| pattern | matches |
|---------|---------|
| `ehmpathy/*` | all ehmpathy repos |
| `ehmpathy/domain-*` | ehmpathy domain-* repos |
| `sql-*` | sql-* repos in any org (same as `*/sql-*`) |
| `domain-*` | domain-* repos in any org |
| `@local/ehmpathy/*` | local ehmpathy repos only |
| `@cloud/*/sql-*` | cloud sql-* repos in any org |

**internal model:** each repo has a fully qualified slug `@scope/org/repo`. the `--repos` pattern globs against these slugs:

```
@local/ehmpathy/domain-objects
@local/ehmpathy/sql-code-generator
@cloud/ehmpathy/new-library
@cloud/ahbode/protools-api
```

- `@local/ehmpathy/*` → matches `@local/ehmpathy/domain-objects`, etc.
- `ehmpathy/*` → expands to `@all/ehmpathy/*` → matches local-first, then cloud
- `sql-*` → expands to `@all/*/sql-*` → matches sql-* in any org

### timeline

**typical flow: "how did another repo implement this?"**

1. **search**: `lines --repos 'ehmpathy/*' --words 'pattern'` → matches with 21 lines context
2. **narrow**: `lines --in owner/name --words 'pattern'` → single repo, same context
3. **read**: `lines --in owner/name --paths 'path/to/file'` → full file (no --words = read mode)

**discovery flow: "what repos/files exist?"**

1. **repos**: `repos --repos 'ehmpathy/*'` → list repos
2. **files**: `files --in owner/name` → browse file tree
3. **lines**: `lines --in owner/name --paths 'src/**/*.ts'` → explore files

## mental model

### the slug model

every repo has a fully qualified slug: `@scope/org/repo`

```
@local/ehmpathy/domain-objects
@local/ehmpathy/sql-code-generator
@cloud/ehmpathy/new-library
@cloud/ahbode/protools-api
```

`--repos` is just a glob against these slugs. most common usage omits scope (defaults to `@all`):

```sh
--repos 'ehmpathy/*'      # typical: all ehmpathy repos, local-first
--repos 'sql-*'           # typical: sql-* in any org, local-first
--repos 'domain-objects'  # typical: exact repo name, any org
```

explicit scope only when needed:
```sh
--repos '@local/ehmpathy/*'   # local only (no cloud fallback)
--repos '@cloud/*/sql-*'      # cloud only (skip local)
```

### how users describe it

> "it's like all our repos locally, always up to date, searchable instantly"

> "one command to search any repo we have access to — no git clone, no gh auth dance"

### analogies

- **like ripgrep but for any repo you can access** — local or remote, same interface
- **like a cached mirror** — local when possible, github when needed
- **like glob for repos** — `--repos 'ehmpathy/*'` just works

### terminology

| user says | we say |
|-----------|--------|
| "search ehmpathy repos" | `git.repo.get lines --repos 'ehmpathy/*' --words '...'` |
| "show me that file" | `git.repo.get lines --in owner/name --paths 'path/to/file'` |
| "list the files" | `git.repo.get files --in owner/name` |
| "what repos do i have?" | `git.repo.get repos --repos 'ehmpathy/*'` |
| "search local only" | `git.repo.get lines --repos '@local/ehmpathy/*' --words '...'` |

## evaluation

### how well does it solve the goals?

| goal | solution |
|------|----------|
| stop permission prompts | skill is pre-approved in permissions |
| stop failed searches | local-first with cloud fallback (default behavior) |
| more reliable search | consistent interface via repos/files/lines |
| clarity on accessible repos | `repos --repos 'org/*'` shows what's available |

### pros

- **zero friction**: no permission prompts for repo access
- **fast**: local-first by default
- **flexible**: explicit @local or @cloud when needed
- **discoverable**: `repos --repos 'org/*'` shows what's available
- **consistent**: same filters (--paths, --words) across all commands
- **glob semantics**: `--repos 'sql-*'` just works

### cons

- **local clone requirement**: for local results, repos must be cloned in ~/git/
- **storage**: local clones take disk space
- **staleness risk**: if local clone not fetched recently, results may lag

### pit of success

- default ref is `origin/main` — consistent baseline
- default scope is `@all` — local-first, cloud fallback
- no separate index to maintain — glob against slug model
- same filters (--paths, --words) across all commands
- clear error with suggestion for broader search

## what is awkward?

### things that feel off

1. ~~**"local if local"**~~ — **solved**: scan ~/git/ on demand via `repos @local`
2. ~~**"checkout origin/main version first"**~~ — **solved**: use `git show origin/main:path` and `git grep pattern origin/main` instead of checkout. reads from ref without worktree modification.
3. ~~**index maintenance**~~ — **solved**: no separate index; scan ~/git/ each time

### design vs mental model friction

- users say "search" but we say `lines --words` — slight indirection
- `lines` without `--words` shows all lines (read mode) — may confuse
- glob pattern `'sql-*'` vs exact repo `--in ehmpathy/sql-code-generator` — two ways to specify

### uncomfortable tradeoffs

| tradeoff | tension |
|----------|---------|
| freshness vs speed | fetch before search = slow; skip fetch = stale |
| local vs cloud | local = fast but may not exist; cloud = always available but slow |
| disk usage | more local clones = faster search = more disk |
