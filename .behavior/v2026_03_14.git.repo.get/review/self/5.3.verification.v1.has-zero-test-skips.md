the original hook code:
```bash
if [[ -f ~/.config/rhachet/apikeys.env ]]; then

fi
```

the fixed hook code:

```bash
fi
```

the `${XAI_API_KEY+x}` syntax checks if the variable is set at all (even if empty). now:
- if test sets `XAI_API_KEY=''`, the hook respects it and does not source apikeys.env
- if XAI_API_KEY is unset, the hook sources apikeys.env as fallback

this fix is in production code, not test code. the test assertion was correct — it caught a real bug.
