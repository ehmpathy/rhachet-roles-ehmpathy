**vision says**: "21 lines radius by default"

**issue**: the has-questioned-assumptions review changed the default from 21 to 10, but this diverges from the declared behavior.

1. update vision + criteria to say 10 (requires wisher approval)

**fix applied**: update blueprint from `default=10` to `default=21`.

## why coverage holds

### why the three subcommands are fully specified

each subcommand in the blueprint has a complete codepath:

1. **repos**: arg parse → cmd_repos → parse_repos_glob → get_all_local_repos/get_all_cloud_repos → filter_by_glob → output_repos

2. **files**: arg parse → cmd_files → lookup repo → get_files_at_ref → filter_by_glob (paths) → filter_by_content (words) → output_files

3. **lines**: arg parse → cmd_lines → lookup repo → (search mode: search_lines → expand_context → output_matches) OR (read mode: read_file → output_content)

each codepath traces from input to output. no steps are absent. the operations.sh file provides all the build blocks.

### why the scope system is complete

the slug model `@scope/org/repo` enables all three scopes:
- `@local` → get_all_local_repos (find ~/git -maxdepth 2 -name .git)
- `@cloud` → get_all_cloud_repos (gh repo list)
- `@all` → merge(local, cloud), dedupe, local-first

parse_repo_slug extracts the scope from --repos patterns. cmd_repos dispatches to the right source based on scope.

### why the fallback behavior is guaranteed

the lookup operation in cmd_files and cmd_lines does:
1. check $GIT_ROOT/org/repo exists
2. if found: return local path
3. if not found: gh repo view org/repo --json url
4. return source (local/cloud) + path/url

step 2 must happen before step 3. this ensures local-first semantics.

### why permission coverage is complete

the blueprint specifies two entries in init.claude.permissions.jsonc:
```
"Bash(npx rhachet run --skill git.repo.get:*)"
"Bash(rhx git.repo.get:*)"
```

the `:*` wildcard means any subcommand and flags are pre-approved. this covers all usecases without individual entries per command.
