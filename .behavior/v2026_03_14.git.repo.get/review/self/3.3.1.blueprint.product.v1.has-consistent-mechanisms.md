### mechanism 2: arg parse

**what blueprint says**:
```
├─ [+] arg parse
│   ├─ ignore rhachet args (--repo, --role, --skill)
│   ├─ extract subcommand (repos, files, lines)
│   └─ extract flags (--repos, --repo, --paths, --words, --radius, --ref)
```

**does codebase have this?** yes — git.commit.set.sh has the pattern:
```bash
while [[ $# -gt 0 ]]; do
  case $1 in
    --skill|--repo|--role) shift 2 ;;  # ignore rhachet args
    --mode) MODE="$2"; shift 2 ;;
    ...
  esac
done
```

**verdict**: pattern reuse, not duplication. we follow the same case/esac structure.

**why it holds**: arg parse is not a shared function — it's a pattern each skill implements. the blueprint follows the established pattern from git.commit.set.sh.
