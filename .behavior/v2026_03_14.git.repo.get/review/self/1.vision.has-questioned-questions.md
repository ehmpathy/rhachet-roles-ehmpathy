### answered questions

**Q4: what about private repos?**
- answer: gh auth handles this transparently
- no filter needed — if gh can read it, skill can access it
- if gh can't read it, skill fails with clear error

**Q7: what does "accessible" mean?**
- answer: indexed AND (locally cloned OR gh-readable)
- first check local clone exists
- if not, fall back to gh api
- if neither works, repo is not accessible

**Q9: default branch detection?**
- answer: auto-detect via:
  ```sh
  git remote show origin | grep 'HEAD branch' | awk '{print $NF}'
  ```
- or via gh: `gh repo view owner/repo --json defaultBranchRef -q '.defaultBranchRef.name'`
- no need to assume `main` — detect it
