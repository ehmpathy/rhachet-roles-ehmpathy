## potential backcompat candidates reviewed

### candidate 1: gh cli version compatibility

**what we might assume**: "support older gh cli versions just to be safe"

**was this requested?** no. wisher did not specify gh cli version requirements.

**evidence needed?** no — we use standard gh cli commands (`gh repo list`, `gh repo view`). these have been stable since gh cli 1.0.

**verdict**: no backcompat concern. use current gh cli semantics. if an old version fails, the error message will be clear.

**why it holds**: the wish says "from github if accessible, via cli" — this implies standard gh cli usage, not legacy support. users who have gh installed have recent versions (gh self-updates).
