## issue found: pre-index may not be needed

**location**: index subcommand design

**problem**: i assumed repos must be pre-indexed before use. but what if:
- first search of `ehmpathy/domain-objects` auto-discovers it
- adds to index if successful
- no upfront `--refresh` needed

**benefits**:
- zero setup friction
- index grows organically
- only repos actually used get indexed

**tradeoffs**:
- first access slower (discovery overhead)
- no upfront "what can i access?" view until used

**verdict**: add to open questions. wisher should decide if upfront index is required or on-demand is acceptable.
