# playtest: git.branch.rebase dispatcher

## prerequisites

- repo with git.branch.rebase skills built (`npm run build`)
- rhachet roles linked (`npx rhachet roles link --role mechanic`)
- terminal access

## sandbox

all commands are safe read-only operations except for `rhx git.branch.rebase begin --mode apply` which requires a feature branch.

## happy paths

### 1. help subcommand

**action:**
```sh
rhx git.branch.rebase help
```

**expected outcome:**
- turtle emoji header
- tree format output shows:
  - subcommands: begin, continue, abort, help
  - examples section with copy-paste commands
- exit code 0

### 2. --help flag

**action:**
```sh
rhx git.branch.rebase --help
```

**expected outcome:**
- same output as `rhx git.branch.rebase help`
- exit code 0

### 3. -h flag

**action:**
```sh
rhx git.branch.rebase -h
```

**expected outcome:**
- same output as `rhx git.branch.rebase help`
- exit code 0

### 4. begin subcommand (plan mode)

**action:**
```sh
rhx git.branch.rebase begin --mode plan
```

**expected outcome:**
- if on feature branch: shows plan preview of rebase
- if on main: shows error "cannot rebase main/master"
- dispatches to git.branch.rebase.begin.sh (not dispatcher output)

### 5. unknown subcommand

**action:**
```sh
rhx git.branch.rebase invalid
```

**expected outcome:**
- error message: "unknown subcommand: invalid"
- shows help with valid subcommands
- exit code 1

### 6. no subcommand

**action:**
```sh
rhx git.branch.rebase
```

**expected outcome:**
- error message: "subcommand required"
- shows help with valid subcommands
- exit code 1

## edgey paths

### rhachet arg passthrough

**action:**
```sh
npx rhachet run --skill git.branch.rebase --repo ehmpathy --role mechanic help
```

**expected outcome:**
- rhachet args (--skill, --repo, --role) are filtered out
- help subcommand is recognized and executed
- shows help output

### subcommand args passthrough

**action:**
```sh
rhx git.branch.rebase begin --mode apply
```

**expected outcome:**
- --mode apply is passed to git.branch.rebase.begin.sh
- if permissions granted: executes rebase
- if no permissions: shows permission error from subskill

## pass/fail criteria

### pass if:
- `rhx git.branch.rebase help` shows turtle vibes tree with subcommands
- `rhx git.branch.rebase --help` and `-h` produce identical output to `help`
- `rhx git.branch.rebase begin` dispatches to begin subskill (output differs from dispatcher)
- `rhx git.branch.rebase invalid` shows error with "invalid" in message
- `rhx git.branch.rebase` (no args) shows error with "required" in message

### fail if:
- any happy path command crashes with stack trace
- help output does not show all four subcommands
- error messages do not guide user to valid options
- dispatcher output appears where subskill output should appear
