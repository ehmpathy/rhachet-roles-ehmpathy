# 3.3.1.blueprint.product.v1: git.branch.rebase dispatcher

## summary

create `git.branch.rebase.sh` dispatcher skill that routes to subskills based on first positional argument:
- `begin` → exec git.branch.rebase.begin.sh
- `continue` → exec git.branch.rebase.continue.sh
- `abort` → exec git.branch.rebase.abort.sh
- `help` → show usage and exit

the dispatcher:
1. filters out rhachet args (--skill, --repo, --role)
2. extracts subcommand from first positional arg
3. passes rest of args to subskill via exec
4. shows help and errors with turtle vibes

## codepath tree

### git.branch.rebase.sh dispatcher

```
git.branch.rebase.sh
├── [+] header block (.what, .why, usage, guarantee)
├── [←] set -euo pipefail
├── [←] SCRIPT_DIR detection (reuse from git.commit.uses.sh)
├── [+] ARGS array initialization
├── [+] arg parse loop
│   ├── [←] rhachet arg filter (reuse from git.branch.rebase.begin.sh)
│   │   └── case --skill|--repo|--role: shift 2
│   ├── [+] case --help|-h: show_usage && exit 0
│   └── [+] case *: capture as SUBCMD or append to ARGS
├── [+] show_usage function
│   ├── [←] source output.sh (reuse from git.commit/)
│   └── [+] print subcommands list
├── [+] validate SUBCMD
│   ├── [+] if empty: error "subcommand required", show_usage, exit 1
│   └── [+] if unknown: error "unknown subcommand: $SUBCMD", show_usage, exit 1
└── [+] dispatch
    └── [+] exec "$SCRIPT_DIR/git.branch.rebase.$SUBCMD.sh" "${ARGS[@]}"
```

## contracts

### dispatcher input contract

```
git.branch.rebase <subcommand> [options]

subcommands:
  begin     start rebase onto origin/main
  continue  continue rebase after conflicts settled
  abort     abandon rebase, restore to pre-rebase state
  help      show usage and available subcommands

options:
  --help, -h    show help
```

### dispatcher output contract

| scenario | output | exit code |
|----------|--------|-----------|
| valid subcommand | (subskill output) | (from subskill) |
| help | usage message | 0 |
| no subcommand | error + subcommands | 1 |
| unknown subcommand | error + subcommands | 1 |
