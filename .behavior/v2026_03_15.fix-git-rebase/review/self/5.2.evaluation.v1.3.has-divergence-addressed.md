### skeptical review of each backup

#### 1. SKILL_DIR vs SCRIPT_DIR

**skeptic question**: is this laziness or a real improvement?

**evidence**:
```
$ grep -n "SKILL_DIR\|SCRIPT_DIR" src/domain.roles/mechanic/skills/git.branch.rebase/*.sh | head -10
git.branch.rebase.abort.sh:17:SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
git.branch.rebase.begin.sh:22:SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
git.branch.rebase.continue.sh:19:SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
git.branch.rebase.operations.sh:15:SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
git.branch.rebase.sh:24:SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
```

all 5 files in git.branch.rebase/ use SKILL_DIR. SCRIPT_DIR would be the outlier.

**verdict**: backup justified — consistency within namespace trumps cross-namespace consistency.

#### 2. early continue pattern

**skeptic question**: did we avoid the else because it was hard to write?

**evidence**:
```
$ grep -n "else" src/domain.roles/mechanic/skills/git.branch.rebase/git.branch.rebase.sh
(no else found)
```

- rule.forbid.else-branches is a mechanic role standard
- the early continue pattern is explicitly documented in rule.prefer.early-returns.[demo].md
- both patterns produce identical behavior
- verified: `grep -n "else" git.branch.rebase.sh` returns empty

**verdict**: backup justified — required by role standards.

#### 3. test case reorder

**skeptic question**: is test reorder just preference or does it matter?

**evidence**:
- blueprint order: dispatch cases first, then help, then errors
- implementation order: help, errors, dispatch, passthrough
- implementation groups by behavior type, not by feature
- this makes test narrative clearer (simple → complex)

**verdict**: backup justified — test order by complexity aids comprehension.

#### 4. setupMinimalRepo vs genTempDir

**skeptic question**: did we use setupMinimalRepo because genTempDir was too hard to configure?

**evidence**:
- genTempDir({ git: true }) creates empty git repo
- subskills (git.branch.rebase.begin) check meter state for permissions
- meter state requires .meter/git.commit.uses.jsonc
- setupMinimalRepo creates this state

**verdict**: backup justified — genTempDir insufficient for meter-dependent subskills.

#### 5. real subskills vs mocks

**skeptic question**: did we skip mocks because they're harder to write?

**evidence**:
```
$ grep -n "jest.mock\|jest.fn" git.branch.rebase.integration.test.ts
(no mocks found)
```

- rule.forbid.mocks in mechanic role standards
- real subskills produce actual error messages
- dispatch verified by expected errors from real subskills
- no behavior is mocked away
- verified: `grep -n "jest.mock\|jest.fn"` returns empty

**verdict**: backup justified — required by role standards.

#### 6. contains vs snapshot

**skeptic question**: are we just lazy about snapshot maintenance?

**evidence**:
- rule.forbid.redundant-expensive-operations requires single call per then block
- snapshot would require separate call to capture output
- contains assertions verify structure without redundant invocation

**verdict**: backup justified — required by role standards.
