## issue found: false consistency claim

**the claim (line 109)**
> "consistency: matches git.commit pattern (git.commit.set, git.commit.push, git.commit.uses)"

**what i assumed without evidence**
that git.commit has a dispatcher pattern like the proposed git.branch.rebase.

**the truth**
there is no `git.commit.sh` file. git.commit.set, git.commit.push, git.commit.uses are separate skills invoked directly, not dispatched.

**why this matters**
a new `git.branch.rebase.sh` would create INCONSISTENCY, not consistency.
git.commit has no dispatcher. git.branch.rebase would have a dispatcher.

**what if the opposite were true?**
if we wanted true consistency, we would NOT add a dispatcher to git.branch.rebase.
or we would add dispatchers to BOTH git.commit and git.branch.rebase.

**fix applied**
updated vision line 109:
- before: `"consistency: matches git.commit pattern (git.commit.set, git.commit.push, git.commit.uses)"`
- after: `"new pattern: establishes dispatcher pattern (git.commit skills are invoked directly, this adds unified entry)"`

lesson: verify claims about extant patterns before asserting consistency.

## issue found: inaccurate mental model claim

**the claim (lines 33-37)**
> "the pattern mirrors natural git mental model"
> "git rebase --continue becomes rhx git.branch.rebase continue"

**what i assumed without evidence**
that a flag-to-subcommand map is "no cognitive translation."

**the truth**
native git uses FLAGS: `git rebase --continue`, `git rebase --abort`
proposed skill uses SUBCOMMANDS: `git.branch.rebase continue`, `git.branch.rebase abort`

these are different patterns. user must learn:
- `--continue` → `continue` (flag becomes subcommand)
- `git rebase origin/main` → `git.branch.rebase begin` (implicit start becomes explicit)

**why this matters**
the "aha" moment is overstated. there IS cognitive translation required.

**fix applied**
updated vision lines 33-37:
- before: `"the pattern mirrors natural git mental model"` + `"no cognitive translation needed"`
- after: `"the pattern is similar to git's mental model"` + `"the translation is simple but not zero-cost"`

lesson: be honest about cognitive cost. similar is not identical.
