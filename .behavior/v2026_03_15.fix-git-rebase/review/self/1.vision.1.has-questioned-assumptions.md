## assumption 1: rhachet maps `git.branch.rebase begin` → `git.branch.rebase.sh begin`

**what do we assume here without evidence?**
that rhachet passes `begin` as first positional arg to `git.branch.rebase.sh`.

**what evidence supports this?**
- git.commit.uses.sh receives positional args and dispatches
- git.branch.rebase.begin.sh shows usage `rhx git.branch.rebase begin`

**what if the opposite were true?**
if rhachet maps `git.branch.rebase begin` directly to `git.branch.rebase.begin.sh`, then
dispatcher may never be invoked for subcommand calls.

**did the wisher actually say this?**
no - wisher said "dispatch to subskills", not how dispatch happens.

**verdict: needs verification**
must verify how rhachet skill lookup works before implementation.
added to research needed section.

## assumption 3: users want space-separated subcommands

**what do we assume here without evidence?**
that `rhx git.branch.rebase begin` is preferred over `rhx git.branch.rebase.begin`.

**what evidence supports this?**
- native git uses spaces: `git rebase --continue`
- current skill docs show space pattern

**what if the opposite were true?**
if users prefer dots, dispatcher adds no value - rhachet already looks up dots.

**what exceptions exist?**
some users may type dots out of habit. both should work.

**verdict: holds with caveat**
space pattern preferred for mental model, but dots should remain functional.
