# execution summary

## implemented

`src/roles/mechanic/.skills/init.bhuild.sh`

a shell script that scaffolds a `.behavior/v${isodate}.${behaviorname}` thoughtroute directory

## usage

```bash
init.bhuild.sh --name <behaviorname> [--dir <directory>]
```

## features

- `--name <behaviorname>` (required): name of the behavior
- `--dir <directory>` (optional): target directory, defaults to `$PWD`
  - auto-trims trailing `.behavior` if provided

## behavior

- creates `.behavior/v${isodate}.${behaviorname}/` directory
- findserts (find-or-insert) each file:
  - `[KEEP]` if file exists
  - `[CREATE]` if file missing
- idempotent: safe to rerun

## scaffolded files

| file                                     | purpose                             |
| ---------------------------------------- | ----------------------------------- |
| `0.wish.md`                              | wish definition                     |
| `1.vision.md`                            | vision statement                    |
| `2.criteria.md`                          | given/when/then acceptance criteria |
| `3.1.research.domain._.v1.src`           | domain research prompt              |
| `3.2.distill.domain._.v1.src`            | domain distillation prompt          |
| `3.3.blueprint.v1.src`                   | implementation blueprint prompt     |
| `4.1.roadmap.v1.src`                     | execution roadmap prompt            |
| `5.1.execution.phase0_to_phaseN.v1.src`  | execution tracking prompt           |
| `.ref.[feedback].v1.[given].by_human.md` | feedback template                   |

## path handling

- file contents use relative paths from caller's `$PWD`
- computed via `realpath --relative-to="$PWD" "$TARGET_DIR"`
